## 为什么重新开始

这版博客先不追求完整系统，也不急着把所有功能一次做满。目标很简单：有一个可以放在 GitHub Pages 上的个人主页，有一个文章列表，有一个能稳定阅读 Markdown 的文章页。

技术选择也刻意收敛：

- 不需要后端。
- 不需要数据库。
- 不需要构建步骤。
- 文章直接写在 `posts/` 目录里。

## 先把写作链路跑通

真正重要的是新增文章足够轻。理想状态是：

1. 新建一个 Markdown 文件。
2. 在 `assets/js/posts.js` 里登记标题、日期、摘要和标签。
3. 推到 GitHub Pages。

后面如果文章数量变多，再考虑搜索、标签归档、RSS 和评论。

## 评论计划

评论系统先不做。后续可以用 GitHub Discussions + Giscus，把每篇文章映射到一个讨论串。这样不需要自己维护登录、数据库和审核后台，也不会影响当前静态站结构。
