# 吃瓜教程 | Datawhale 10月（3）

## 決定木

### 基本フロー

木構造に基づいて意思決定を行う。

一般に、1 本の決定木は 1 つの根ノード、複数の内部ノード、複数の葉ノードだけを含む。

葉ノードは決定結果に対応し、それ以外の各ノードは属性テストに対応する。根ノードから各葉ノードまでの経路は、一連の判定テスト列に対応する。

決定木学習の目的は、汎化能力が高い、つまり未知の事例をうまく処理できる決定木を生成することである。

論理的な観点では、決定木は大量の `if else` 文の組み合わせである。

幾何学的な観点では、ある基準に従って特徴空間を分割する操作である。

最終目的は、サンプルをできるだけ「純粋」に分けること。

```
# 入力：訓練集合 D = {(x_1,y_1),(x_1,y_1),...,(x_m,y_m)};
#      属性集合 A = {a_1,a_2,...,a_d}.
# 手順：（再帰処理）
'''
TreeGenerate(D,A):
	ノード node を生成する
    if D 中のサンプルがすべて C に属する:
        node を C クラスの葉ノードとしてマークする
        return
    if A = NULL or D が A 上で同じ値を取る:
    	node を葉ノードとしてマークし、カテゴリは D 中で最も多いクラスにする
    	return
    else:
    	A から最適な分割属性 a_* を選ぶ
    	for a_* の各値 a_*_v:
    		node に分岐を一つ生成する
    		Dv を a_* 上で値 a_*_v を取るサンプル部分集合とする
    		if Dv = NULL:
    			分岐ノードを葉ノードとしてマークし、カテゴリは D 中で最も多いクラスにする
    			return
    		else：
    			TreeGenerate(Dv,A/{a_*}) を分岐ノードとする
    return
'''
```

### 分割選択

**自己情報量**:

\\[I(X)=-\\log_{b}{p(x)}
\\]

`b = 2` のとき単位は bit、`b = e` のとき単位は nat である。

下の式と合わせると、次のようにも書ける。

\\[I(D)=-\\log_{b}{p_{k}}
\\]

**情報エントロピー**、つまり自己情報量の期待値は、サンプル集合の純度を測るために最もよく使われる指標である。

現在のサンプル集合 `D` において、第 `k` クラスのサンプル割合を \\(p_{k}(k=1,2,...,|\\mathcal{Y}|)\\) とすると、`D` の情報エントロピーは次のように定義される。

\\[H(D)=E[I(D)]=\\operatorname{Ent}(D)=-\\sum_{k=1}^{|\\mathcal{Y}|} p_{k} \\log _{2} p_{k}
\\]

\\(\\operatorname{Ent}(D)\\) が小さいほど、`D` の純度は高い。

情報エントロピーを計算するとき、\\(p_{k}=0\\) なら \\(p_{k} \\log _{b} p_{k}=0\\) と約束する。\\(X\\) のある値の確率が 1 のとき、情報エントロピーは最小、つまり最も確定的で、サンプル集合の純度は最も高く、値は 0 になる。\\(X\\) の各値の確率が等しいとき、情報エントロピーは最大、つまり最も不確定になり、値は \\(\\log _{b}|D|\\) になる。ここで \\(|D|\\) は `D` が取り得る値の個数を表す。

**条件付きエントロピー**: `Y` の情報エントロピーについて、確率分布 `X` に関する期待値を取ったもの。

`X` が既知のときの `Y` の不確実性を表す。

\\[H(Y|X)=\\sum_{x}p(x)H(Y|X=x)
\\]

単一属性、つまり特徴 `a` の観点から見る。属性 `a` の可能な値を \\(\\{a^{1},a^{2},...,a^{V}\\}\\) とし、\\(D^{v}\\) を属性 `a` の値が \\(a^{v}\\in\\{a^{1},a^{2},...,a^{V}\\}\\) であるサンプル集合とする。\\(\\frac{|D^{v}|}{D}\\) は割合を表す。このとき、属性 `a` の値が既知である場合のサンプル集合 `D` の条件付きエントロピーは次のようになる。

\\[\\sum_{v=1}^{V} \\frac{\\left|D^{v}\\right|}{|D|} \\operatorname{Ent}\\left(D^{v}\\right)
\\]

**情報利得**: 属性、つまり特徴 `a` の値が既知になった後、`y` の不確実性がどれだけ減るかを表す。言い換えると純度の向上量であり、情報エントロピー - 条件付きエントロピーである。

\\[\\operatorname{Gain}(D, a)=\\operatorname{Ent}(D)-\\sum_{v=1}^{V} \\frac{\\left|D^{v}\\right|}{|D|} \\operatorname{Ent}\\left(D^{v}\\right)
\\]

#### ID3 決定木

情報利得を基準に分割属性を選ぶ決定木である。情報利得が最大の属性を分割属性として採用する。

\\[a_{*}=\\underset{a \\in A}{\\arg \\max } \\operatorname{Gain}(D, a)
\\]

#### C4.5 決定木

情報利得基準は、取り得る値の数が多すぎる属性を好みやすい。たとえば識別番号のような属性である。ただしこれは、各値に含まれるサンプルが少なすぎることに起因する。この偏りによる悪影響を減らすため、C4.5 決定木では情報利得の代わりに利得率を使う。

**利得率**

\\[\\operatorname{Gain_{-}\\operatorname{ratio}}(D, a)=\\frac{\\operatorname{Gain}(D, a)}{\\operatorname{IV}(a)}
\\]

ここで、

\\[\\operatorname{IV}(a)=-\\sum_{v=1}^{V} \\frac{\\left|D^{v}\\right|}{|D|} \\log _{2} \\frac{\\left|D^{v}\\right|}{|D|}
\\]

を属性 \\(a\\) の「固有値」と呼ぶ。属性 \\(a\\) の可能な値の個数 \\(V\\) が大きいほど、通常その固有値 \\(\\operatorname{IV}(a)\\) も大きくなる。ただし、利得率は逆に、取り得る値の数が少ない属性を好みやすい。

そのため C4.5 は情報利得を完全に利得率で置き換えるわけではない。まず情報利得が平均水準を上回る属性を選び、その中から利得率が最も高いものを選ぶ。

#### CART 決定木

**Gini 値**

サンプル集合 \\(D\\) から二つのサンプルをランダムに取り出したとき、そのカテゴリラベルが一致しない確率である。したがって Gini 値が小さいほど、異なるクラスに当たる確率が小さく、純度は高くなる。

\\[\\begin{aligned}
\\operatorname{Gini}(D) &=\\sum_{k=1}^{|\\mathcal{Y}|} \\sum_{k^{\\prime} \\neq k} p_{k} p_{k^{\\prime}} \\\\
&=\\sum_{k=1}^{|\\mathcal{Y}|} p_{k}\\left(1-p_{k}\\right) \\\\
&=1-\\sum_{k=1}^{|\\mathcal{Y}|} p_{k}^{2}
\\end{aligned}
\\]

**Gini 指数**

属性 `a` の Gini 指数:

\\[\\operatorname{Gini_{-}\\operatorname{index}}(D, a)=\\sum_{v=1}^{V} \\frac{\\left|D^{v}\\right|}{|D|} \\operatorname{Gini}\\left(D^{v}\\right)
\\]

CART 決定木は、Gini 指数が最小の属性を最適な分割属性として選ぶ。

\\[a_{*}=\\underset{a \\in A}{\\arg \\text { min }} \\operatorname{Gini_{-}\\operatorname{index}}(D, a)
\\]

CART 決定木の実際の構築アルゴリズムは次のとおり。

- まず、各属性 \\(a\\) の各可能値 \\(v\\) について、データセット \\(D\\) を \\(a=v\\) と \\(a \\neq v\\) の二つに分けて Gini 指数を計算する。

\\[\\operatorname{Gini_{-}\\operatorname{index}}(D, a)=\\frac{\\left|D^{a=v}\\right|}{|D|} \\operatorname{Gini}\\left(D^{a=v}\\right)+\\frac{\\left|D^{a \\neq v}\\right|}{|D|} \\operatorname{Gini}\\left(D^{a \\neq v}\\right)
\\]

- 次に、Gini 指数が最小の属性と、その対応値を、最適分割属性および最適分割点として選ぶ。

- 最後に、停止条件を満たすまで上の二ステップを繰り返す。

### 剪定処理

過学習を防ぐため、主に事前剪定と事後剪定がある。

- 事前剪定: 決定木の生成過程で、各ノードを分割する前に評価する。現在ノードの分割が決定木の汎化性能向上につながらない場合、分割を停止し、現在ノードを葉ノードとしてマークする。

- 事後剪定: 先に完全な決定木を生成し、その後で葉ノードを下から上へ確認する。そのノードに対応する部分木を葉ノードに置き換えた方が汎化性能を高められるなら、置き換える。

汎化性能の向上を判断する方法はホールドアウト法である。つまり一部データを検証集合として残し、評価に使う。

事後剪定は過少適合のリスクが小さく、汎化性能も一般に事前剪定よりよい。ただし時間コストは大きい。

### 連続値と欠損値

#### 連続値の処理

ここまで議論しているのは離散属性である。連続属性に出会った場合は、離散化して処理できる。

最も単純なのは二分法である。隣接する二つの値の中点を分割点として使う。

例は [南瓜书](https://datawhalechina.github.io/pumpkin-book/#/chapter4/chapter4?id=_47) を参照。

#### 欠損値の処理

第一のケースは、属性値が欠損している場合である。

訓練集合 \\(D\\) と属性 `a` が与えられているとする。\\(\\tilde{D}\\) を、\\(D\\) のうち属性 `a` に欠損値がないサンプル部分集合とする。

属性 `a` に `V` 個の可能値 \\(\\{a^{1},a^{2},...,a^{V}\\}\\) があると仮定する。\\(\\tilde{D}^{v}\\) は \\(\\tilde{D}\\) のうち、属性 `a` の値が \\(a^{v}\\) であるサンプル部分集合を表す。\\(\\tilde{D}_{k}\\) は \\(\\tilde{D}\\) のうち第 `k` クラスに属するサンプル部分集合を表す。各サンプル `x` に重み \\(w_{x}\\) を付けるとし、次を定義する。

\\[\\rho=\\frac{\\sum_{x \\in \\tilde{D}} w_{x}}{\\sum_{x \\in D} w_{x}}
\\]

\\[\\tilde{p}_{k}=\\frac{\\sum_{x \\in \\tilde{D}_{k}} w_{x}}{\\sum_{x \\in \\widetilde{D}} w_{x}} \\quad(1 \\leq \\mathrm{k} \\leq|y|) \\\\
\\]

\\[\\tilde{r}_{v}=\\frac{\\sum_{x \\in \\tilde{D}}^{v} w_{x}}{\\sum_{x \\in \\widetilde{D}} w_{x}} \\quad(1 \\leq \\mathrm{v} \\leq \\mathrm{V})
\\]

属性 `a` に対して、\\(\\rho\\) は欠損値がないサンプルの割合を表し、\\(\\tilde{p}_{k}\\) は欠損値がないサンプルのうち第 `k` クラスが占める割合を表す。\\(\\tilde{r}_{v}\\) は、欠損値がないサンプルのうち、属性 `a` の値が \\(a^{v}\\) であるサンプルの割合を表す。

したがって、情報利得の計算式を次のように拡張する。

\\[\\begin{array}{l}
\\operatorname{Gain}(D, a)=\\rho \\times \\operatorname{Gain}(\\tilde{D}, a)=\\rho \\times\\left(\\operatorname{Ent}(\\tilde{D})-\\sum_{v=1}^{V} \\tilde{r}_{v} \\operatorname{Ent}\\left(\\tilde{D}^{v}\\right)\\right) \\\\
\\operatorname{Ent}(\\tilde{D})=-\\sum_{k=1}^{|y|} \\tilde{p}_{k} \\log _{2} \\tilde{p}_{k}
\\end{array}
\\]

第二のケースは、分割属性が与えられているが、サンプルがその属性上で欠損値を持つ場合である。

- サンプル \\(\\boldsymbol{x}\\) の分割属性 \\(a\\) 上の値が既知なら、\\(\\boldsymbol{x}\\) をその値に対応する子ノードへ割り当て、サンプル重みは子ノード内でも \\(w_{x}\\) のまま保持する。

- サンプル \\(\\boldsymbol{x}\\) の分割属性 \\(a\\) 上の値が未知なら、\\(\\boldsymbol{x}\\) をすべての子ノードへ同時に割り当てる。ただし、属性値 \\(a^{v}\\) に対応する子ノード内では、サンプル重みを \\(\\tilde{r}_{v} \\cdot w_{x}\\) に調整する。直感的には、同じサンプルを異なる確率で異なる子ノードへ割り当てることに相当する。

### 多変量決定木

決定木が生成する分類境界には明確な特徴がある。軸に平行である、という点だ。つまり分類境界は、座標軸に平行な複数の区間で構成される。しかし、実際の学習タスクの分類境界が複雑な場合、よりよい近似を得るには多数の分割区間が必要になる。

多くの場合、斜めに分割できるとモデルをより簡潔にできる。多変量決定木はそのための考え方である。

多変量決定木では、非葉ノードは特定の一属性だけをテストするのではなく、属性の線形結合をテストする。これにより、学習過程では各非葉ノードについて最適な分割属性を探すのではなく、適切な線形分類器を構築しようとする。
