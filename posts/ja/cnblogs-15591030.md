# Emotional Chatting Machine 論文メモ

## 背景

当時の研究、つまりこの論文が発表される前の研究は、多くが心理学に着想を得たもので、ルールベースであるか、小規模データに限られていた。大規模データで訓練されたモデルは、いずれも感情要因を考慮していなかった。

この研究にはいくつかの課題がある。

- 感情ラベル付きデータベースが不足している。

- 文法性と感情表現のバランスを取る必要がある。

- 既存モデルは感情を単純に埋め込むだけで、多くの場合、感情のない発話、つまりかなり控えめで曖昧な発話を生成する。

本文では以下の改善を行っている。

- 大規模会話生成において感情要因を扱う手法を提案した。著者らの知る限り、このテーマに関する最初の研究である。

- Emotional Chatting Machine、ECM と呼ばれる end-to-end フレームワークを提案し、感情の影響を大規模対話生成に統合した。ECM には、感情カテゴリ埋め込み、内部感情記憶、外部記憶という三つの新しい機構がある。

- 結果から、従来の Seq2Seq モデルと比べて、ECM は内容面と感情面のスコアが高い応答を生成できることが示された。共感的コンピュータエージェントや感情インタラクションモデルなど、今後の研究は ECM を基盤にできると考えられる。

既存研究と比べて、この研究は主に二つの点で異なる。

- 先行研究はテキスト生成において言語ツールやカスタムパラメータに大きく依存していたが、このモデルは完全にデータ駆動であり、手動調整を必要としない。

- 先行研究は入力と応答の間にある複数の感情的相互作用をモデル化できず、生成されるテキストは主に文脈の感情をそのまま引き継ぐだけだった。

## モデル

![](/assets/posts/cnblogs-15591030/image-01.png)

入力 \\(\\boldsymbol{X}=\\left(x_{1}, x_{2}, \\cdots, x_{n}\\right)\\) と、生成する応答の感情カテゴリ \\(e\\) が与えられる。目標は、感情カテゴリ \\(e\\) と一致する応答 \\(\\boldsymbol{Y}=\\left(y_{1}, y_{2}, \\cdots, y_{m}\\right)\\) を生成することである。根本的には、このモデルは次の確率を推定している。

\\[P(\\boldsymbol{Y} \\mid \\boldsymbol{X}, e)=\\prod_{t=1}^{m} P\\left(y_{t} \\mid y_{<t}, \\boldsymbol{X}, e\\right)
\\]

感情カテゴリは `{ Angry, Disgust, Happy, Like, Sad, Other }` で、中国語感情分類チャレンジタスクを採用している。

提案された Emotional Chatting Machine、ECM は、感情表現を生成するために三つの機構を使う。

- 感情カテゴリは感情表現の高レベルな抽象であるため、ECM は感情カテゴリを埋め込み、その埋め込みをデコーダへフィードバックする。

- デコード過程には内部感情状態が存在すると仮定する。この状態の暗黙的な変化を捉え、文法状態と感情状態の重みを動的にバランスさせるため、ECM は内部記憶モジュールを採用している。

- 外部記憶モジュールによって、一般語、つまり非感情語、または感情語を明示的に選択し、感情の明示的表現をモデル化する。

### 感情カテゴリ埋め込み

各感情カテゴリ \\(e\\) について、感情カテゴリ \\(\\boldsymbol{v}_{e}\\) のベクトルをランダム初期化し、その後訓練によって感情カテゴリのベクトルを学習する。感情カテゴリ埋め込み \\(\\boldsymbol{v}_{e}\\)、単語埋め込み \\(\\boldsymbol{e}(y_{t-1})\\)、コンテキストベクトル \\(\\boldsymbol{c}_ {t}\\) はデコーダへ入力され、デコーダ状態 \\(\\boldsymbol{s}_{\\boldsymbol{t}}\\) を更新する。

\\[\\boldsymbol{s}_{\\boldsymbol{t}}=\\mathbf{G} \\mathbf{R} \\mathbf{U}\\left(\\boldsymbol{s}_{t-1},\\left[\\boldsymbol{c}_{t} ; \\boldsymbol{e}\\left(y_{t-1}\\right) ; \\boldsymbol{v}_{e}\\right]\\right)
\\]

\\(\\boldsymbol{s}_{\\boldsymbol{t}}\\) に基づいて、前述の式から対応するデコード確率分布を計算し、次のトークン \\(y_{t}\\) を生成できる。

### 内部記憶

![](/assets/posts/cnblogs-15591030/image-02.png)

内部記憶の役割は、文法性と感情性のバランスを取ることにある。この論文で使われている方法では、各ステップで感情状態がある程度減衰する。デコード過程が完了すると、感情状態は 0 まで減衰しているべきであり、これは感情が完全に表現されたことを示す。

私の理解では、感情状態がずっと変わらないままだと、感情の重みが高すぎて文全体の文法が崩れる可能性がある。たとえば `I'm happy because of the weather.` では、前半では感情状態の重みが高いため `happy` を選んで感情を表現する。その後、感情重みが徐々に下がることで、後半では `because of the weather` のような叙述的な従属節を選べる。もし感情重みが下がらなければ、`because of the happy` のような不自然な形になる可能性がある。

内部記憶モジュールの詳細は図 2 に示されている。各ステップ `t` で、ECM は直前にデコードされた単語の単語埋め込み \\(\\boldsymbol{e}(y_{t}-1)\\)、デコーダの前状態 \\(\\boldsymbol{s}_{t-1}\\)、現在のコンテキストベクトル \\(\\boldsymbol{c}_{t}\\) を入力として読み取りゲート $\\boldsymbol{g}*{t}^{r} $ を計算する。書き込みゲート $\\boldsymbol{g}*^{w} $ はデコーダ状態ベクトル \\(s_{t}\\) 上で計算される。読み取りゲートと書き込みゲートは次のように定義される。

\\[\\begin{aligned}
\\boldsymbol{g}_{t}^{r} &=\\operatorname{sigmoid}\\left(\\mathbf{W}_{\\mathbf{g}}^{\\mathbf{r}}\\left[\\boldsymbol{e}\\left(y_{t-1}\\right) ; \\boldsymbol{s}_{t-1} ; \\boldsymbol{c}_{t}\\right]\\right) \\\\
\\boldsymbol{g}_{t}^{w} &=\\operatorname{sigmoid}\\left(\\mathbf{W}_{\\mathbf{g}}^{\\mathbf{w}} \\boldsymbol{s}_{t}\\right)
\\end{aligned}
\\]

読み取りゲートと書き込みゲートは、それぞれ内部記憶モジュールからの読み取りと書き込みに使われる。したがって、感情状態は各ステップで一定量、つまり $\\boldsymbol{g}_{t}^{w} $ によって消去される。最後のステップでは、内部感情状態は 0 へ減衰する。この過程は次のように形式化される。

\\[\\begin{aligned}
\\boldsymbol{M}_{r, t}^{I} &=\\boldsymbol{g}_{t}^{r} \\otimes \\boldsymbol{M}_{e, t}^{I} \\\\
\\boldsymbol{M}_{e, t+1}^{I} &=\\boldsymbol{g}_{t}^{w} \\otimes \\boldsymbol{M}_{e, t}^{I}
\\end{aligned}
\\]

ここで、\\(\\otimes\\) は [element-wise 乗算](https://www.zhihu.com/question/65866370) であり、\\(r/w\\) はそれぞれ読み取り / 書き込み、\\(I\\) は内部を意味する。GRU は、前のターゲット単語 \\(\\boldsymbol{e}(y_{t}-1)\\)、デコーダ前状態 \\(\\boldsymbol{s}_{t-1}\\)、コンテキストベクトル \\(\\boldsymbol{c}_{t}\\)、感情状態更新 \\(\\boldsymbol{M}_{r, t}^{I}\\) に基づき、状態 \\(\\boldsymbol{s}_{t}\\) を更新する。

\\[\\boldsymbol{s}_{\\boldsymbol{t}}=\\mathbf{G} \\mathbf{R} \\mathbf{U}\\left(\\boldsymbol{s}_{t-1},\\left[\\boldsymbol{c}_{t} ; \\boldsymbol{e}\\left(y_{t-1}\\right) ; \\boldsymbol{M}_{r, t}^{I}\\right]\\right)
\\]

この状態に基づいて単語生成分布 \\(\\boldsymbol{o}_{t}\\) を得て、次の単語 \\(y_{t}\\) をサンプリングできる。次の単語を生成した後、\\(\\boldsymbol{M}_{r, t+1}^{I}\\) が内部記憶へ書き戻される。式 9 を複数回実行することは、行列を連続して乗算することに相当し、減衰効果を生む。これは \\(1≤ sigmoid(·) ≤ 1\\) だからである。記憶ネットワークにおける削除操作に似ている。

簡単に言えば、内部記憶が関与する部分では、まず読み取りゲートのパラメータを計算し、内部記憶と element-wise に乗算して GRU へ渡す。その後、時刻 `t` のデコーダ状態を計算し、書き込みゲートを通じて内部記憶へ書き込む。このとき書き込みゲートの値は 0 から 1 の数値なので、減衰が起きる。

### 外部記憶

一般語を選ぶか感情語を選ぶかを、タイプセレクタで決める機構である。

![](/assets/posts/cnblogs-15591030/image-03.png)

デコーダの現在状態 \\(s_{t}\\) が与えられると、外部記憶モジュールから読み出す感情語彙表と一般語彙について、それぞれ感情 softmax \\(P_{e}(y_{t}＝w_{e})\\) と一般 softmax \\(P_{g}(y_{t}＝w_{g})\\) を計算する。タイプセレクタ \\(α_{t}\\) は、感情語または一般語 \\(y_{t}\\) を生成する重みを制御する。最後に、次単語確率、つまり二つの重み付き確率の [concatenation](https://blog.csdn.net/Frank_LJiang/article/details/104333272)、から次の単語をサンプリングする。この過程は次のように定式化される。

\\[\\begin{aligned}
\\alpha_{t} &=\\operatorname{sigmoid}\\left(\\mathbf{v}_{\\mathbf{u}}{ }^{\\top} s_{t}\\right), \\\\
P_{g}\\left(y_{t}=w_{g}\\right) &=\\operatorname{softmax}\\left(\\mathbf{W}_{\\mathbf{g}}^{\\mathrm{o}} s_{t}\\right), \\\\
P_{e}\\left(y_{t}=w_{e}\\right) &=\\operatorname{softmax}\\left(\\mathbf{W}_{\\mathrm{e}}^{\\mathrm{o}} s_{t}\\right), \\\\
y_{t} \\sim \\boldsymbol{o}_{t}=P\\left(y_{t}\\right) &=\\left[\\begin{array}{r}
\\left(1-\\alpha_{t}\\right) P_{g}\\left(y_{t}=w_{g}\\right) \\\\
\\alpha_{t} P_{e}\\left(y_{t}=w_{e}\\right)
\\end{array}\\right],
\\end{aligned}
\\]

ここで、\\(\\alpha_{t}\\in[0,1]\\) はスカラーであり、感情語 \\(w_{e}\\) と一般語 \\(w_{g}\\) の選択をバランスさせる。\\(P_{g}/P_{e}\\) はそれぞれ一般語 / 感情語の分布であり、\\(P_{(y_{t})}\\) は最終的な単語デコード分布である。なお、この二つの語彙表に交差はなく、最終分布 \\(P_{(y_{t})}\\) は二つの分布の concatenation である。

## 実験

### 実装詳細

提案モデルの実装には TensorFlow を使用した。

エンコーダとデコーダはいずれも二層 GRU 構造で、各層に 256 個の隠れユニットを持ち、それぞれ異なるパラメータ集合を使う。単語埋め込みサイズは 100 に設定した。

語彙数は 40000 に制限した。感情カテゴリの埋め込みサイズは 100 に設定した。

内部記憶はサイズ 6×256 の訓練可能な行列であり、外部記憶は 40000 語のリストである。このリストには一般語と感情語が含まれる。ただし感情語には異なるマークが付いている。

多様な応答を生成するため、デコード過程ではビームサーチを採用し、ビームサイズは 20 に設定した。その後、未知語を含む応答を除去し、生成確率に基づいて応答を並べ替えた。
小バッチ確率的勾配降下、SGD、を使用した。バッチサイズと学習率は、それぞれ 128 と 0.5 に設定した。

訓練を速めるため、STC データセット上で事前訓練済み単語埋め込みを持つ seq2seq モデルを訓練した。その後、ESTC データセット上でこのモデルを訓練し、パラメータは事前訓練済み seq2seq モデルのパラメータで初期化した。

20 epoch 実行し、各モデルの訓練段階は Titan X GPU マシン上で約 1 週間かかった。

### 比較モデル

一般的な seq2seq モデルと、著者らが作成した感情カテゴリ埋め込みモデル、Emb、を比較する。Emb では感情カテゴリをベクトルへ埋め込み、そのベクトルを各デコード位置の入力として使う。これはユーザー埋め込みの考え方に似ている。感情カテゴリは感情表現の高レベルな抽象なので、これはこのモデルに対する適切なベースラインである。

### 結果

**自動評価**

期待される感情カテゴリ、つまりモデルへの入力、と、感情分類器が生成応答から予測した感情カテゴリとの一致性を検出するため、感情正確率を採用した。

![](/assets/posts/cnblogs-15591030/image-04.png)

ECM は感情正確率で最も良い性能を示した。また、[perplexity](https://zhuanlan.zhihu.com/p/44107044) では Seq2Seq より良いが、Emb よりは悪い。

**人手評価**

アノテータには、内容、評価範囲 0、1、2、と感情、評価範囲 0、1、に基づいて応答を採点し、二つのシステムのどちらを好むかを示すよう求めた。

![](/assets/posts/cnblogs-15591030/image-05.png)

ECM の全応答スコアは、二つの指標において他手法より有意に優れている。ECM が生成した応答のうち、内容スコアが 2、感情スコアが 1 のものは 27.2% であり、Emb と Seq2Seq はそれぞれ 22.8% と 9.0% だった。これは、ECM が内容と感情の両面で、より高品質な応答を生成できることを示している。

![](/assets/posts/cnblogs-15591030/image-06.png)

すべての感情カテゴリで、ECM は感情面において他手法より優れている。しかし Disgust と Angry カテゴリでは、ECM の内容面の性能はベースラインモデルに及ばない。この二カテゴリには十分な訓練データがないためである。たとえば ESTC データセットでは、Angry カテゴリの応答は 234635 件しかなく、他カテゴリよりかなり少ない。

![](/assets/posts/cnblogs-15591030/image-07.png)

感情モデル、つまり Emb と ECM、は Seq2Seq より好まれている。明確な感情表現と内容の適切性により、ECM はさらに好まれている。

## 結論と今後の課題

本文では、大規模対話生成における感情の影響をモデル化するために Emotional Chatting Machine、ECM、を提案した。感情カテゴリ埋め込み、内部感情記憶、外部記憶という三つの感情要因モデルを導入している。客観評価と人手評価の結果から、ECM は内容面で適切な応答を生成できるだけでなく、感情面でも適切な応答を生成できることが示された。

今後の研究では、感情と ECM の相互作用を探索する。つまり、感情カテゴリを指定するのではなく、モデル自身が応答に最も適した感情カテゴリを決定するべきである。ただし、そのようなタスクはトピック、文脈、あるいはユーザーの感情に依存するため、難しい可能性がある。
