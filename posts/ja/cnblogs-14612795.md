### embedding

単語埋め込みをざっくり言うと、文字や単語を一連の数字に変換することである。数字はコンピュータがより認識しやすい表現形式だからである。

単語埋め込みの過程は、コンピュータ用の辞書を作っているようなものだ。コンピュータはこの辞書を通じて、間接的に文字を識別できる。

単語埋め込みベクトルは、ニューラルネットワークにおける単語のベクトル表現と理解してもよい。

詳しくは [嵌入(embedding)层的理解](https://www.cnblogs.com/USTC-ZCC/p/11068791.html) を参照。

### PyTorch における embedding

入力はインデックスのリストで、出力は対応する単語埋め込みである。

```
torch.nn.Embedding(num_embeddings, embedding_dim, padding_idx=None,
                   max_norm=None,  norm_type=2.0,   scale_grad_by_freq=False, 
                   sparse=False,  _weight=None)
```

#### パラメータ

- num_embeddings（int）: 辞書のサイズ。たとえば全部で 5000 語が出現するなら 5000 を入力する。このとき index は 0 から 4999 になる。

- embedding_dim（int）: 埋め込みベクトルの次元。つまり 1 つの記号を何次元で表すか。

- padding_idx（int, optional）: padding 用の ID。たとえば入力長が 100 でも、各文の長さは毎回同じとは限らない。そのため後ろを統一した数字で埋める必要がある。ここではその数字を指定する。ネットワークは padding ID に遭遇したとき、それと他の記号との関連性を計算しない。（初期値は 0）

- max_norm（float, optional）: 最大ノルム。埋め込みベクトルのノルムがこの上限を超えた場合、再正規化する。

- norm_type（float, optional）: どのノルムを使って計算し、`max_norm` と比較するかを指定する。デフォルトは 2 ノルム。

- scale_grad_by_freq（boolean, optional）: mini-batch 内で単語が出現する頻度に応じて勾配をスケーリングする。デフォルトは False。

- sparse（bool, optional）: True の場合、重み行列に関連する勾配を疎テンソルに変換する。デフォルトは False。

#### 使用例

```
import torch
import numpy as np

#建立词向量层，词数为13，嵌入向量维数设为3
embed = torch.nn.Embedding(13,3)
#句子对['I am a boy.','How are you?','I am very lucky.']
#batch = [['i','am','a','boy','.'],['i','am','very','lucky','.'],['how','are','you','?']]

#将batch中的单词词典化，用index表示每个词（先按照这几个此创建词典）
#batch = [[2,3,4,5,6],[2,3,7,8,6],[9,10,11,12]]

#每个句子实际长度
#lens = [5,5,4]

#加上EOS标志且index=0
#batch = [[2,3,4,5,6,0],[2,3,7,8,6,0],[9,10,11,12,0]]

#每个句子实际长度（末端加上EOS）
lens = [6,6,5]

#PAD过后，PAD标识的index=1
batch = [[2,3,4,5,6,0],[2,3,7,8,6,0],[9,10,11,12,0,1]]

#RNN的每一步要输入每个样例的一个单词，一次输入batch_size个样例
#所以batch要按list外层是时间步数(即序列长度)，list内层是batch_size排列。
#即[seq_len,batch_size]
batch = np.transpose(batch)

batch=torch.LongTensor(batch)

embed_batch = embed(batch)

print(embed_batch)
```
