## 第三章 Java 言語基礎

### 3.4 演算子

#### 代入演算子

`=`。1 つの式の中に 2 つ以上の `=` がある場合、最も右側の `=` から処理される。

#### 算術演算子

- 加算 `+`

- 減算 `-`

- 乗算 `*`

- 除算 `/`（0 は除数にできない）

- 剰余 `%`

#### インクリメントとデクリメント

- `++a`: 先に +1 してから `a` を使う。`--a`: 先に -1 してから使う。

- `a++`: 先に `a` を使ってから +1 する。`a--`: 先に使ってから -1 する。

#### 比較演算子

結果は boolean 型になる。比較的簡単で他の言語とも共通しているので省略。

#### 論理演算子

結果は boolean 型になる。これも比較的簡単なので省略。

- `&` (`&&`) AND

- `||` OR

- `!` NOT

##### `&` と `&&` の違い

- 短絡: 論理式の左側の式から式全体の値を推定できることを短絡という。言い換えると、すべての部分の真偽を判断しなくても式全体の最終結果が分かる場合を短絡と呼ぶ。

- `&`: 非短絡演算子。途中まで判断すれば最終結果が分かる場合でも、すべての部分を判断する。

- `&&`: 短絡演算子。途中まで判断すれば最終結果が分かる場合、その時点で判断を終了する。

例:

`(false || false) & (!false)` は前後両方を判断する。

`(false || false) && (!false)` は前半だけを判断する。

#### ビット演算子

ビット演算は完全にビットに対する操作である。

整数データはメモリ内で 2 進数形式で表される。たとえば 7 は `00000000 00000000 00000000 00000111`。

左端の最上位ビットは符号ビットで、1 は負、0 は正を表す。

負数は補数で表される。たとえば -8 は `11111111 11111111 11111111 11111000`。

原码・反码・补码について: [https://www.cnblogs.com/zhangziqiu/archive/2011/03/30/ComputerCode.html](https://www.cnblogs.com/zhangziqiu/archive/2011/03/30/ComputerCode.html)

- ビット AND `&`: 各ビットで論理 AND を行う。

- ビット OR `|`: 各ビットで論理 OR を行う。

- ビット反転 `~`: 各ビットで論理 NOT を行う。

- ビット XOR `^`: 同じなら 0、異なれば 1。

- シフト操作: `byte`、`short`、`char`、`int`、`long` にのみ適用できる。

`<<` 左シフト。全体を左へ移動し、右側を 0 で埋める。

- `>>` 右シフト。最上位ビットが 0 なら右シフト後に左側を 0 で埋め、最上位ビットが 1 なら左側を 1 で埋める。

- `>>>` 符号なし右シフト。最上位ビットが 0 でも 1 でも、右シフト後は 0 で埋める。

- シフトで `*2` や `/2` の効果を実現できる。たとえば `y<<1` は `y*2`、`y>>1` は `y/2`。

`y>>n` は `y/2` の n 乗に相当する。

#### 三項演算子

`条件式?値1:値2;`

条件式が真なら値1、偽なら値2を取る。

例: `boolean b = 20<45?true:false;` の場合、`b=true`。

#### 演算子の優先順位

括弧 > 正負号 > インクリメント・デクリメントと `!` > 算術演算 > シフト演算 > 比較演算 > ビット演算 > 論理演算 > 代入演算

```
Java3_4 {

    public static void main(String[] args){
        int a, b, c;
        a = 15;
        /*
         可以但不建议 最好还是写成
         c = a + 4;
         b = a + 4;
         或者b是跟c有逻辑相连关系写成
         b = c;
         */
        c = b = a + 4;
        System.out.println("c值为：" + c);
        System.out.println("b值为：" + b);

        float number1 = 45.56f;
        int number2 = 152;
        System.out.println("和为：" + (number1 + number2));
        System.out.println("差为：" + (number1 - number2));
        System.out.println("积为：" + (number1 * number2));
        System.out.println("商为：" + (number1 / number2));

        int m, n = 1, o = 1;
        // 此时n++是先使用n原本的值再+1
        m = n++;
        // 故此时赋于m的值还是+1之前的1
        System.out.println(m);
        // 此时n已经+1 故输出为已经+1之后的2
        System.out.println(n);
        // ++o是先+1再使用o
        m = ++o;
        // 故此时m为已经+1之后的2
        System.out.println(m);

        boolean flag = 20<45?true:false;
        System.out.println(flag);
    }
}
```

### 3.5 データ型変換

- 低精度 → 高精度: オーバーフローせず、常に成功する。

- 高精度 → 低精度: 情報が失われるため、失敗する可能性がある。

#### 暗黙的型変換

プログラムが自動で実行する。ただし低精度から高精度への変換だけ。

`byte < short < int < long < float < double`

#### 明示的型変換

高精度から低精度へ強制変換できる。ただし情報が失われる可能性がある。

形式: `(対象型名)値`

整数値を `byte`、`short`、`int`、`long` に代入するとき、範囲を超えてはいけない。超える場合は強制変換が必要。

```
public class Java3_5 {
    public static void main(String[] args){
        System.out.println("隐式类型转换");
        int x = 50;
        float y = x;
        System.out.println(y);
        double z = 89.897;
        System.out.println(x + z);

        System.out.println("显式类型转换");
        int a = (int)45.678;
        System.out.println(a);
        byte b = (byte)129;
        System.out.println(b);
    }
}
```
