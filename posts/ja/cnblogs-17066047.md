# py2neo で Neo4j を操作する

## インストール

### Java 17

#### ダウンロード

Java をインストールしていない場合は、先に Java をインストールする必要がある。具体的なバージョンは公式マニュアルを参照する。

[https://neo4j.com/docs/operations-manual/current/installation/windows/](https://neo4j.com/docs/operations-manual/current/installation/windows/)

ダウンロード先:

[https://www.oracle.com/java/technologies/downloads/#java17](https://www.oracle.com/java/technologies/downloads/#java17)

#### インストール

ダウンロード後、実行ファイルをクリックして手順に従ってインストールする。

インストール後、環境変数を設定する。

Windows の場合:

- ユーザー変数: `JAVA_HOME` 変数を新規作成し、値をインストールディレクトリにする。

例: `D:\Java\jdk-17`

- 環境変数: `Path` に `%JAVA_HOME%\bin` を追加する。

#### テスト

`win+R` で cmd を開き、`java -version` と入力する。Java バージョンが 17 と表示されればよい。

### Neo4j

Neo4j Community Edition:

[https://neo4j.com/download-center/#community](https://neo4j.com/download-center/#community)

ダウンロード後の案内に従って進める。

その後、`win+R` で cmd を開き、`インストールディレクトリ\bin\neo4j console` を実行する。

例: `D:\neo4j\bin\neo4j console`

`http://localhost:7474/` にアクセスすれば操作ページが見える。

## Python から操作する

### ドライバ

#### neo4j-driver

元の名前は `neo4j-driver` だったが、現在は `neo4j` に変わっている。

```
pip install neo4j
```

#### py2neo

ここでは主にこれを使う。

```
pip install py2neo
```

### 基本例

```
from py2neo import Node, Relationship, Graph

USERNAME = username
PASSWORD = password

# 新建图形
graph = Graph('http://localhost:7474', auth=(USERNAME, PASSWORD), name='neo4j')

# 新建node
# Node(labels, *properties)
a = Node('label', name='a')
b = Node('label', name='b')

# 在graph中绘制node
graph.create(a)
graph.create(b)

# 新建关系并绘制
# 可以增加一些属性，也可以省略
# Relationship(start_node, type, end_node, **properties)
r1 = Relationship(a, 'to', b, name='to')
graph.create(r1)
```

### データベース操作（主にノードとリレーション）

すべての操作の前に以下を用意する。

```
USERNAME = your username
PASSWORD = your password
NAME = your database name
graph = Graph('http://localhost:7474', auth=(USERNAME, PASSWORD), name=NAME)
```

#### 追加

- ノードを追加する

```
node = Node("Person", name="Alice", id=node_1)
graph.create(node)
```

第 1 引数はノードの label、第 2 引数はノード名で、Neo4j の図上に表示される。第 3 引数はノード ID で、自分で定義した主キーとして検索しやすくするためのもの。

どれも必須ではない。後ろ 2 つはノードの Properties である。

- label を追加する

```
node.add_label("Women")
```

- Properties を追加する

```
node["age"] = 25
```

`new_name` プロパティの値を `value` に設定する。

- リレーションを追加する

```
node_1 = Node("Person", name="Alice", id="node_1")
node_2 = Node("Person", name="Bob", id="node_2")

# 1.第一种方法
relationship = Relationship(node_1, "KNOWS", node_2)
graph.create(relationship)

# 2.第二种方法
class Knows(Relationship): pass
ab = Knows(node_1, node_2)
graph.create(ab)

# 3.第三种方法
KNOWS = Relationship.type("KNOWS")
ab = KNOWS(node_1, node_2)
graph.create(ab)
```

- サブグラフを追加する

```
node_1 = node_matcher.match("Person").where(id='node_1').first()
node_2 = node_matcher.match("Person").where(id='node_2').first()
node_3 = node_matcher.match("Person").where(id='node_3').first()
node_4 = node_matcher.match("Person").where(id='node_4').first()

KNOWS = Relationship.type("KNOWS")
ab = KNOWS(node_1, node_2)
ac = KNOWS(node_1, node_3)
s = ab | ac
graph.create(s)
```

#### 検索

検索前に以下を用意する。

```
from py2neo import NodeMatcher, RelationshipMatcher
node_matcher = NodeMatcher(graph)
relation_matcher = RelationshipMatcher(graph)
```

- ノードを検索する

```
node = node_matcher.match("Person").where(name="Alice").first()
```

`match()` の引数は label で、必須ではない。`where()` の引数は Properties で、これも必須ではない。

`match` の結果は list なので、`.first()` を追加する必要がある。

- リレーションを検索する

グラフ内のすべてのリレーションを検索する。

```
relation = list(relation_matcher.match((node, None), r_type='idea'))
```

ノードまたはリレーション名で検索したい場合、その前提として対象ノードがすでに検索済みであるか、リレーションが何か分かっている必要がある。

```
# 只知道起始节点
relation = list(relation_matcher.match((start_node, None)))

# 只知道结束节点
relation = list(relation_matcher.match((None, end_node)))

# 只知道关系type
relation = list(relation_matcher.match(r_type='Knows'))

# 其余的可用以上三种结合查询所有关系的格式组合得到

# 获得的relation可以通过遍历relation数组获得start_node和end_node以及其属性
relation[0].start_node['id']
```

- label を検索する

```
# 查询node是否含有label
node.has_label(label)

# 查询node的label
node.labels
```

- Properties を検索する

```
# 查询名为name的特性值
# 1.第一种方法
node["name"]
# 2.第二种方法
node_1.get("name")

# 返回特性的个数
len(node)

# 返回特性的字典
dict(node)
# 将节点的特性和值以列表二元组形式返回
node.items()
# 将节点的特性名字返回
node.key()
# 将节点的特性值返回
node.values()
```

#### 変更

変更の前提は、対象がすでに検索済みであること。

- ノードを変更する

```
# 修改节点的特性
node['name'] = 'May'

# setdefault():如果有该特性，则返回其值，若没有则设置为'May'
node.setdefault('name', default = 'May')　

# 修改到这里只是对本地数据进行了修改
# 若想在Neo4j中生效一定要添加这句
graph.push(node)
```

- リレーションを変更する

```
# 修改节点的特性
relation['name'] = 'has'

# setdefault():如果有该特性，则返回其值，若没有则设置为'May'
relation.setdefault('name', default = 'has')　

# 修改到这里只是对本地数据进行了修改
# 若想在Neo4j中生效一定要添加这句
graph.push(relation)
```

#### 削除

ノードを削除するには、先にそのノード上のリレーションを削除する必要がある。

- リレーションを削除する

```
graph.separate(relationship)
```

- ノードを削除する

```
graph.delete(node)
```

- データベースを空にする

```
graph.delete_all()
```

- label を削除する

```
# 删除一个label，如果存在的话
node.remove_label('Person')

#删除所有label
node.clear_label()
```

- Properties を削除する

```
# 删除一个特性，如果不存在就会有KeyError
del node["name"]

# 删除所有特性
node.clear()
```
