# JavaScript メモ

## JS 入門

### ブラウザが JS を実行する仕組み

ブラウザは大きく次の二つに分けられる。

- レンダリングエンジン: HTML と CSS を解析するためのもの。俗にブラウザのカーネルと呼ばれる。たとえば Chrome の Blink。

- JS エンジン: JS インタプリタとも呼ばれる。JS コードを読み取り、バックグラウンドで処理する。たとえば Chrome の V8。

### JS の構成

- ECMAScript: JavaScript の構文。JS のプログラミング構文と基礎的なコア知識を規定する。すべてのブラウザベンダーが共通して従う JS 構文仕様である。

- DOM: ページ文書オブジェクトモデル。DOM が提供するインターフェースを通じて、ページ上の各種要素を操作できる。サイズ、位置、色など。

- BOM: ブラウザオブジェクトモデル。BOM を通じてブラウザウィンドウを操作できる。

変数宣言の特殊ケース:

- 宣言だけして代入しない

表示される値は `undefined` になる。

- 宣言も代入もしない

- 宣言せず代入だけする

実行でき、エラーも出ない。正しく動くが、推奨されない。

## データ型

JS のデータ型は、プログラム実行中に、等号右側の値によって決まる。

1 つの変数のデータ型は変わり得る。

```
var x = 12;
x = 'a';
```

### 分類

- プリミティブ型: Number、String、Boolean、Undefined、Null。String の既定値は `""`、Boolean の既定値は `false`。

- 複合データ型: object。

#### 数値型

整数型と浮動小数点型を含む。既定値は 0。

8 進数: 数値の前に `0` を付けると 8 進数を表す。

```
var num01 = 010;
console.log(num01);
//8
```

16 進数: 数値の前に `0x` を付けると 16 進数を表す。

```
var num02 = 0x11;
console.log(2);
//17
```

最大値と最小値:

```
console.log(Number.MAX_VALUE);
console.log(Number.MIN_VALUE);
//1.7976931348623157e+308
//5e-324
```

三つの特殊値:

```
// 三つの特殊値
// 正の無限大
alert(Infinity);
console.log(Number.MAX_VALUE * 2);
// 負の無限大
alert(-Infinity);
console.log(-Number.MAX_VALUE * 2);
// 数値ではない値
alert(NaN);
console.log('num' - 100);
```

`isNaN`:

```
// isNaN は、数値なら false、数値でなければ true を返す
console.log(isNaN(12));
console.log(isNaN('num'));
// 注: console.log(isNaN('12')); の場合も false を返す
```

### データ型変換

- 文字列型へ変換

- 数値型へ変換

- Boolean 型へ変換

#### 文字列型へ変換

| 方式 | 説明 | 例 |
| --- | --- | --- |
| toString() | 文字列へ変換 | var num=1; alert(num.toString()); |
| String() 強制変換 | 文字列へ変換 | var num=1; alert(String(num)); |
| プラス記号による文字列連結、暗黙変換 | 文字列と連結した結果はすべて文字列になる | var num=1; alert(num+''); |

#### 数値型へ変換

| 方式 | 説明 | 例 |
| --- | --- | --- |
| parseInt(string) 関数 | `string` を整数数値型に変換 | parseInt('78') |
| parseFloat(string) 関数 | `string` を浮動小数点数値型に変換 | parseFloat('78.21') |
| Number() 強制変換 | `string` を数値型に変換 | Number('12') |
| JS の暗黙変換（- * /） | 算術演算を利用して暗黙的に数値型へ変換 | '12'-0 |

- 前二つの変換は、文字列の先頭から形式に合う数値部分だけを数値型に変換する。

例: `parseInt(12.22.1)` は 12、`parseFloat(12.22.1)` は 12.22。

`parseInt(12.2aaaaa2.1)` は 12、`parseFloat(12.2aaaaa2.1)` は 12.2。

一方で `Number()` は、文字列全体が形式に合う数値でなければならない。

例: `Number('12.22.1')` は `NaN`。

## JS 演算子

### 算術演算子

`%` は剰余、つまり余りを求める。

```
console.log(5 % 3);
//2
```

浮動小数点数の算術演算には問題が出ることがある。精度問題があるため、浮動小数点数をそのまま直接演算するのは避ける。

演算後の浮動小数点数と別の浮動小数点数を、直接等しいかどうか比較しない。

```
var num = 0.2 + 0.1;
console.log(num == 0.3);
//false
```

### インクリメント・デクリメント演算子

- 前置インクリメント演算子:

`++num`。1 加算し、先に加算してから値を返す。

- 後置インクリメント演算子:

`num++`。1 加算し、先に値を返してから加算する。

```
// 前置インクリメント演算子。先に加算してから値を返す
var p = 10;
console.log(++p + 10);//21
// 後置インクリメント演算子。先に値を返してから加算する
p = 10;
console.log(p++ + 10);//20
//1.p++ = 11 だが返す値は 10
//2.++p = 12
p = 10;
console.log(p++ + ++p);//22
```

### 比較演算子

```
// == は文字列型を数値型へ変換できる
console.log(18 == '18');//true
console.log(18 === '18');//false
```

### 論理演算子

| 論理演算子 | 説明 | 例 |
| --- | --- | --- |
| && | 論理 AND | true&&false |
| \|\| | 論理 OR | true\|\|false |
| ! | 論理 NOT | !true |

#### 論理短絡、ショートサーキット

- `&&` の短絡

`式1 && 式2` で、式1 が真なら式2を返し、式1 が偽なら式1を返す。

```
console.log(123 && 456);//456
console.log(0 && 456);//0
console.log(0 && 456 && 78945);//0
```

- `||` の短絡

`式1 || 式2` で、式1 が真なら式1を返し、式1 が偽なら式2を返す。

```
console.log(123 || 456);//123
console.log(0 || 456);//456
console.log(0 || 456 || 2367);//456
console.log(0 || 0 || 2367);//2367
```

### 代入演算子

| 代入演算子 | 説明 | 例 |
| --- | --- | --- |
| = | 直接代入 | var usrName = 'Jack'; |
| +=、-= | 加算または減算してから代入 | var age = 10; age += 5;//15 |
| *=、/=、%= | 乗算、除算、剰余を取ってから代入 | var age = 10; age *= 5;//50 |

### 演算子の優先順位

| 優先順位 | 演算子 | 順序 |
| --- | --- | --- |
| 1 | 丸括弧 | () |
| 2 | 単項演算子 | ++ -- ! |
| 3 | 算術演算子 | **先に */%、後に +-** |
| 4 | 関係演算子 | > >= < <= |
| 5 | 等価演算子 | == != !== === |
| 6 | 論理演算子 | **先に &&、後に \|\|** |
| 7 | 代入演算子 | = |
| 8 | カンマ演算子 | , |

## ループ

例 1: 合計を求める

```
    }

// 1 から 100 までの 3 で割り切れるすべての数の和を求める

    }

```

二層に分けて考える。
外側は、1 回の走査比較を 1 ラウンドとし、合計で配列長 - 1 ラウンド比較する。隣り合う二つを比較するため、最後の比較では二つの数の位置を一気に確定できると考えられる。

内側は左右を相互に比較し、左が右より大きければ位置を交換する。外側が 1 ラウンド進むごとに、内側の比較回数は 1 つ減る。外側の各ラウンドで 1 つの数の位置が確定するためである。

```
var arr = [3, 2, 5, 1, 4];
var tmp = 0;
for (var i = arr.length - 1; i > 0; i--) {
    for (var j = 0; j < i; j++) {
        if (arr[j] > arr[j + 1]) {
            tmp = arr[j];
            arr[j] = arr[j + 1];
            arr[j + 1] = tmp;
        }
    }
}
console.log(arr);
```

## 関数

### 関数宣言

```
function 関数名(引数1,引数2,...){
    関数本体
}
```

例 1:

```
function getSum(num1, num2) {
    console.log(num1 + num2);
}
function getTotalSum(num1, num2) {
    var min = num1, max = num2, tmp = 0, sum = 0;
    if (num1 > num2) {
        tmp = min;
        min = max;
        max = tmp;
    }
    for (var i = min; i <= max; i++) {
        sum += i;
    }
    // return sum;
    console.log('sum:' + sum);
}
getSum(2, 4);//6
getTotalSum(12, 5);//68
// 引数の個数が合っていなくてもエラーにはならず、結果が正しく表示されないだけである
// 仮引数より実引数が多い場合、仮引数の個数分だけ実引数を取る
// 仮引数より実引数が少ない場合、不足した引数は自動的に undefined と定義される
```

### return

- 戻り値。返せる値は 1 つだけで、複数の値を返す場合は最後の値だけが採用される。

- `return` 以降のコードは実行されない。

- `return` がない場合、関数は `undefined` を返す。
