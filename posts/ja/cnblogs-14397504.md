先週、教授がついに私のサボり具合を見かねたので、対話システムのコードを探して動かしてみることにした。ところが、自分の PC があまりに弱くて GPU が使えず、CPU で訓練すると 30 時間かかることが分かった。昨日教授に相談したところ Colaboratory を紹介してもらった。初めて使ったので、利用手順を記録しておく。あとで自分が見返すのにも便利だと思う。

## Deep QA

GitHub アドレス: [Deep QA](https://github.com/Conchylicultor/DeepQA)

This work tries to reproduce the results of [A Neural Conversational Model](http://arxiv.org/abs/1506.05869) (aka the Google chatbot). It uses a RNN (seq2seq model) for sentence predictions. It is done using python and TensorFlow.

## Colaboratory を設定する

参考リンク: [Google AI工具Colaboratory的使用方法](https://www.jianshu.com/p/e6f1058614c0?isappinstalled=0)

注: 中国国内で Colaboratory を使うには、何らかの手段が必要。

Google Drive にログインし、「新規」をクリックする。

![](/assets/posts/cnblogs-14397504/image-01.png)

「その他」-「アプリを追加」をクリックする（ここではすでに連携済みなので表示されている）。

![](/assets/posts/cnblogs-14397504/image-02.png)

その後表示されるページで Colaboratory を検索し、連携する。

連携が完了すると上図のようになる。Google Colaboratory をクリックすると、下図のページが表示される。

![](/assets/posts/cnblogs-14397504/image-03.png)

「編集」-「ノートブックの設定」をクリックし、"GPU" を選択して「保存」をクリックする。

![](/assets/posts/cnblogs-14397504/image-04.png)

![](/assets/posts/cnblogs-14397504/image-05.png)

右側で「接続」の横の三角をクリックし、「ホストされているランタイムに接続」を選ぶ。

![](/assets/posts/cnblogs-14397504/image-06.png)

## フォルダを Google Drive にアップロードする

「新規」-「フォルダをアップロード」をクリックすればよい。

![](/assets/posts/cnblogs-14397504/image-07.png)

## Google Drive に接続する

ノートブック内のコードセルに以下を入力する。

```
!apt-get install -y -qq software-properties-common python-software-properties module-init-tools
!add-apt-repository -y ppa:alessandro-strada/ppa 2>&1 > /dev/null
!apt-get update -qq 2>&1 > /dev/null
!apt-get -y install -qq google-drive-ocamlfuse fuse
from google.colab import auth
auth.authenticate_user()
from oauth2client.client import GoogleCredentials
creds = GoogleCredentials.get_application_default()
import getpass
!google-drive-ocamlfuse -headless -id={creds.client_id} -secret={creds.client_secret} < /dev/null 2>&1 | grep URL
vcode = getpass.getpass()
!echo {vcode} | google-drive-ocamlfuse -headless -id={creds.client_id} -secret={creds.client_secret}
```

実行後、結果に表示されたリンクをクリックして認証と検証を行う。実行結果:

![](/assets/posts/cnblogs-14397504/image-08.png)

Google Drive をマウントする。新しいコードセルを作り、以下を入力する。

```
!mkdir -p drive
!google-drive-ocamlfuse drive
```

![](/assets/posts/cnblogs-14397504/image-09.png)

クラウド上のファイルを確認したい場合は、次を入力する。

```
! ls drive/
```

![](/assets/posts/cnblogs-14397504/image-10.png)

## コードを実行する

まず、現在 Colaboratory のデフォルトは TensorFlow 2 なので、そのまま実行するとエラーになる。先に 1.x 系に変更する必要がある。

新しいコードセルを作り、以下を実行する。

```
%tensorflow_version 1.10.0
```

![](/assets/posts/cnblogs-14397504/image-11.png)

また、このコードはデータファイルをデフォルトでルートディレクトリから探す。そのため Colaboratory で実行するとファイルが見つからない。コードを少し修正する必要がある。

Google Drive 上で `chatbot/chatbot.py` を直接開き、Text Editor で開く。152 行目を以下に変更する。

```
self.args.rootDir = os.getcwd() + '/drive/DeepQA-master'  #for colab run
            #self.args.rootDir = os.getcwd()
```

`punkt` データを事前にダウンロードしておく。以下を入力する。

```
import nltk
nltk.download('punkt')
```

![](/assets/posts/cnblogs-14397504/image-12.png)

コードを実行してモデルを訓練する（およそ 2 時間必要）。

```
! python3 drive/DeepQA-master/main.py
```

![](/assets/posts/cnblogs-14397504/image-13.png)

テストする。

```
! python3 drive/DeepQA-master/main.py --test
```

![](/assets/posts/cnblogs-14397504/image-14.png)

結果は `save / model / samples_predictions.txt` に表示される。

![](/assets/posts/cnblogs-14397504/image-15.png)

自分で対話したい場合:

```
! python3 drive/DeepQA-master/main.py --test interactive
```

## 補足（bug）

### E: Package 'python-software-properties' has no installation candidate

ある実行時に、Google Drive へ接続するコードをうっかりもう一度実行してしまい、以下が出た。

![](/assets/posts/cnblogs-14397504/image-16.png)

参考: [E: Package 'python-software-properties' has no installation candidate](https://www.jianshu.com/p/5b62ff3d0a13)

`sudo apt-get install python-software-properties` を入力して解決した。

### Colaboratory の切断

この問題にはあまり困らなかった。ときどきページをクリックして操作していたので、最後まで無事に訓練できた。

ただし、この問題を解決しているブログもある: [如何让google colab不断连](https://blog.csdn.net/liupeng19970119/article/details/105625334)
