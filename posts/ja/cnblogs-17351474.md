## 概要

この記事は Prompt Pattern Catalog 論文の読書メモです。Prompt Engineering を場当たり的な文面調整として扱うのではなく、再利用可能なパターンとして整理する視点が中心です。

## Prompt Pattern とは

Prompt は LLM に与える命令であり、モデルの振る舞いを調整するためのインターフェースです。Prompt Pattern は、その命令の書き方を目的別に整理したものです。たとえば、出力形式を固定する Template Pattern、モデルに質問させる Flipped Interaction Pattern、役割を指定する Persona Pattern などがあります。

## 代表的なパターン

Meta Language Creation は、記号や短縮表現に独自の意味を与えるパターンです。Output Automater は、手順を説明するだけでなく、スクリプトや設定ファイルなど実行可能な成果物を生成させます。Question Refinement は、ユーザーの質問をより良い形に言い換え、Cognitive Verifier は回答前に補助質問を生成します。Fact Check List は、回答が依存している事実を列挙し、検証しやすくします。

## 感想

パターンとして名前を付けることで、Prompt を改善するときの議論がしやすくなります。一部は LangChain などのツールで置き換えられますが、ツールを使う前段階の設計としても有用です。
