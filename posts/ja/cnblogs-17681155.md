# HTML メモ

## Web ページまわりの基本概念

HTML は Hyper Text Markup Language、つまりハイパーテキストマークアップ言語であり、Web ページを記述するための言語である。

HTML はプログラミング言語ではなく、マークアップ言語である。

ハイパーテキストには、主に次のような意味がある。

- テキストだけに閉じない。画像、音声、アニメーション、マルチメディアなどを組み込める。

- ハイパーリンクを持つテキストである。あるファイルから別のファイルへ移動したり、世界中のホスト上にあるファイルとつながったりできる。

## よく使われるブラウザとそのエンジン

ブラウザエンジン、つまりレンダリングエンジンは、Web ページの情報を読み取り、整理し、ページをどのように表示するかを計算して画面に描画する部分である。

| IE | Trident | IE、猎豹安全浏览器、360 极速浏览器、百度浏览器 |
| Firefox | Gecko |  |
| Safari | Webkit |  |
| Chrome/Opera | Blink | Blink は Webkit から派生したエンジン |

中国国内のブラウザは、だいたい後ろ二つのエンジンを使っている。

## Web 標準

Web 標準は、W3C やその他の標準化団体によって策定された一連の標準の集合である。

W3C、World Wide Web Consortium は、国際的に最も有名な標準化団体の一つである。

Web 標準は主に、構造（Structure）、表現（Presentation）、振る舞い（Behavior）の三つから成る。

| 標準 | 説明 |
| --- | --- |
| 構造 | Web ページ上の要素を整理・分類するためのもの。現段階で主に学ぶのは HTML |
| 表現 | Web ページ要素のレイアウト、色、サイズなど、見た目のスタイルを設定するもの。主に CSS を指す |
| 振る舞い | Web ページモデルの定義やインタラクションの実装を指す。現段階で主に学ぶのは JavaScript |

## HTML タグ

- HTML タグは、山括弧で囲まれたキーワードで構成される。

- HTML タグは通常ペアで現れる。開始タグと終了タグからなるものを、ここでは双タグと呼ぶ。

- 一部の特殊なタグは単独で使う。たとえば `<br />` のようなタグである。

タグ同士の関係:

- 包含関係、つまり親子関係

- 並列関係、つまり兄弟関係

### HTML の基本構造タグ

各 Web ページには、基本となる構造タグ、いわば骨格となるタグがある。

| タグ名 | 定義 | 説明 |
| --- | --- | --- |
| html | HTML タグ | ページ内で最も大きなタグ。ルートタグと呼ばれる |
| head | 文書のヘッダー | `head` 内で必ず設定したいタグは `title` |
| title | 文書タイトル | ページに固有のタイトルを持たせる |
| body | 文書本体 | 文書内のすべての表示内容を含む。ページ内容は基本的に `body` 内に置く |

### DOCTYPE、lang、文字セットの役割

```
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>VS Code で作成した最初のページ</title>
</head>
<body>
    hello world again.
</body>
</html>
```

#### DOCTYPE

`<!DOCTYPE>` は文書型宣言であり、ブラウザにどの HTML バージョンでページを表示すべきかを伝える。

- 文書の先頭、`html` タグより前に置く必要がある。

- HTML タグではなく、文書型を宣言するための記述である。

#### lang

現在の文書の表示言語を定義する。

- `en`: 英語

- `zh-CN`: 中国語

これは実際に表示できる言語を制限するものではない。ブラウザに対するヒントとして働く。たとえば自動翻訳などで参照される。

#### 文字セット

文字セット（Character set）は複数の文字をまとめた集合であり、コンピュータが各種の文字を識別して保存できるようにするためのものだ。

`head` タグ内では、`meta` タグの `charset` 属性によって HTML 文書で使用する文字エンコーディングを指定できる。

```
<meta charset="UTF-8">
```

よく使われる値には、GB2312、BIG5、GBK、UTF-8 がある。このうち UTF-8 はユニコード系の代表的な文字コードとして広く使われる。

### タグのセマンティクス

セマンティクスとは、タグが持つ意味のことである。

タグの意味に合わせて適切な場所に適切なタグを使うと、ページ構造がより明確になる。

### 見出しタグ

```
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=<device-width>, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <h1>見出しタグ</h1>
    <h1>見出しは全部で六段階、</h1>
    <h2>文字は太く一行表示。</h2>
    <h3>大きい順に少しずつ小さくなり、</h3>
    <h4>重みも段階的に軽くなる。</h4>
    <h5>構文どおりに書いたら、</h5>
    <h6>更新して表示を確認する。</h6>
                ---------pink 先生
</body>
</html>
```

### 段落タグと改行タグ

`<p>` は Web ページを複数の段落に分けるために使う。

- 段落内のテキストは、ブラウザの幅に応じて自動的に折り返される。

- 段落と段落の間には余白が入る。

`<br />` は強制改行である。単に次の行へ移るだけで、段落間の余白は発生しない。

### テキスト整形タグ

文字を太字、斜体、下線などにして、テキストを強調するために使う。

| セマンティクス | タグ | 説明 |
| --- | --- | --- |
| 太字 | `<strong>` または `<b>` | より意味が強いので **太字には `<strong>` が推奨** |
| 斜体 | `<em>` または `<i>` | より意味が強いので *斜体には `<em>` が推奨* |
| 取り消し線 | `<del>` または `<s>` | より意味が強いので削除には `<del>` が推奨 |
| 下線 | `<ins>` または `<u>` | より意味が強いので挿入・下線には `<ins>` が推奨 |

### div と span タグ

この二つのタグ自体には特別な意味はなく、内容を入れるための箱として使う。

特徴:

- `div` はレイアウト用で、基本的に一行に一つ置かれる。

- `span` もレイアウト用だが、一行の中に複数置くことができる。

### 画像タグ

画像は `img` タグで挿入する。これは単独タグである。

`src` は画像タグの必須属性で、ファイルパスとファイル名を指定する。

その他の属性:

| 属性 | 属性値 | 説明 |
| --- | --- | --- |
| src | 画像パス | 必須属性 |
| alt | テキスト | 代替テキスト。画像を表示できないときに表示される文字 |
| title | テキスト | ツールチップ。マウスを画像に乗せたときに表示される文字 |
| width | ピクセル | 画像の幅を設定する |
| height | ピクセル | 画像の高さを設定する |
| border | ピクセル | 画像の枠線の太さを設定する |

- 画像タグには複数の属性を指定できる。属性はタグ名の後ろに書く。

- 属性の順番に決まりはない。タグ名と属性、属性同士はスペースで区切る。

- 属性はキーと値のペアで書く。

#### パス

- ディレクトリフォルダ: 通常のフォルダ。素材などを保存する。たとえば `web` フォルダの中に HTML ファイルや `img` フォルダを置く。

- ルートディレクトリ: ディレクトリフォルダを開いた最初の階層。たとえば `web` フォルダを開いた直下の階層。

画像を `image` フォルダに保存する場合、画像ファイルの位置を指定するためにパスを使う。パスは次のように分けられる。

- 相対パス: 参照元ファイルの位置を基準に作るディレクトリパスである。簡単に言えば、画像が使用するページから見てどこにあるか、という指定になる。

以下では、ページのパスが `web/pages/index.html` の場合を例にする。

相対パスの分類
記号
説明

同じ階層のパス

画像がページの横にある。`web/pages/img.jpg` の場合、相対パスは `img.jpg`。

一つ下の階層のパス
/
画像がページ横のフォルダ内にある。`web/pages/img/img.jpg` の場合、相対パスは `img/img.jpg`。

一つ上の階層のパス
../
画像がページのあるフォルダの隣にある。`web/img.jpg` の場合、相対パスは `../img.jpg`。

- 絶対パス: 完全なアドレス。通常はドライブ名から始まる。画像を右クリックして、プロパティの場所に表示されるパスとファイル名を合わせたもの。

### ハイパーリンクタグ

`<a>` はページ遷移に使う。

リンクの構文:

```
<a href="遷移先" target="リンク先ウィンドウの開き方">テキストまたは画像</a>
```

| 属性 | 役割 |
| --- | --- |
| href | リンク先の URL を指定する。必須属性 |
| target | リンク先ページの開き方を指定する。`_self` は既定値、`_blank` は新しいウィンドウで開く |

リンクの分類:

- 外部リンク

- 内部リンク

- 空リンク

- ダウンロードリンク: `href` に `.exe` や `.zip` ファイルを指定する

- Web ページ要素リンク: Web ページ上の要素にリンクを付ける

- アンカーリンク

#### アンカーリンク

リンクをクリックすると、ページ内の特定位置へ素早く移動できる。

- リンクテキストの `href` 属性に、`#名前` という形式の属性値を設定する。

```
<a href="#two">第 2 話</a>
```

- 移動先のタグを見つけ、その中に先ほどの名前と同じ `id` 属性を追加する。

```
<h3 id="two">第 2 話の紹介</h3>
```

### 表タグ

表はデータを表示するために使う。

```
<table>
    <tr><!-- 行。必ず table タグ内に置く -->
        <td>セル内のテキスト</td><!-- セル。必ず tr 内に置く -->
        ...
    </tr>
    ...
</table>
```

表見出し:

```
<table>
    <tr><!-- 行。必ず table タグ内に置く -->
        <th>表見出し</th><!-- 太字かつ中央寄せで表示される -->
        <td>セル内のテキスト</td><!-- セル。必ず tr 内に置く -->
        ...
    </tr>
    ...
</table>
```

表の属性はあまり使わない。通常は CSS で設定する。ただし、これらの単語は後で CSS でも使うので、表の見た目を直感的に理解するには役に立つ。

| 属性名 | 属性値 | 説明 |
| --- | --- | --- |
| align | left、center、right | 表を周囲の要素に対してどのように揃えるかを指定する |
| border | 1 または `""` | 表セルに枠線を付けるかどうかを指定する。既定値は `""` で、枠線なし |
| cellpadding | ピクセル値 | セルの端と内容の間の余白を指定する。既定値は 1 ピクセル |
| cellspacing | ピクセル値 | セル同士の間隔を指定する。既定値は 2 ピクセル |
| width | ピクセル値またはパーセント | 表の幅を指定する |

#### 表の構造タグ

表が長くなることもあるため、表の意味をより明確にするには、表ヘッダーと表本体の二つの大きな部分に分けられる。

`<thead>` は表のヘッダー領域を表す。`th` とは別物で、`th` は見出しセル、`thead` は領域を指す。

`<tbody>` は表の本体領域を表す。

#### セルの結合

- 行方向の結合: `rowspan="結合するセル数"`

- 列方向の結合: `colspan="結合するセル数"`

対象セル、つまり結合コードを書くセル:

- 行方向: 一番上のセルを対象セルとして、結合コードを書く。

- 列方向: 一番左のセルを対象セルとして、結合コードを書く。

手順:

- 行方向に結合するのか、列方向に結合するのかを決める。

- 対象セルを見つけ、結合コードを書く。

- 余分なセルを削除する。

### リストタグ

表がデータ表示のためのものなら、リストはレイアウトのために使う。

- 順序なしリスト

- 順序付きリスト

- 定義リスト

#### 順序なしリスト

```
<ul><!-- ul 内には li だけを入れる。その他の文字やタグを直接追加するのは不可 -->
    <li>リスト項目 1</li>
    <li>リスト項目 2</li>
    <li>リスト項目 3</li>
    ...
</ul>
```

#### 順序付きリスト

```
<ol><!-- ol 内には li だけを入れる。その他の文字やタグを直接追加するのは不可 -->
    <li>リスト項目 1</li>
    <li>リスト項目 2</li>
    <li>リスト項目 3</li>
    ...
</ol>
```

#### 定義リスト

定義リストは、用語や名詞の説明・記述によく使われる。

```
<dl><!-- dl 内には dt と dd だけを入れる。その他の文字やタグを直接追加するのは不可 -->
    <dt>名詞 1</dt><!-- dt と dd の数に制限はない -->
    <dd>名詞 1 の説明 1</dd>
    <dd>名詞 1 の説明 2</dd>
</dl>
```

### フォームタグ

完全なフォームは通常、フォーム領域、フォームコントロール、そして説明テキストの三つで構成される。

#### フォーム領域

フォーム要素を含む領域である。`<form>` タグはフォーム領域を定義し、その範囲内にあるフォーム要素の情報をサーバーへ送信する。

```
<form action="url" method="方式" name="フォーム領域名">
   各種フォーム要素コントロール
</form>
```

| 属性 | 属性値 | 役割 |
| --- | --- | --- |
| action | URL アドレス | フォームデータを受け取り処理するサーバープログラムの URL を指定する |
| method | get/post | フォームデータの送信方式を設定する。値は `get` または `post` |
| name | 名前 | フォーム名を指定し、同じページ内の複数フォームを区別する |

#### フォームコントロール

- `input` 入力フォーム要素

- `select` ドロップダウンフォーム要素

- `textarea` テキストエリア要素

##### input フォーム要素

```
<input type="属性値" />
```

| 属性値 | 説明 |
| --- | --- |
| button | クリック可能なボタン。多くの場合 JavaScript でスクリプトを起動する |
| checkbox | チェックボックス |
| file | 入力フィールドと参照ボタン。ファイルアップロード用 |
| hidden | 非表示の入力フィールド |
| image | 画像形式の送信ボタン |
| password | パスワードフィールド |
| radio | ラジオボタン |
| reset | リセットボタン。フォーム内のすべてのデータを消去する |
| submit | 送信ボタン。フォームデータをサーバーへ送信する |
| text | 一行入力フィールド。テキストを入力できる。既定幅は 20 文字 |

`type` のほかにも、`input` には次の属性がある。

| 属性 | 属性値 | 説明 |
| --- | --- | --- |
| name | 任意 | `input` の名前 |
| value | 任意 | `input` 要素の値 |
| checked | checked | 初回ロード時にこの `input` 要素が選択済みであることを指定する |
| maxlength | 正の整数 | 入力フィールドに入力できる最大文字数を指定する |

`name` と `value` は、各フォーム要素が持つ属性値で、主にバックエンド側で利用される。

同じグループのラジオボタンやチェックボックスには、同じ `name` を付ける。

##### label タグ

ラベル付け用のタグである。

フォーム要素と紐付けるために使う。`<label>` タグ内のテキストをクリックすると、ブラウザは自動的に対応するフォーム要素へフォーカス、または選択状態を移す。これによりユーザー体験がよくなる。

```
<label for="sex">男</label>
<!-- for の後ろには id の値を書く -->
<input type="radio" name="sex" id="sex" />
```

##### select タグ

複数の選択肢があり、ページ上のスペースを節約したい場合に使う。

```
<select>
    <option>選択肢 1</option>
    <option>選択肢 2</option>
    <option>選択肢 3</option>
    ...
</select>
```

属性に `selected` を書くと、既定で選択された状態にできる。

##### textarea テキストエリアタグ

ユーザーが多めのテキストを入力する場合は、`textarea` を使う。

```
<textarea rows="3" cols="20">
	テキスト内容
</textarea>
```

`cols` と `rows` は、実際の開発では CSS で制御することが多い。
