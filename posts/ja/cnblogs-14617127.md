まず、`packed` は「包装済み」、`padded` は「埋められた」という意味である。`pack` には包装する、詰めるという意味があり、`pad` には埋めるという意味がある。

`pack_padded_sequence` は、padding 済みの文を詰めてパックするという意味になる。

`pad_packed_sequence` は、詰めてパックされた文を再び padding するという意味になる。

模式図:

**pack_padded_sequence**

![](/assets/posts/cnblogs-14617127/image-01.png)

**pad_packed_sequence** は、上の過程を逆にしたもの。

### pack_padded_sequence

```
#官方函数
torch.nn.utils.rnn.pack_padded_sequence(input, lengths, batch_first=False)
#↓'返回'
return PackedSequence  #一个PackedSequence对象
```

機能: padding 済みの可変長系列を圧縮する。`PackedSequence` オブジェクトを返す。

#### パラメータ

- input: 圧縮するデータ。`batch_first` が False の場合、入力 shape は `[B,S,*]` で、B は batch_size、S は seq_len（その batch の最長系列長）、`*` は任意の次元である。`batch_first` が True の場合、対応するデータ形式は `[S,B,*]` でなければならない。input は系列長の長い順にソートされている必要があり、長いものが前、短いものが後ろに来る。最初の時間ステップのデータは最長データでなければならない。

- lengths: 入力データに含まれる各系列の長さ。

- batch_first: True の場合、データ形式は `[B,S,*]` でなければならない。そうでなければデフォルトは False。

### pad_packed_sequence

```
torch.nn.utils.rnn.pad_packed_sequence(sequence, batch_first=False) → tuple
#↓'返回'
return (sequence_pad , list) # 这个元组包含被填充后的序列 , 和batch中序列的长度列表。
```

機能: 上で述べた関数は padding 済みの可変長系列を圧縮する。この操作は `pack_padded_sequence()` の逆である。圧縮された系列を再び padding し、デフォルトでは戻り値の list の最大値に合わせて padding する。

#### パラメータ

- sequence: padding される batch。`PackedSequence` オブジェクト。

- batch_first: 上記 `pack_padded_sequence` の `batch_size` と同じ役割。ただし影響するのは出力データ形式である。

### 例

```
import torch

batch_size = 3   # 这个batch有3个序列
max_len = 6       # 最长序列的长度是6
embedding_size = 8 # 嵌入向量大小8
hidden_size = 16   # 隐藏向量大小16
vocab_size = 20    # 词汇表大小20

input_seq = [[3, 5, 12, 7, 2, ], [4, 11, 14, ], [18, 7, 3, 8, 5, 4]]
lengths = [5, 3, 6]   # batch中每个seq的有效长度。
# embedding
embedding = torch.nn.Embedding(vocab_size, embedding_size, padding_idx=0)
# LSTM的RNN循环神经网络
lstm = torch.nn.LSTM(embedding_size, hidden_size)
```

`pack_padded_sequence` では、input が系列長の長い順にソートされている必要がある。長いものが前、短いものが後ろで、最初の時間ステップのデータが最長でなければならない。

そのため、先にデータをソートする。

*zip は補足に置く。*

```
#由大到小排序
input_seq = sorted(input_seq, key = lambda tp: len(tp), reverse=True)
lengths = sorted(lengths, key = lambda tp: tp, reverse=True)

print(input_seq)
print(lengths)

PAD_token = 0 # 填充下标是0
def pad_seq(seq, seq_len, max_length):
    seq = seq
    seq += [PAD_token for _ in range(max_length - seq_len)]
    return seq

pad_seqs = []  # 填充后的数据
for i,j in zip(input_seq, lengths):
    pad_seqs.append(pad_seq(i, j, max_len))

print(pad_seqs)
```

[[18, 7, 3, 8, 5, 4], [3, 5, 12, 7, 2], [4, 11, 14]]
[6, 5, 3]
[[18, 7, 3, 8, 5, 4], [3, 5, 12, 7, 2, 0], [4, 11, 14, 0, 0, 0]]

padding 後の文は次の形式になる。

```
[[18, 7, 3, 8, 5, 4], 
 [3, 5, 12, 7, 2, 0], 
 [4, 11, 14, 0, 0, 0]]
```

つまり `[batch_size,seq_len,*]` なので、`pack_padded_sequence` では `batch_first=True` になる。

LSTM の出力次元:

LSTM が双方向の場合は `num_directions=2`、そうでなければ `num_directions=1`。

| パラメータ | 次元1 | 次元2 | 次元3 |
| --- | --- | --- | --- |
| output | seq_len | batch | num_directions×hidden_size |

```
pad_seqs = torch.tensor(pad_seqs)
embeded = embedding(pad_seqs)

# 压缩，设置batch_first为true
pack = torch.nn.utils.rnn.pack_padded_sequence(embeded, lengths, batch_first=True)
#这里如果不写batch_first,你的数据必须是[s,b,e]，不然会报错lenghth错误

#　利用lstm循环神经网络测试结果
state = None
pade_outputs, _ = lstm(pack, state)
# 设置batch_first为true;你可以不设置为true,为false时候只影响结构不影响结果
pade_outputs, others = torch.nn.utils.rnn.pad_packed_sequence(pade_outputs, batch_first=True)

# 查看输出的元祖
print(pade_outputs.shape)
print(others)
```

torch.Size([3, 6, 16])
tensor([6, 5, 3])
