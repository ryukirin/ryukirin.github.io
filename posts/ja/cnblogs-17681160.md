# CSS メモ

CSS は Web ページの見た目を整えるためのものだ。

CSS は Cascading Style Sheets、つまりカスケーディングスタイルシートの略である。

CSS スタイルシート、または階層スタイルシートと呼ばれることもある。

CSS も一種のマークアップ寄りの言語として扱える。

## CSS の概要

セレクタ + 宣言で構成する。

```
h1 {
    color: red;
    ...
}
```

セレクタ `{ 属性: 値 }` という形で書く。

### CSS コードスタイル

- スタイルの書き方: 展開形式で、1 つのスタイルを 1 行に書く。

- 大文字小文字: 小文字で書く。

- スペース規約:

属性値の前、つまりコロンの後ろにはスペースを 1 つ入れる。

- セレクタと波括弧の間にもスペースを 1 つ入れる。

## CSS の基本セレクタ

セレクタは、どのタグを選ぶかを指定するためのものだ。

基本セレクタは、単体のセレクタで構成される。

- タグセレクタ: HTML タグ名をセレクタとして使う。同じタグをすべて選択してスタイルを設定する。

```
タグ名 {
	属性1: 属性値1;
    属性2: 属性値2;
    属性3: 属性値3;
    ...
}
```

- クラスセレクタ: 1 つ、または複数のタグを個別に選択できる。差分のあるスタイル指定に向いている。

```
.クラス名 {
	属性1: 属性値1;
	...
}
```

クラスセレクタは `.` で表し、その後ろにクラス名を続ける。

- 対象タグに名前を付ける、と理解すると分かりやすい。

- 長い名前や複数語の名前は、ハイフンでつなげて命名できる。

- 純数字や中国語名は避け、できるだけ英語で命名する。

- 名前には意味を持たせ、見た人がすぐ用途を分かるようにする。

クラスセレクタには、複数クラス名という特殊な使い方もある。

```
<div class="クラス名1 クラス名2">
    ...
</div>
```

共通スタイルと固有スタイルを分けられるため、コード量を減らし、修正もしやすくなる。

id セレクタ:

```
#id名 {
    属性1: 属性値1;
    ...
}
```

クラスセレクタとの最大の違いは、同じ id は 1 回しか呼び出さない点である。

開発では通常、ページ内で一意な要素に使い、JavaScript と組み合わせることが多い。

ユニバーサルセレクタは、すべてのタグを選択する。

```
* {
    属性1: 属性値1;
    ...
}
```

呼び出しは不要で自動的に適用される。特殊な場面でだけ使う。

| 基本セレクタ | 役割 | 特徴 | 使用頻度 | 使い方 |
| --- | --- | --- | --- | --- |
| タグセレクタ | 同じタグをすべて選べる。例: p | 個別指定はできない | 多い | p {
color: red;
} |
| クラスセレクタ | 1 つまたは複数のタグを選べる | 要件に応じて選択できる | 非常に多い | .nav {
color: red;
} |
| id セレクタ | 一度に 1 つのタグだけ選ぶ | id 属性は HTML 文書内で一度だけ使う | JS と組み合わせることが多い | #nav {
color: red;
} |
| ユニバーサルセレクタ | すべてのタグを選ぶ | 選択範囲が広すぎる場合がある | 特殊な場面 | * {
color: red;
} |

## CSS のフォント属性

### フォントファミリー

`font-family` でテキストのフォントファミリーを定義する。

```
p {
    font-family: "微软雅黑";
}
div {
    font-family: Arial,"微软雅黑","Microsoft Yahei";
}
```

どのユーザーのブラウザでも正常に表示できるように、なるべくシステム既定フォントを使う。

よく使われるフォントは `"Microsoft Yahei", tahoma, arial, "Hiragino Sans GB"` など。

### フォントサイズ

`font-size` には必ず `px` を付ける。Chrome の既定値は 16px だが、既定に頼らず明示的に指定する。

見出しタグは少し特殊なので、サイズを個別に指定する必要がある。

### フォントの太さ

`font-weight`:

- `normal`: 通常

- `bold`: 太字

- `bolder`: より太い

- `lighter`: 細い

- `number`: 数値で太さを制御する。単位は付けない。100 から 900 まで、100 刻み。実務ではこちらが推奨される。400 は `normal` 相当。

### フォントスタイル

`font-style`:

- `normal`: 既定値。標準のフォントスタイル。

- `italic`: 斜体。

### 複合属性

```
body {
    font: font-style font-weight font-size/line-height font-family;
}
```

順序は入れ替えられない。不要な属性は省略できるが、`font-size` と `font-family` は必ず残す。そうしないと効かない。

| 属性 | 意味 | 注意点 |
| --- | --- | --- |
| font-size | 文字サイズ | 通常使う単位は px。必ず単位を付ける |
| font-family | フォント | 実務ではチームの規約に従う |
| font-weight | 太さ | 太字は 700 または bold、通常は normal または 400。数値には単位を付けない |
| font-style | フォントスタイル | 斜体は italic、非斜体は normal。実務では normal で斜体を解除することも多い |
| font | フォント一括指定 | 順序があり、位置を入れ替えられない。文字サイズとフォントは必須 |

## CSS のテキスト属性

### テキストカラー

`color`

- 事前定義された色名: `red`、`blue` など。

- 16 進数: `#FF0000` など。開発では最もよく使う。

- RGB コード: `rgb(255,0,0)` または `rgb(100%,0%,0%)` など。

### 揃え

`text-align`

- `left`: 左揃え。既定値。

- `right`: 右揃え。

- `center`: 中央揃え。

### テキスト装飾

`text-decoration`

- `none`: 既定値。何も付けない。最もよく使う。`a`、つまりリンクの既定の下線を消せる。

- `underline`: 下線。よく使う。

- `overline`: 上線。ほぼ使わない。

- `line-through`: 取り消し線。あまり使わない。

### テキストインデント

`text-indent`

- `px`: ピクセル。

- `em`: 相対単位。現在要素の `font-size` における 1 文字分の大きさ。未設定なら親要素の 1 文字分。

### 行間

`line-height`。行間 = 上の余白 + テキスト高さ + 下の余白。

行高は、前の行の最下端から現在行の最下端までで測る。

| 属性 | 意味 | 注意点 |
| --- | --- | --- |
| color | テキスト色 | 16 進数をよく使う。`#FFF` のような省略形も使う |
| text-align | テキスト揃え | 揃え方を設定する |
| text-indent | テキストインデント | 先頭行を 2 文字分下げる: `text-indent: 2em;` |
| text-decoration | テキスト装飾 | 重点は下線追加の `underline` と下線解除の `none` |
| line-height | 行高 | 行と行の距離を制御する。行間 = 上余白 + テキスト高さ + 下余白 |

## CSS の導入方式

スタイルシートは三種類ある。

- インラインスタイルシート

- 内部スタイルシート、埋め込み式

- 外部スタイルシート、リンク式

### 内部スタイルシート

```
<style>
	css...
</style>
```

理論上は HTML 文書のどこにでも置けるが、一般的には `head` タグ内に置く。ページ全体のスタイルを制御でき、構造とスタイルはある程度分離されるが、完全には分離されない。

### インラインスタイルシート

簡単なスタイルを変更する場合に向いている。

```
<div style="color: red;">
    ...
</div>
```

現在のタグだけを制御できる。ただし大量使用は推奨されない。

### 外部スタイルシート

最もよく使う方式。CSS を単独の `.css` ファイルに書き、HTML から読み込む。

- まず拡張子 `.css` のスタイルファイルを作り、CSS コードをすべて入れる。

- HTML ファイルに読み込む。

```
<link rel="stylesheet" href="xxx.css">
```

- `rel`: 現在の文書とリンク先文書の関係を示す。ここでは `stylesheet` を指定する。

- `href`: リンク先文書のパス。絶対パスでも相対パスでもよい。

| スタイルシート | 長所 | 短所 | 使用頻度 | 制御範囲 |
| --- | --- | --- | --- | --- |
| インライン | 書きやすく、優先度が高い | 構造とスタイルが混ざる | 少ない | 1 つのタグ |
| 内部 | 構造とスタイルを部分的に分離できる | 完全には分離されない | 多い | 1 ページ |
| 外部 | 構造とスタイルを完全に分離できる | 読み込みが必要 | 最も多い | 複数ページ |

設定が衝突した場合、`style` と `link` のうち、後ろ、つまり `body` に近い方が有効になる。

## Emmet 構文

### HTML 構造を素早く生成する構文

- タグを生成するには、タグ名を入力して Tab キー。

- 同じタグを複数生成するには、タグ名 + `*` + Tab キー。

- 親子関係のタグは `>` を使う。例: `ul>li` で順序なしリストを生成できる。

- 兄弟関係のタグは `+` を使う。

- クラス名や id 名付きのタグは、`#id` または `.class` と書けばよい。既定は `div`。前にタグ名を付けると他のタグも生成できる。例: `ul>li#two` は次のようになる。

```
<ul>
    <li id="two"></li>
</ul>
```

- 生成する `div` に連番を付けたい場合は、自動採番記号 `$` を使う。例: `.demo$*5` は次のようになる。

```
<div class="demo1"></div>
<div class="demo2"></div>
<div class="demo3"></div>
<div class="demo4"></div>
<div class="demo5"></div>
```

- 生成したタグの内部に内容を書きたい場合は `{}` を使う。例: `div{test}*5` は次のようになる。

```
<div>test</div>
<div>test</div>
<div>test</div>
<div>test</div>
<div>test</div>
```

## CSS の複合セレクタ

複合セレクタを使うと、対象要素をより正確かつ効率的に選択できる。

- 子孫セレクタ

- 子セレクタ

- グループセレクタ

- 疑似クラスセレクタ

### 子孫セレクタ

包含セレクタとも呼ばれる。

親要素の中にある子孫要素を選択できる。

### 子セレクタ

ある要素の直近一階層の子要素だけを選択する。簡単に言えば、直接の子だけを選ぶ。

```
div>p {
    ...
}
```

### グループセレクタ

複数組のタグを選択し、同時にスタイルを定義できる。

```
div,
p {
    ...
}
```

### 疑似クラスセレクタ

特定のセレクタに特殊効果を追加するために使う。たとえばリンクに状態別の効果を付けたり、1 番目または n 番目の要素を選んだりする。

特徴はコロンで表すこと。例: `:hover`。

種類は多く、リンク疑似クラス、構造疑似クラスなどがある。

#### リンク疑似クラス

```
/* 未訪問リンクをすべて選択 */
a:link
/* 訪問済みリンクをすべて選択 */
a:visited
/* マウスポインタが乗っているリンクを選択 */
a:hover
/* アクティブなリンク、つまりマウスが押されてまだ離されていないリンクを選択 */
a:active
```

順序は変えない。LVHA の順で書く。順序を間違えると効かない場合がある。

リンクにはブラウザ独自の既定スタイルがある。`body` にスタイルを設定してもリンクには効かないことがあるので、リンクには必ず個別にスタイルを指定する。

#### focus 疑似クラス

フォーカス、つまりカーソルを取得したフォーム要素を選択するために使う。

一般的には `input` 系のフォームだけが取得できる。

```
input:focus {
    background-color: yellow;
}
```

| セレクタ | 役割 | 特徴 | 使用頻度 | 区切り記号/使用例 |
| --- | --- | --- | --- | --- |
| 子孫セレクタ | 子孫要素を選択 | 子や孫を選べる | 多い | スペース / `div a` |
| 子セレクタ | 直近一階層の要素を選択 | 直接の子だけ | 少ない | 大なり記号 / `div>a` |
| グループセレクタ | 複数タグの同種要素を選択 | まとめて宣言できる | 多い | カンマ / `div,a` |
| リンク疑似クラス | 状態ごとのリンクを選択 | リンクに関係する | 多い | 開発では通常 `a{}` で通常状態を制御し、`a:hover` でマウスホバー時を定義すればよい |
| focus 疑似クラス | フォーカス中のフォームを選択 | フォームに関係する | 少ない | `input:focus` |

## CSS の要素表示モード

要素がどの方式で表示されるかを指す。

一般的にはブロック要素とインライン要素に分けられる。

### ブロック要素

`h1` から `h6`、`p`、`div`、`ul`、`ol`、`li` など。なかでも `div` は最も典型的なブロック要素である。

- 1 行を独占する。

- 高さ、幅、外側余白、内側余白を制御できる。

- 幅の既定値はコンテナ、つまり親要素の幅の 100%。

- コンテナであり箱でもある。中にインライン要素またはブロック要素を入れられる。

注意:

- 文字系要素の中にはブロック要素を入れない。

- `p`、`h1` から `h6` などは文字系要素である。

### インライン要素

インライン要素とも呼ばれる。

代表例は `a`、`strong`、`b`、`em`、`i`、`del`、`s`、`ins`、`u`、`span` など。

なかでも `span` が最も典型的である。

- 高さと幅の指定は効かない。

- インライン要素はテキスト、または他のインライン要素だけを含める。

- `a` の中にさらに `a` を入れてはいけない。

- 例外的に `a` の中にブロック要素を入れることはできるが、その場合は `a` をブロック表示に変換しておくのが望ましい。

### インラインブロック要素

`img`、`input`、`td` などは、ブロック要素とインライン要素の特徴をあわせ持つ。資料によってはインラインブロック要素と呼ぶ。

- 一行に並ぶが隙間が出る。一行内に複数置ける。これはインライン要素の特徴。

- 既定の幅は自身の内容の幅。これもインライン要素の特徴。

- 高さ、行高、内外余白を制御できる。これはブロック要素の特徴。

| 要素モード | 要素の並び | スタイル設定 | 既定幅 | 含められるもの |
| --- | --- | --- | --- | --- |
| ブロック要素 | 1 つで 1 行を占める | 幅と高さを設定できる | コンテナの 100% | 任意のタグを含められる |
| インライン要素 | 一行に複数置ける | 幅と高さは設定できない | 内容自体の幅 | テキストまたは他のインライン要素 |
| インラインブロック要素 | 一行に複数置ける | 幅と高さを設定できる | 内容自体の幅 |  |

### 要素表示モードの変換

特殊な場面では、要素モードを変換する必要がある。つまり、あるモードの要素に別のモードの特性を持たせたい場合である。

たとえばリンク `a` のクリック可能範囲を広げたい場合。

- ブロック要素へ変換: `display:block;`

- インライン要素へ変換: `display:inline;`

- インラインブロック要素へ変換: `display:inline-block;`

### 垂直中央揃え

一行テキストは、行高 = ボックスの高さにすることで垂直中央揃えにできる。

たとえば `div` 内の文字なら、`line-height` の値を `height` の値と同じにすればよい。

## CSS の背景

### 背景色

`background-color: transparent|color;`

透明、つまり既定値、または色を指定する。

### 背景画像

位置を制御しやすい。

`background-image: none|url();`

### 背景の繰り返し

`background-repeat: repeat|no-repeat|repeat-x|repeat-y;`

繰り返し / 繰り返さない / 横方向に繰り返す / 縦方向に繰り返す。

### 背景画像の位置

`background-position: x y;`

`x` と `y` には次の値を指定できる。

- 正確な単位: パーセント、長さ値。1 つ目は必ず X、2 つ目は必ず Y。

- 方位名詞: `top` / `center` / `bottom` / `left` / `right`。この場合、前後の順序は問わない。2 つ目の引数を省略すると既定で中央になる。

- 混合単位: 片方を正確な単位、片方を方位にできる。順序は必ず X、Y の順。

### 背景画像の固定、背景アタッチメント

`background-attachment: fixed|scroll;`

固定 / スクロール。

### 複合指定

`background: 色 画像 繰り返し スクロール 位置`

厳密な固定順序はないが、よく使われる順序は上記である。

### 背景色の半透明

`background: rgba(0,0,0,0);`

最後の値は透明度を表す。0 から 1 の範囲で、1 は不透明、0 は完全透明。

内容自体の透明度には影響しない。

| 属性 | 役割 | 値 |
| --- | --- | --- |
| background-color | 背景色 | 事前定義色 / 16 進数 / RGB |
| background-image | 背景画像 | `url(パス)` |
| background-repeat | 繰り返すかどうか | repeat\|no-repeat\|repeat-x\|repeat-y |
| background-position | 背景位置 | x y。正確な長さ、または方位名詞 |
| background-attachment | 背景アタッチメント | scroll\|fixed |
| background 一括指定 | 書き方を簡略化する | 色 画像 繰り返し スクロール 位置 |
| 背景色半透明 | 背景色を半透明にする | `rgba(0,0,0,0)` の最後の値が透明度。0 から 1、1 は不透明、0 は完全透明 |

## CSS の三大特性

カスケード、継承、優先度。

### カスケード

同じセレクタに同じスタイルを設定すると、カスケードの原則が働く。

- スタイルが衝突した場合は、構造に近い方が実行される。外部ファイルであれば、下にある定義が有効になる。

- スタイルが衝突しなければ、上書きは起きない。

### 継承

子タグは親タグの一部のスタイルを継承する。これによりコードを簡潔にできる。

継承されやすいものには `text-`、`font-`、`line-`、`color` などがある。

#### 行高の継承

行高には単位を付けても付けなくてもよい。単位を付けない場合、現在の子要素のフォントサイズに対する倍率になる。

単位なしの大きな利点は、子要素が自分の文字サイズに応じて自動的に調整できる点である。

### 優先度

1 つの要素に複数のセレクタが指定された場合:

- セレクタが同じなら、カスケードが適用される。

- セレクタが違うなら、セレクタの重みによって決まる。

重みは次のとおり。

| セレクタ | セレクタの重み（変化せず、繰り上がらず、同じ桁だけ加算し、左から右へ比較する） |
| --- | --- |
| 継承または `*` | 0,0,0,0 |
| 要素セレクタ | 0,0,0,1 |
| クラスセレクタ、疑似クラスセレクタ | 0,0,1,0 |
| ID セレクタ | 0,1,0,0 |
| インラインスタイル `style=""` | 1,0,0,0 |
| `!important` | 無限大相当 |

リンクには、ブラウザが既定で青色・下線付きのスタイルを指定している。そのため継承された `text-decoration` と `color` はどちらも無効になる。

#### 重みの加算

複合セレクタの場合、重みが加算されるため、計算が必要になる。

重みは変化せず、繰り上がらず、同じ桁だけ加算し、比較は左から右に行う。

注:

`a:hover` の重みは `0,0,1,1` であり、`0,0,1,0` ではない。前にタグセレクタ `a` があるためである。

例: [hover が効かない理由の一つ、重みに関係するケース](https://blog.csdn.net/weixin_46102597/article/details/104228185)

## ボックスモデル

Web ページレイアウトの流れ:

- 関連する Web ページ要素を用意する。Web ページ要素は基本的にすべて Box、つまり箱である。

- CSS でボックスのスタイルを設定し、対応する位置に配置する。

- ボックスの中に内容を入れる。

Web ページレイアウトの本質は、ボックスを使ってレイアウトを組むことだ。

### 構成要素

ボックスモデルは、HTML ページ内のレイアウト要素を矩形の箱、つまり内容を入れるコンテナとして扱う考え方である。

CSS ボックスモデルの本質は 1 つの箱であり、周囲の HTML 要素を、枠線、外側余白、内側余白、実際の内容として包み込む。

- `border`: 枠線

- `content`: 内容

- `padding`: 内側余白。枠線と内容の距離

- `margin`: 外側余白。ボックス同士の距離

#### 枠線 border

枠線の幅、枠線スタイル、枠線色を指定する。

```
border : border-width || border-style || border-color
```

`border` の幅、スタイル、色に固定順序はない。

```
border-top: 1px solid red;
border-bottom: 5px solid red;
border-left: 5px solid yellow;
border-right: 5px solid purple;
```

表の細線ボーダー:

```
/* 隣接する枠線を結合する */
 border-collapse: collapse;
```

- 枠線はボックスの実際のサイズに影響する。たとえば 200 * 200px のボックスに 10px の枠線を付けると、実際のサイズは 220 * 220px になる。

#### 内側余白 padding

| 値の個数 | 意味 |
| --- | --- |
| padding: 5px; | 上下左右すべて 5px |
| padding: 5px 10px; | 上下 5px、左右 10px |
| padding: 5px 10px 20px; | 上 5px、左右 10px、下 20px |
| padding: 5px 10px 20px 30px; | 上 5px、右 10px、下 20px、左 30px。時計回り |

- 内側余白もボックスの実際のサイズに影響する。たとえば 200 * 200px のボックスに 10px の padding を付けると、実際のサイズは 220 * 220px になる。

- `width` を指定していない場合、ボックス幅は変わらない。

```
h1 {
    /* この時点で h1 には幅を設定していない。
    そのため長さは既定で body の長さになる。
    padding は高さだけを変え、幅は変えない。 */
    height: 40px;
    padding: 30px;
}
h2 {
    /* 比較:
    この時点で h2 には幅を設定している。
    padding は幅も変える。 */
    height: 40px;
    padding: 30px;
    width: 1000px;
}
```

#### 外側余白 margin

| 値の個数 | 意味 |
| --- | --- |
| margin: 5px; | 上下左右すべて 5px |
| margin: 5px 10px; | 上下 5px、左右 10px |
| margin: 5px 10px 20px; | 上 5px、左右 10px、下 20px |
| margin: 5px 10px 20px 30px; | 上 5px、右 10px、下 20px、左 30px。時計回り |

指定方法は `padding` と完全に同じ。

`margin` でブロックボックスを水平中央寄せするには:

- ボックスに幅を指定していること。

- 左右の外側余白をどちらも `auto` にする。

- インライン要素またはインラインブロック要素は、親要素に `text-align:center` を追加すればよい。

#### 外側余白の結合、またはマージンの相殺

- 隣接するブロック要素の垂直方向マージン結合: 上下に隣接した 2 つのブロック要素、つまり兄弟関係の要素が接するとき、上の要素に `margin-bottom`、下の要素に `margin-top` があると、両者の間隔は大きい方の値になる。

- 入れ子ブロック要素の垂直方向マージン相殺: 親子関係の 2 つのブロック要素で、親要素に `margin-top` があり、子要素にも `margin-top` がある場合、親要素の `margin-top` は二つの値の大きい方になり、子要素と親要素の間に上余白は発生しない。注: この現象は、親要素の最初の子要素で発生する。

解決方法:

親要素に上枠線を定義する。

- 親要素に上 padding を定義する。

- 親要素に `overflow:hidden` を追加する。

#### 内外余白のリセット

多くの Web ページ要素には既定の内外余白があり、ブラウザによっても異なる。そのためレイアウト前に内外余白をリセットする必要がある。

```
* {
    padding: 0;
    margin: 0;
}
```

インライン要素には、できるだけ左右の内外余白だけを設定する。インライン要素の上下余白は効かないためである。どうしても上下余白が必要なら、`display: block;` または `display: inline-block;` にする。

### 角丸ボーダー

```
border-radius:length;
```

`length` は、矩形の角に接する円の半径である。数値でもパーセントでも指定できる。パーセントの場合、多くは楕円と矩形角が接する形になり、長軸と短軸は幅・高さの x% になる。ボックスが正方形なら円になる。

- 正方形の場合、`length = 高さの半分` または `50%` で円を描ける。

- 長方形の場合、`length = 高さの半分` にすると、両端が半円で中央が矩形の形にできる。

- この属性はショートハンドで、4 つの値を指定できる。順序は左上、右上、右下、左下。個別指定は `border-top-left-radius`、`border-top-right-radius`、`border-bottom-right-radius`、`border-bottom-left-radius`。

### ボックスシャドウ

```
box-shadow: h-shadow v-shadow blur spread color inset;
```

| 値 | 説明 |
| --- | --- |
| h-shadow | 必須。水平シャドウの位置。正の値は右、負の値は左 |
| v-shadow | 必須。垂直シャドウの位置。正の値は下、負の値は上 |
| blur | 任意。ぼかし距離 |
| spread | 任意。シャドウのサイズ |
| color | 任意。シャドウの色 |
| inset | 任意。外側シャドウ（outset）を内側シャドウに変更する |

シャドウはスペースを占有せず、レイアウトにも影響しない。

### テキストシャドウ

```
text-shadow: h-shadow v-shadow blur color;
```

| 値 | 説明 |
| --- | --- |
| h-shadow | 必須。水平シャドウの位置。正の値は右、負の値は左 |
| v-shadow | 必須。垂直シャドウの位置。正の値は下、負の値は上 |
| blur | 任意。ぼかし距離 |
| color | 任意。シャドウの色 |

## CSS float

### float

#### 従来の Web ページレイアウトの三つの方式

Web ページレイアウトの本質は、CSS でボックスを配置し、対応する位置に置くことである。

- 通常フロー、標準フロー

- float

- position

一般的な Web ページでは、三つすべてのレイアウト方式が含まれる。

float は既定の並び方を変えられる。

最も典型的な用途は、複数のブロック要素を隙間なく一行に並べること。

**複数のブロック要素を縦に並べるなら標準フロー、横に並べるなら float を使う。**

#### 通常フロー、標準フロー、文書フロー

タグの既定の方式に従って並ぶ。

- ブロック要素は 1 行を独占し、上から下へ並ぶ。

- インライン要素は左から右へ並び、親要素の端に到達すると自動で折り返す。

#### float

`float` は浮動ボックスを作り、左端または右端、包含ブロックまたは別の浮動ボックスの端に当たるまで片側へ移動させる。

```
セレクタ {
    float: 属性;
}
```

| 属性値 | 説明 |
| --- | --- |
| none | 要素を浮動させない。既定値 |
| left | 左に浮動 |
| right | 右に浮動 |

#### float の特徴

- float された要素は標準フローから外れる。元の位置は保持されず、上のレイヤーへ移動して、次の標準フローのボックスを覆うことがある。

- float 要素は一行内に表示され、要素の上端が揃う。親要素に入りきらない場合だけ次の行へ移る。

- float 要素はインラインブロック要素の特徴を持つ。

float 要素の位置を制御するため、Web ページレイアウトでは一般的に次のようにする。

- 標準フローの親要素で上下位置を配置し、内部の子要素は float で左右位置を配置する。

- 先にボックスサイズを設定し、それからボックス位置を設定する。

#### float レイアウトの注意点

- float と標準フローの親ボックスを組み合わせる。

まず標準フローの親要素で上下位置を並べ、その後、内部の子要素を float で左右に並べる。

- ある要素を float させるなら、理論上は他の兄弟要素も float させる。float ボックスは後続の標準フローにだけ影響する。例:

ABC という三つの `div` がすべて標準フローなら、三つは縦に並ぶ。A が float、B が標準フロー、C が float の場合、B は A に覆われ、C は依然として B の下に縦並びで置かれる。

### float の解除

親ボックスには高さを付けにくいことが多い。しかし子ボックスが float すると場所を占有しない。そのため親ボックスに高さがなく、子ボックスが float している場合、下にある標準フローのボックスが現在の子ボックスに一部覆われる。そこで float を解除する必要がある。

float 解除の本質は、float 要素が引き起こす影響を取り除くことだ。

```
セレクタ {
    clear: 属性値;
}
```

| 属性値 | 説明 |
| --- | --- |
| left | 左側に float 要素を許可しない。左 float の影響を解除 |
| right | 右側に float 要素を許可しない。右 float の影響を解除 |
| both | 左右両側の float 影響を同時に解除。最もよく使う。基本的にはこれだけ使う |

float 解除の方針は、float を閉じ込めることである。

- 追加タグ法、隔壁法

- 親要素に `overflow` 属性を追加

- 親要素に `after` 疑似要素を追加

- 親要素に二重疑似要素を追加

#### 追加タグ法

float 要素の末尾に空タグを追加する。例:

```
<div style="clear:both"></div>
または
<br />
```

例:

```
<style>
    .clear {
        clear: both;
    }
</style>
<!-- body 内に空タグを追加する。
その他 class の詳細スタイルは省略。 -->
<body>
    <div class="box">
        <div class="one"></div>
        <div class="two"></div>
        <!-- 新しく追加するタグはブロック要素または br でなければならない -->
        <div class="clear"></div>
    </div>
    <div class="footer"></div>
</body>
```

- 長所: 分かりやすく、書きやすい。

- 短所: 意味のないタグが多くなり、構造としてはあまりよくない。

#### 親要素に overflow 属性を追加

```
セレクタ {
    overflow: hidden/auto/scroll;
}
```

- 長所: コードが簡潔。

- 短所: はみ出した部分を表示できない。

#### 親要素に after 疑似要素を追加

追加タグ法の改良版である。

```
<style>
    .clearfix::after {
            content: "";
            display: block;
            height: 0;
            clear: both;
            visibility: hidden;
        }

        .clearfix {
            /* IE6、7 専用 */
            *zoom: 1;
        }
</style>

親要素の元の class が box なら、次のように書く。

<div class="box clearfix">
    子要素
</div>
```

- 長所: 追加タグを増やさず、構造がより簡潔。

- 短所: 古いブラウザへの対応が必要。

#### 親要素に二重疑似要素を追加

```
<style>
        .clearfix::before,
        .clearfix::after {
            content: "";
            display: table;
        }

        .clearfix::after {
            clear: both;
        }

        .clearfix {
            /* IE6、7 専用 */
            *zoom: 1;
        }
</style>

親要素の元の class が box なら、次のように書く。

<div class="box clearfix">
    子要素
</div>
```

- 長所: コードがさらに簡潔。

- 短所: 古いブラウザへの対応が必要。

### よく使う画像形式

- gif: 最大 256 色までしか保存できない。通常は単純な図形や文字に使うが、透過背景とアニメーション効果も保存できる。小さなアニメーション画像でよく使う。

## CSS position

### なぜ position が必要なのか

標準フローでは実現できないケースがある。

- ある要素をボックス内で自由に動かし、他のボックスの上に重ねたい。

- ウィンドウをスクロールしても、ボックスを画面上に固定したい。いわゆる浮動ウィンドウ。

そのため position が必要になる。

- float は複数のブロックボックスを隙間なく並べられるため、横並びレイアウトでよく使う。

- position は、ボックスを特定のボックス内で自由に動かしたり、画面内の特定位置に固定したり、他のボックスの上に重ねたりできる。

position もボックスを配置するための仕組みであり、位置指定の方式に従ってボックスを移動させる。

### position の構成

position = 位置指定モード + オフセット。

#### 位置指定モード

CSS の `position` 属性で設定する。値は主に四つ。

| 値 | 意味 |
| --- | --- |
| static | 静的配置 |
| relative | 相対配置 |
| absolute | 絶対配置 |
| fixed | 固定配置 |

#### オフセット

位置指定されたボックスを最終位置まで移動するための値で、`top` / `bottom` / `left` / `right` の四つの属性がある。

| オフセット属性 | 例 | 説明 |
| --- | --- | --- |
| top | top: 80px; | 上端オフセット。要素が親要素の上辺からどれだけ離れるかを定義する |
| bottom | bottom: 80px; | 下端オフセット。要素が親要素の下辺からどれだけ離れるかを定義する |
| left | left: 80px; | 左側オフセット。要素が親要素の左辺からどれだけ離れるかを定義する |
| right | right: 80px; | 右側オフセット。要素が親要素の右辺からどれだけ離れるかを定義する |

#### 静的配置 static

既定の配置方式であり、位置指定しないという意味である。

```
セレクタ {
    position: static;
}
```

標準フローに従って配置され、オフセットは使えない。

ほとんど使わない。

#### 相対配置 relative

要素が移動するとき、自分自身の元の位置を基準にする。

```
セレクタ {
    position: relative;
}
```

元の標準フロー内の位置は引き続き占有するため、後続のボックスには影響しない。たとえば上下に密着していた二つのボックスのうち、上のボックスを下へ移動しても、下のボックスは動かない。その結果、下のボックスを覆うことがある。

#### 絶対配置 absolute

絶対配置では、要素は祖先要素を基準に移動する。

```
セレクタ {
    position: absolute;
}
```

- 祖先要素がない、または祖先要素に position がない場合は、ブラウザを基準に配置される。

- 祖先要素に position がある場合は、最も近い position 付き祖先要素を基準点として移動する。

- 絶対配置は元の位置を占有しない。標準フローから外れる。

#### 子は absolute、親は relative

子要素を絶対配置にする場合、親要素は相対配置にするのが基本である。

- 子要素は絶対配置で場所を占有しないため、親要素内の任意の位置に置ける。他の兄弟ボックスには影響しない。

- 親ボックスに position を付けることで、子ボックスを親ボックス内に制限できる。

- 親ボックスはレイアウト上の位置を占有する必要があるため、通常は相対配置にする。

#### 固定配置 fixed

固定配置は、要素をブラウザの可視領域内の位置に固定する。主に、ページをスクロールしても要素位置が変わらない場面で使う。浮動ウィンドウなど。

```
セレクタ {
    position: fixed;
}
```

- ブラウザの可視ウィンドウを基準点として要素を移動する。

- 元の位置は占有しない。

特殊な絶対配置と見なすこともできる。

##### tips: 版心の右側に固定する

- 固定配置のボックスに `left: 50%;` を設定し、ブラウザ可視領域、あるいは版心の半分まで移動させる。

- 固定配置のボックスに `margin-left: 版心幅の半分` を設定し、さらに版心幅の半分だけ右へ移動させる。

例:

```
<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
        .page {
            height: 1000px;
            width: 1200px;
            margin: auto;
            background-color: bisque;
        }

        .fixed {
            position: fixed;
            top: 200px;
            left: 50%;
            margin-left: 600px;
            width: 100px;
            height: 100px;
            background-color: cornflowerblue;
        }
    </style>
</head>

<body>
    <div class="page">
        <div class="fixed">fixed</div>
    </div>
</body>

</html>
```

#### sticky 配置

相対配置と固定配置を組み合わせたもの。ページをスクロールすると、要素はいったん通常どおり動き、一定位置に到達した後で固定される。

```
セレクタ {
    position: sticky;
    top: 10px;
}
```

- ブラウザの可視ウィンドウを基準点にして要素を移動する。固定配置の特徴。

- 元の位置を占有する。相対配置の特徴。

- `top` / `left` / `right` / `bottom` のいずれかを必ず追加しなければ有効にならない。固定される位置を指定するためである。

互換性が弱いためあまり使わない。通常この効果は JavaScript で実装する。

#### position のまとめ

| 配置モード | 標準フローから外れるか | 移動基準 | 使用頻度 |
| --- | --- | --- | --- |
| static | いいえ | オフセット不可 | 少ない |
| relative | いいえ | 自身の位置を基準に移動 | 多い |
| absolute | はい | position 付きの親要素 | 多い |
| fixed | はい | ブラウザ可視領域 | 多い |
| sticky | いいえ | ブラウザ可視領域 | 現段階では少ない |

#### 重なり順 z-index

ボックスが重なる場合、`z-index` は z 軸、つまりレイヤーの前後関係を制御する。

```
セレクタ {
    z-index: 1;
}
```

- 値は正の整数、負の値、または 0 にできる。既定値は `auto`。値が大きいほど上に表示される。

- 属性値が同じ場合は、後に書かれたものが上に表示される。

- `z-index` には単位を付けない。

- position のあるボックスだけが `z-index` 属性を持てる。

#### position の補足

##### 絶対配置ボックスの中央寄せ

```
position: absolute;
/* 水平方向の中央寄せ */
left: 50%;
margin-left: -width の半分
/* 垂直方向の中央寄せ */
top: 50%;
margin-top: -height の半分
```

##### position の特殊な特徴

絶対配置と固定配置は、float と少し似ている。

- インライン要素に絶対配置または固定配置を追加すると、高さと幅を直接設定できる。

- ブロック要素に絶対配置または固定配置を追加し、高さと幅を指定しなければ、既定サイズは内容のサイズになる。

##### 標準フローから外れたボックスはマージン相殺を起こさない

絶対配置と固定配置はマージン相殺を起こさない。

##### position は標準フローの文字を覆う

float 要素は文字や画像を覆わず、ボックスだけを覆う。

## Web ページレイアウトまとめ

- 標準フロー

ボックスを上下、または左右に並べられる。縦方向のブロックボックス表示には標準フローを使う。

- float

複数のブロック要素を一行に表示したり、ボックスを左右に揃えたりできる。複数のブロックボックスを横に表示したい場合は float レイアウトを使う。

- position

最大の特徴は重なりの概念があること。複数のボックスを前後に重ねて表示できる。要素を特定のボックス内で自由に移動させたいなら position レイアウトを使う。

## 要素の表示と非表示

- `display` による表示・非表示

- `visibility` による表示・非表示

- `overflow` によるはみ出し表示・非表示

### display による表示・非表示

- `display: none;` オブジェクトを非表示にし、位置も保持しない。

- `display: block;` 要素を表示する。ブロック要素へ変換する意味もある。

非常によく使う。

### visibility による表示・非表示

- `visibility: hidden;` 非表示にするが、位置は保持する。

- `visibility: visible;` 表示する。

### overflow によるはみ出し表示・非表示

- `overflow: visible;` はみ出した部分を表示する。既定値。

- `overflow: hidden;` はみ出した部分を表示しない。

- `overflow: scroll;` はみ出した部分をスクロールバー付きで表示する。はみ出していなくてもスクロールバーを表示する。

- `overflow: auto;` はみ出していなければスクロールバーを表示せず、はみ出した場合だけ表示する。

position 付きのボックスがある場合は慎重に使う。ボックス外へ配置された余分な部分も隠れてしまうためである。
