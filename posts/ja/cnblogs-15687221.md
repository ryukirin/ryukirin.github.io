### Linux のディレクトリ構造

以下の内容はすべて [Linux 系统目录结构](https://www.runoob.com/linux/linux-system-contents.html?accessToken=eyJhbGciOiJIUzI1NiIsImtpZCI6ImRlZmF1bHQiLCJ0eXAiOiJKV1QifQ.eyJhdWQiOiJhY2Nlc3NfcmVzb3VyY2UiLCJleHAiOjE2Mzk0MTAwNTYsImciOiJ2NVlCR25qSkRBTXJLeWhrIiwiaWF0IjoxNjM5NDA5NzU2LCJ1c2VySWQiOjB9.PuMUd4D9l5edf61NqfyDSbvuN1g3ni-MQ5pFES3VGnE) から。

- **/bin**:
bin は Binaries（二進ファイル）の略で、このディレクトリには最もよく使われるコマンドが保存されている。

- **/boot:**
Linux 起動時に使われるいくつかの中核ファイルが置かれている。リンクファイルやイメージファイルも含まれる。

- **/dev:**
dev は Device（デバイス）の略。このディレクトリには Linux の外部デバイスが保存される。Linux ではデバイスへのアクセス方法はファイルへのアクセス方法と同じである。

- **/etc:**
etc は Etcetera（その他）の略。このディレクトリには、システム管理に必要なすべての設定ファイルとサブディレクトリが置かれる。

- **/home**:
ユーザーのホームディレクトリ。Linux では各ユーザーが自分のディレクトリを持ち、通常はユーザーアカウント名がディレクトリ名になる。上図の alice、bob、eve のようなもの。

- **/lib**:
lib は Library（ライブラリ）の略。このディレクトリには、システムの最も基本的な動的リンク共有ライブラリが保存される。Windows の DLL ファイルに似ている。ほぼすべてのアプリケーションがこれらの共有ライブラリを必要とする。

- **/lost+found**:
通常は空のディレクトリ。システムが不正終了した後、ここにいくつかのファイルが保存される。

- **/media**:
Linux システムは USB メモリや光学ドライブなどのデバイスを自動認識する。認識後、Linux はそのデバイスをこのディレクトリにマウントする。

- **/mnt**:
ユーザーが別のファイルシステムを一時的にマウントするためのディレクトリ。たとえば光学ドライブを `/mnt/` にマウントし、そのディレクトリに入って内容を確認できる。

- **/opt**:
opt は optional（任意）の略。ホストに追加インストールするソフトウェアを置くディレクトリ。たとえば ORACLE データベースをインストールする場合、このディレクトリに置ける。デフォルトでは空。

- **/proc**:
proc は Processes（プロセス）の略。`/proc` は擬似ファイルシステム、つまり仮想ファイルシステムで、現在のカーネル実行状態を表す一連の特殊ファイルを保存している。このディレクトリは仮想ディレクトリであり、システムメモリのマッピングである。直接アクセスすることでシステム情報を取得できる。
このディレクトリの内容はハードディスク上ではなくメモリ上にある。一部のファイルは直接変更できる。たとえば次のコマンドでホストの ping 応答を無効化し、他人が自分のマシンへ ping できないようにできる。

```
echo 1 > /proc/sys/net/ipv4/icmp_echo_ignore_all
```

- **/root**:
システム管理者、つまりスーパーユーザーのホームディレクトリ。

- **/sbin**:
s は Super User の意味で、Superuser Binaries（スーパーユーザー用バイナリ）の略。ここにはシステム管理者が使うシステム管理プログラムが置かれる。

- **/selinux**:
Redhat/CentOS 固有のディレクトリ。Selinux は Windows のファイアウォールに似たセキュリティ機構だが、仕組みはかなり複雑。このディレクトリには selinux 関連ファイルが保存される。

- **/srv**:
サービス起動後に必要となるデータが保存される。

- **/sys**:

これは Linux 2.6 カーネルにおける大きな変更点。このディレクトリには、2.6 カーネルで新しく登場したファイルシステム sysfs がインストールされている。

sysfs ファイルシステムは、プロセス情報用の proc ファイルシステム、デバイス用の devfs ファイルシステム、擬似端末用の devpts ファイルシステムという 3 種類のファイルシステム情報を統合している。

このファイルシステムは、カーネルデバイスツリーを直感的に反映したものである。

カーネルオブジェクトが作成されると、対応するファイルとディレクトリもカーネルオブジェクトサブシステム内に作成される。

- **/tmp**:
tmp は temporary（一時）の略。このディレクトリは一時ファイルを保存するために使われる。

- **/usr**:
usr は unix shared resources（共有リソース）の略。非常に重要なディレクトリで、ユーザーの多くのアプリケーションやファイルがここに置かれる。Windows の `program files` ディレクトリに似ている。

- **/usr/bin:**
システムユーザーが使うアプリケーション。

- **/usr/sbin:**
スーパーユーザーが使う、比較的高度な管理プログラムとシステムデーモン。

- **/usr/src:**
カーネルソースコードのデフォルト配置ディレクトリ。

- **/var**:
var は variable（変数）の略。このディレクトリには、絶えず増えていくものが保存される。通常、頻繁に変更されるディレクトリをここに置く。各種ログファイルも含まれる。

- **/run**:
一時ファイルシステムで、システム起動後の情報を保存する。システム再起動時、このディレクトリ内のファイルは削除またはクリアされるべきである。システムに `/var/run` ディレクトリがある場合は、`run` を指すようにする。

Linux システムには重要なディレクトリがいくつかあり、普段から誤って削除したり内部ファイルをむやみに変更したりしないよう注意が必要である。

**/etc**: 上でも触れたように、システムの設定ファイルがある。ここにある特定のファイルを変更すると、システムが起動できなくなる可能性がある。

**/bin, /sbin, /usr/bin, /usr/sbin**: システムにあらかじめ用意された実行ファイルの配置ディレクトリ。たとえば `ls` は `/bin/ls` にある。

補足すると、`/bin` と `/usr/bin` はシステムユーザー（root 以外の通常ユーザー）向けのコマンドで、`/sbin` と `/usr/sbin` は root 向けのコマンドである。

**/var**: 非常に重要なディレクトリ。システム上では多くのプログラムが動いており、それぞれに対応するログが生成される。これらのログはこのディレクトリに記録され、具体的には `/var/log` に置かれる。また mail のデフォルト配置場所もここである。

### Linux のファイルとディレクトリ管理

- 絶対パス: ルートディレクトリ `/` から書き始める。例: `/usr/share/doc`

- 相対パス: たとえば `/usr/share/doc` から `/usr/share/man` に移動する場合、`cd ../man` と書ける。

#### よく使うコマンド

- `ls`（英語フルスペル: list files）: ディレクトリとファイル名を列挙する。

`-a`: すべてのファイルを、隠しファイル（`.` で始まるファイル）も含めて表示する（よく使う）。

- `-d`: ディレクトリ内のファイルデータではなく、ディレクトリ自身だけを表示する（よく使う）。

- `-l`: ファイル属性や権限などを含む詳細なデータ列で表示する（よく使う）。

`cd`（英語フルスペル: change directory）: ディレクトリを切り替える。

```
#使用 mkdir 命令创建 runoob 目录
[root@www ~]# mkdir runoob

#使用绝对路径切换到 runoob 目录
[root@www ~]# cd /root/runoob/

#使用相对路径切换到 runoob 目录
[root@www ~]# cd ./runoob/

# 表示回到自己的家目录，亦即是 /root 这个目录
[root@www runoob]# cd ~

# 表示去到目前的上一级目录，亦即是 /root 的上一级目录的意思；
[root@www ~]# cd ..
```

`pwd`（英語フルスペル: print work directory）: 現在のディレクトリを表示する。

- **-P**: リンクパスではなく、実際のパスを表示する。

`mkdir`（英語フルスペル: make directory）: 新しいディレクトリを作成する。

- `-m`: ファイル権限を直接設定する。デフォルト権限（umask）に依存しない。

- `-p`: 必要なディレクトリ（上位ディレクトリを含む）を再帰的に作成する。

`rmdir`（英語フルスペル: remove directory）: 空のディレクトリを削除する。

- **-p:** 指定ディレクトリから始めて、複数階層の空ディレクトリをまとめて削除する。

`cp`（英語フルスペル: copy file）: ファイルまたはディレクトリをコピーする。

- **-a:** `-pdr` 相当。`pdr` は以下の説明を参照（よく使う）。
- **-d:** コピー元がリンクファイル属性の場合、ファイル本体ではなくリンクファイル属性をコピーする。
- **-f:** force の意味。対象ファイルがすでに存在し開けない場合、削除してから再試行する。
- **-i:** 対象ファイルがすでに存在する場合、上書き前に操作確認を行う（よく使う）。
- **-l:** ファイル本体をコピーするのではなく、ハードリンクを作成する。
- **-p:** デフォルト属性ではなく、ファイル属性も含めてコピーする（バックアップでよく使う）。
- **-r:** 再帰的にコピーする。ディレクトリコピーでよく使う。
- **-s:** シンボリックリンク、つまり「ショートカット」ファイルとしてコピーする。
- **-u:** destination が source より古い場合だけ destination を更新する。

`rm`（英語フルスペル: remove）: ファイルまたはディレクトリを削除する。

- `-f`: force の意味。存在しないファイルを無視し、警告を出さない。
- `-i`: 対話モード。削除前にユーザーへ確認する。
- `-r`: 再帰削除。ディレクトリ削除でよく使うが、非常に危険なオプション。

`mv`（英語フルスペル: move file）: ファイルやディレクトリを移動する、または名前を変更する。

- `-f`: force の意味。対象ファイルが存在しても確認せず直接上書きする。
- `-i`: 対象ファイルが存在する場合、上書きするか確認する。
- `-u`: 対象ファイルが存在し、source の方が新しい場合だけ更新する。

*man [コマンド]* で各コマンドの使用ドキュメントを確認できる。
