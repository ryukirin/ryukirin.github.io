# 吃瓜教程 | Datawhale 9月（1）

## 緒論

### はじめに

条件 → 結果。

機械学習研究の主な内容は、コンピュータ上でデータからモデルを生成するアルゴリズム、つまり学習アルゴリズムである。

### 基本用語

*自分が分かっていなかったものだけ説明する。*

- データセット data set

- 事例 instance / サンプル sample

- 属性 attribute / 特徴 feature

- 属性値 attribute value

- 属性空間 attribute space / サンプル空間 sample space / 入力空間

- 特徴ベクトル feature vector: あるサンプルが、属性によって形成される多次元空間内で属する点の座標ベクトル。

- 次元 dimensionality: データセット内の属性数。

- 訓練データ training data

- 訓練サンプル training set

- 仮説 hypothesis: データに潜む何らかの規則。

- 真相 / ground-truth: その潜在規則そのもの。

- 学習器 learner: モデル。

- ラベル / 標記 label: 訓練サンプルの結果情報。

- ラベル空間 / 出力空間 label space: すべてのラベルの集合。Γ で表す。

- サンプル例 example: label を持つ instance。一般に `(xi,yi)` で第 `i` 個の例を表す。ここで `yi∈Γ` は事例 `xi` のラベル。

- 汎化 generalization: 学習して得たモデルが新しいサンプルに適用できる能力。

- 分布 distribution: サンプル空間内の全サンプルが従うと仮定される未知の規則。`D`。

- 独立同分布 independent and identically distributed、i.i.d: 得られる各サンプルが、それぞれ独立に `D` からサンプリングされること。

### 仮説空間

#### バージョン空間

すべての可能な仮説が仮説空間である。そこから、訓練集合上の正例と一致しない仮説、反例と一致する仮説を削除した残りがバージョン空間を構成する。

#### 帰納 induction

特殊 → 一般。つまり汎化。

#### 演繹 deduction

一般 → 特殊。つまり特化。

### 帰納バイアス

機械学習アルゴリズムが学習過程で、ある種類の仮説を好む傾向。

原則: Occam の剃刀。最も単純なものを選ぶ。

#### 1-1

```
# 表 1.1 に番号 1 と 4 の二つの事例だけが含まれる場合、対応するバージョン空間を与える。
# バージョン空間とは、仮説空間の中から正例と一致し、反例と反対になるものを見つけること。
# 考え方: 仮説空間を書く -> 正例と反例を明確にする -> 仮説空間から正例と一致し反例と反対のものを探す。

import numpy
if __name__ == '__main__':
    # 16 進数でスイカの種類を表す。属性は三つで、各属性には三つの可能な状態があるため、3*3*3 通り。
    # 2 進数で表すなら一つの属性を 00、01、10 で表すが、2 進数は長いので 16 進数に変換する。
    # ここでは 乌黑:10、青绿:11；稍蜷:10、蜷缩:11；沉闷:10、浊响:11。
    # 下のリストが仮説空間。
    melon = (0x3f, 0x3d, 0x3e, 0x37, 0x3b, 0x1f, 0x2f, 0x35, 0x36, 0x39, 0x3a, 0x1d, 0x1e, 0x2d,
             0x2e, 0x17, 0x1b, 0x27, 0x2b, 0x15, 0x16, 0x19, 0x1a, 0x25, 0x26, 0x29, 0x2a)
    # 二つの事例は、それぞれ青绿蜷缩浊响と乌黑稍蜷沉闷。
    # 前者が正例、後者が反例。
    sample = (0x15,0x2a)
    # sum はカウント用。
    sum = 0
    for i in range(0,27):
        # バージョン空間: 仮説空間から、正例と一致しない仮説、または反例と一致する仮説を削除する。
        # つまり、仮説空間の中で反例と一致せず、かつ正例と一致する仮説。
        if( (melon[i] | sample[1] ) != melon[i] and (melon[i] | sample[0]) == melon[i] ):
            sum = sum+1
            print(bin(melon[i]),i+1)
    print(sum)
```

#### 1-2

```
# 単一の連言式で仮説を表す場合と比べて、「選言標準形」を使うと仮説空間の表現力が高くなる。
# 最大 k 個の連言式を含む選言標準形で 1.1 のスイカ分類問題の仮説空間を表す場合、可能な仮説数を推定する。
# code:https://blog.csdn.net/qq_40273675/article/details/89856447
# このコードでは 3*4*4 個の基本仮説を使う考え方を採用している。別案は 2*3*3。
# (A=a)∨(A=*) は (A=*) と等価なので、全仮説空間を求めた後でこの形式、つまり冗長なものを削除する。
# 考え方: 組み合わせ C(48,k) -> * があるか確認 -> ある -> * 以外の仮説が他に包含されているか確認。例 (AAB)∨(AA*) -> はい -> 削除
#                                      ┕> ない -> 冗長ではない                               ┕> ない -> 冗長ではない

import re
import itertools as it

def get_combine_hyps(list_attr, k):
    # 組み合わせ
    # Without Empty set
    list_hyps = []
    # 三つの特徴がある
    for value_attr0 in list_attr[0]:
        for value_attr1 in list_attr[1]:
            for value_attr2 in list_attr[2]:
                x = value_attr0 + value_attr1 + value_attr2
                list_hyps.append(x)
    list_com = list(it.combinations(list_hyps, k))  # 組み合わせ C48,k
    return list_com

def delete_repeated(list_in):
    # 冗長なものを削除する
    list_outer = []
    list_general = []
    flag_repeating = 0  # 1 のとき冗長ではなく、0 のとき冗長

    for com_hyp in list_in:
        """各組の選言標準形から汎化仮説を見つける"""
        for a_hyp in com_hyp:
            # * があるか検査する
            if "." in a_hyp:
                # 存在する場合は次に進み、* を含む連言式を list_general に入れる
                list_general.append(a_hyp)
            else:
                pass
        """汎化仮説が他の仮説を包含しているかを判断する"""
        if len(list_general) == 0:
            flag_repeating = 1
        else:
            for b_hyp in com_hyp:
                for general in list_general:
                    # まず同一のものを除外する
                    if general == b_hyp:
                        pass
                    else:
                        # re.search は文字列全体を走査し、最初に成功したマッチを返す。マッチしなければ None を返す。
                        # None が返る場合は冗長ではないことを示す。
                        hyp_re = re.search(general, b_hyp)
                        if hyp_re is None:
                            flag_repeating = 1
                        else:
                            pass
                # list_general を初期化する。そうしないと、複数ループで * を含む項が大量に追加される。
                list_general = []
        if flag_repeating == 1:
            list_outer.append(com_hyp)
            flag_repeating = 0

    return list_outer

def main(k):
    list_attribute = [["A", "B", "."], ["A", "B", "C", "."], ["A", "B", "C", "."]]
    hypothesis = get_combine_hyps(list_attribute, k)
    if k < 1 or k > 48:
        print("k 値は 48 未満の正整数でなければならない!")
    elif k == 1:
        print(len(hypothesis), hypothesis)
    else:
        hyps = delete_repeated(hypothesis)
        print(len(hyps), hyps)

if __name__ == "__main__":
    main(2)
```

## モデル評価と選択

### 経験誤差と過学習

誤り率: サンプル数 `m` のうち `a` 個が誤分類サンプルなら、誤り率 `E = a/m`、精度は `1-a/m` である。

誤差: 予測出力と真の出力の差。

経験誤差: 学習器の訓練集合上での誤差。訓練誤差とも呼ぶ。

過学習: 学習器が訓練データ自身の特徴を、すべての潜在サンプルに共通する一般的特徴だとみなしてしまい、汎化性能が低くなること。

### 評価方法

テスト集合。

- ホールドアウト法

データセットを二つの互いに素な集合へ直接分割する。正例・反例の割合は一致させる。

- K 分割交差検証

- Leave-One-Out、略して LOO: 毎回一つだけをテスト集合として使い、残りすべてを訓練集合として使う。

- ブートストラップ法

一つのサンプルが `m` 回のサンプリングで一度も選ばれない確率は `(1−1/m)^m` であり、極限を取ると次のようになる。

\\[\\lim_{m \\to \\infty} (1-\\frac{1}{m} )^{m} =\\frac{1}{e}≈0.368
\\]

### 性能指標

平均二乗誤差、最もよく使われる。

#### 誤り率と精度

- 誤り率: 誤分類の比率。

- 精度: 1 - 誤り率。

#### 適合率、再現率、F1

- 混同行列: 真陽性（TP）、偽陽性（FP）、真陰性（TN）、偽陰性（FN）からなる分類結果表。

- 適合率 P: 正例と予測されたもののうち、本当に正例である確率。

- 再現率 R: 本当の正例のうち、正例と予測された確率。

一般に、この二つはトレードオフの関係にある。一方が高い場合、もう一方は低くなりやすい。

- 適合率-再現率曲線: 略して P-R 曲線。

二つのモデルを比較するときは、平衡点を比較することも考えられる。

- 平衡点: 適合率 = 再現率となる点。

\\[F1=\\frac{2\\times P\\times R}{P+R}=\\frac{2\\times TP}{样例总数+TP-TN}
\\]

F1 は適合率と再現率の調和平均に基づいて定義される。

\\[\\frac{1}{F1}=\\frac{1}{2}\\cdot (\\frac{1}{P}+\\frac{1}{R})
\\]

\\(F_\\beta\\) は重み付き調和平均である。

\\[\\frac{1}{F_\\beta }=\\frac{1}{1+\\beta ^2}\\cdot (\\frac{1}{P}+\\frac{\\beta ^2}{R})
\\]

\\[F_\\beta=\\frac{(1+\\beta ^2)\\times P\\times R}{(\\beta ^2\\times P)+R}
\\]

ここで `β>0` は、再現率が適合率に対してどれだけ重要かを表す。`β>1` なら再現率の影響が大きく、`β<1` なら適合率の影響が大きい。`β=1` のとき標準的な F1 になる。

複数の混同行列、つまり複数回の訓練や複数データセットから得られた結果で適合率と再現率を考える場合、macro-P、macro-R、macro-F1 を使える。

\\[macro-P=\\frac{1}{n}\\sum_{n}^{i=1}P_i
\\]

\\[macro-R=\\frac{1}{n}\\sum_{n}^{i=1}R_i
\\]

\\[macro-F1=\\frac{2\\times macro-P\\times macro-R}{macro-P+macro-R}
\\]

### 練習問題

#### 2-1

データセットには 1000 個のサンプルがあり、そのうち 500 個が正例、500 個が反例である。70% のサンプルを含む訓練集合と、30% のサンプルを含むテスト集合に分けてホールドアウト法で評価する場合、分割方法はいくつあるかを推定する。

分布の一貫性を保つ必要があるため、500 * 70% 個の正例サンプルと反例サンプルを訓練集合へ分け、500 * 30% 個の正例サンプルと反例サンプルをテスト集合へ分ける。したがって分割方法は \\(C_{500}^{350}\\times C_{500}^{350}=C_{500}^{150}\\times C_{500}^{150}\\) 通り。

#### 2-2

データセットには 100 個のサンプルがあり、正例と反例が半分ずつである。学習アルゴリズムが生成するモデルは、新しいサンプルを訓練サンプル数が多いカテゴリへ予測するものとする。訓練サンプル数が同じ場合はランダムに推測する。このとき、10 分割交差検証法と Leave-One-Out 法でそれぞれ誤り率を評価した結果を示す。

10 分割交差検証法では、正例と反例が半分ずつなので誤り率は 50%。

Leave-One-Out 法では、正例と反例が半分ずつであるため、どちらを一つ取り除いても、残りの同類は必ず少数派になる。したがって予測は必ず間違い、誤り率は 1 になる。

#### 2-3

学習器 A の F1 値が学習器 B より高い場合、A の BEP 値も B より高いと言えるか。

P-R 図では、各点に F1 値を持たせられる。しかし BEP は `P=R` となる一点だけである。したがって両者にはそれほど直接的な関係はない。
