このモジュールは、特定の目的を持つコンテナを実装しており、Python 標準の組み込みコンテナである `dict`、`list`、`set`、`tuple` の代替として利用できる。

- ChainMap: `dict` に似たコンテナクラス。複数のマッピングを 1 つのビューにまとめる。

- Counter: `dict` のサブクラス。ハッシュ可能なオブジェクトを数える機能を提供する。

- deque: `list` に似たコンテナ。両端で高速に `append` と `pop` ができる。

- defaultdict: `dict` のサブクラス。辞書検索時にデフォルト値を提供するファクトリ関数を持つ。

- namedtuple(): 名前付きタプルのサブクラスを作成するファクトリ関数。

- OrderedDict: `dict` のサブクラス。要素が追加された順序を保持する。

- UserDict: 辞書オブジェクトをラップし、辞書のサブクラス化を簡単にする。

- UserList: リストオブジェクトをラップし、リストのサブクラス化を簡単にする。

- UserString: 文字列オブジェクトをラップし、文字列のサブクラス化を簡単にする。

## ChainMap

`ChainMap` クラスは、複数のマッピングを素早く連結し、それらを 1 つの単位として扱うためのもの。

通常、新しい辞書を作って何度も `update()` を呼ぶよりかなり速い。

```
collections.ChainMap(*maps)
```

`maps` が指定されていない場合は、デフォルトで空の辞書が 1 つ提供される。つまり、新しいチェーンには少なくとも 1 つのマッピングが含まれる。

```
from collections import ChainMap

d1 = {'apple':1,'banana':2}
d2 = {'orange':2,'apple':3,'pike':1}

combined_d = ChainMap(d1,d2)
reverse_combind_d = ChainMap(d2,d1)

print(combined_d) 
print(reverse_combind_d)

'''
逐项输出时，从第二个dict的第一项开始输出;
当输出到两个dict都有的key，输出且只输出第一个dict的value
'''
for k,v in combined_d.items():
    print(k,v)
print('------------------')    
for k,v in reverse_combind_d.items():
    print(k,v)
```

## Counter

`Counter` は `dict` のサブクラスで、主にアクセスしたオブジェクトの出現頻度を数えるために使う。高速で便利なカウンターツールである。

よく使うメソッド:

- elements(): イテレータを返す。各要素はカウント数だけ繰り返され、カウントが 1 未満の要素は無視される。

- most_common([n]): アクセス頻度が最も高い n 個の要素とカウントをリストで返す。

- subtract([iterable-or-mapping]): イテラブルから要素を差し引く。入力・出力は 0 または負数になってもよい。

- update([iterable-or-mapping]): イテラブルから要素を数える、または別のマッピングオブジェクト（またはカウンター）から追加する。

```
from collections import Counter

# 统计字符出现的次数
print(Counter('hello world'))

# 统计单词数
c = Counter('hello world hello world hello nihao'.split())
print(c)

cnt = Counter()
for word in ['red', 'blue', 'red', 'green', 'blue', 'blue']:
    cnt[word] += 1
print(cnt)

# 获取指定对象的访问次数，也可以使用get()方法
print(c['hello'])
print(c.get('hello'))

# 查看元素
print(list(c.elements()))
print(c.elements())

# 追加对象，或者使用c.update(d)
c = Counter('hello world hello world hello nihao'.split())
d = Counter('hello world'.split())
print(c)
print(d)
print(c + d)
c.update(d)
print(c)

#减少对象，或者使用c.subtract(d)
c = Counter('hello world hello world hello nihao'.split())
print(c - d)
c.subtract(d)
print(c)

# 清除
c.clear()
print(c)
```

## deque

`collections.deque` は新しい双方向キューオブジェクトを返す。`iterable` からデータを作成し、左から右へ初期化する（方法は `append()`）。`iterable` が指定されていない場合、新しいキューは空になる。

`deque` キューはスタックまたは queue から生成される。`deque` はスレッドセーフで、両端からの `append` または `pop` は計算量 O(1) である。

`list` オブジェクトも似た操作をサポートしているが、ここでは固定長操作（`pop(0)`、`insert(0,v)`）のコストが最適化されている。

`maxlen` が指定されていない、または `None` の場合、deque は任意の長さまで伸びる。そうでなければ、deque は指定された最大長に制限される。

長さ制限のある deque が満杯になった状態で新しい項目が追加されると、同じ数の項目が反対側から取り除かれる。

サポートされるメソッド:

- append(x): x を右端に追加する。

- appendleft(x): x を左端に追加する。

- clear(): すべての要素を削除し、長さを 0 にする。

- copy(): 浅いコピーを作成する。

- count(x): キュー内で x と等しい要素の数を数える。

- extend(iterable): キューの右側に iterable 内の要素を追加する。

- extendleft(iterable): キューの左側に iterable 内の要素を追加する。注意: 左側に追加するとき、iterable 引数の順序は反転して追加される。

- index(x[,start[,stop]]): deque 内で x がある位置を返す。start 以降、stop より前で最初に一致したものを返す。見つからない場合は ValueError を送出する。

- insert(i,x): 位置 i に x を挿入する。注意: 挿入により長さ制限付き deque が `maxlen` を超える場合、IndexError が発生する。

- pop(): 最右端の要素を取り除く。

- popleft(): 最左端の要素を取り除く。

- remove(value): 最初に見つかった value を取り除く。見つからない場合は ValueError を送出する。

- reverse(): deque を逆順に並べる。戻り値は None。

- maxlen: キューの最大長。制限がなければ None。

```
from collections import deque
d = deque(maxlen=20)
print(d)
d.extend('python')
print(d)

for elem in d:
    print(elem.upper())
```

## defaultdict

新しい辞書風オブジェクトを返す。

`defaultdict` は組み込み `dict` クラスのサブクラスである。1 つのメソッドをオーバーライドし、書き込み可能なインスタンス変数を 1 つ追加している。

その他の機能は `dict` クラスと同じなので、ここでは繰り返さない。

このオブジェクトには `default_factory` という属性がある。構築時の第 1 引数がこの属性の初期値として使われ、デフォルトは None である。

その他のすべての引数（キーワード引数を含む）は、`dict` のコンストラクタに渡されるのと同じである。

```
from collections import defaultdict
d = defaultdict()
print(d)

#使用list做default_factory
s = [('yellow', 1), ('blue', 2), ('yellow', 3), ('blue', 4), ('red', 1)]
d = defaultdict(list)

#直接添加，如果要加value在[]后加.append(value)即可
d['hello']
print(d)
print(sorted(d.items()))

for k, v in s:
    d[k].append(v)

print(sorted(d.items()))
```

```
defaultdict(None, {})
defaultdict(<class 'list'>, {'hello': []})
[('hello', [])]
[('blue', [2, 4]), ('hello', []), ('red', [1]), ('yellow', [1, 3])]
```

```
#设置 default_factory 为 int，使 defaultdict 用于计数
s = 'mississippi'
d = defaultdict(int)
for k in s:
    d[k] += 1

print(sorted(d.items()))
```

```
[('i', 4), ('m', 1), ('p', 2), ('s', 4)]
```

## namedtuple()

名前付きタプルは、それぞれの位置に意味を与え、可読性と自己文書化性を提供する。

通常のタプルとしてどこでも使えるうえ、名前で値を取得する機能が追加されている。インデックス値でも取得できる。

（例を見ると、少し Java のエンティティクラスのような感じがする。）

```
from collections import namedtuple
Person = namedtuple('Person', ['age', 'height', 'name'])
Human = namedtuple('Human', 'age, height, name')
student = namedtuple('student', 'age height name')

tom = Person(30,178,'Tom')
jack = Human(20,179,'Jack')

print(jack)
print(tom)

print(tom.age)

print(jack.height)
```

```
Human(age=20, height=179, name='Jack')
Person(age=30, height=178, name='Tom')
30
179
```

## OrderedDict

順序付き辞書は通常の辞書に似ているが、並び順操作に関する追加機能を持つ。

組み込みの `dict` クラスが挿入順序を記憶できるようになったため（Python 3.7 ではこの新しい挙動が保証されている）、以前ほど重要ではなくなった。

それでも `dict` との違いはいくつか残っている。

- 通常の `dict` は、マッピング操作に非常に優れるよう設計されている。挿入順序の追跡は副次的な機能である。

- `OrderedDict` は、並べ替え操作に優れるよう設計されている。空間効率、イテレーション速度、更新操作の性能は副次的である。

- アルゴリズム上、`OrderedDict` は頻繁な並べ替え操作を `dict` よりうまく処理できる。これにより、最近のアクセスを追跡する用途（たとえば LRU cache）に適している。

- `OrderedDict` では、等価性の判定で順序も確認される。

- `OrderedDict` クラスの `popitem()` メソッドは異なるシグネチャを持つ。どの要素を取り出すかを指定する任意引数を受け取る。

- `OrderedDict` クラスには `move_to_end()` メソッドがあり、要素をどちらかの端へ効率的に移動できる。

- Python 3.8 より前の `dict` には `reversed()` メソッドがなかった。

```
from collections import OrderedDict

od = OrderedDict()
od['country'] = '1'
od['sex'] = '0'
od['age'] = '34'

print(od)
```

```
OrderedDict([('country', '1'), ('sex', '0'), ('age', '34')])
```

```
od.move_to_end('country')
print(od)
```

```
OrderedDict([('sex', '0'), ('age', '34'), ('country', '1')])
```

## UserDict

辞書オブジェクトのラッパーとして使われる。このクラスへの需要は、`dict` のサブクラスを直接作成できる機能によって一部置き換えられている。

ただし、このクラスの方が扱いやすい。基礎となる辞書に属性としてアクセスできるためである。

## UserList

リストオブジェクトをラップする。これは有用な基底クラスであり、カスタムしたいリスト風クラスに対して、継承して既存メソッドをオーバーライドしたり、新しいメソッドを追加したりできる。

これにより、リストに新しい振る舞いを追加できる。

このクラスへの需要は、`list` のサブクラスを直接作成できる機能によって一部置き換えられている。

ただし、このクラスの方が扱いやすい。基礎となるリストに属性としてアクセスできるためである。

## UserString

このクラスは、文字列オブジェクトのラッパーとして使われる。このクラスへの需要は、`str` のサブクラスを直接作成できる機能によって一部置き換えられている。

ただし、このクラスの方が扱いやすい。基礎となる文字列に属性としてアクセスできるためである。
