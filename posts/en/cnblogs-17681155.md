# HTML Notes

## Web Page Concepts

HTML stands for Hyper Text Markup Language. It is a language used to describe web pages.

HTML is not a programming language. It is a markup language.

Hypertext means:

- Going beyond plain text: images, sound, animation, multimedia, and other content can be added.

- Hyperlink text: one file can jump to another file and connect to files hosted around the world.

## Common Browsers and Their Engines

Browser engine, or rendering engine: responsible for reading web page information, organizing it, calculating how the page should be displayed, and rendering the page.

| Firefox | Gecko |  |

Chinese domestic browsers generally use one of the latter two engines.

## Web Standards

Web standards are a collection of standards developed by the W3C and other standardization organizations.

W3C, the World Wide Web Consortium, is one of the most well-known international standardization organizations.

Web standards mainly include three aspects: Structure, Presentation, and Behavior.

| Standard | Description |
| --- | --- |
| Structure | Used to organize and classify page elements. At this stage, this mainly means HTML. |
| Presentation | Used to set the layout, color, size, and other visual styles of page elements. This mainly means CSS. |
| Behavior | Refers to the definition of page models and interaction logic. At this stage, this mainly means JavaScript. |

## HTML Tags

- HTML tags are keywords surrounded by angle brackets.

- HTML tags usually appear in pairs. These are called paired tags. The first tag is the start tag, and the second is the end tag.

- Some special tags must be single tags, such as `<br />`.

Tag relationships:

- Containment relationship, or parent-child relationship.

- Parallel relationship, or sibling relationship.

### Basic HTML Structure Tags

Every web page has a basic structure tag, also called a skeleton tag.

| Tag name | Definition | Description |

### DOCTYPE, lang, and Character Set

```
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>First page created with VS Code</title>
</head>
<body>
    hello world again.
</body>
</html>
```

#### DOCTYPE

The `<!DOCTYPE>` document type declaration tells the browser which HTML version to use to display the page.

- It must be placed at the very beginning of the document, before the `html` tag.

- It is not an HTML tag. It is a document type declaration.

#### lang

Defines the display language of the current document.

- `en`: English.

- `zh-CN`: Chinese.

This does not restrict the displayed language. It only provides a hint to the browser, for example for automatic translation.

#### Character Set

A character set is a collection of characters that allows computers to recognize and store different kinds of text.

Inside a tag, the `charset` attribute can specify which character encoding the HTML document should use.

```
<meta charset="UTF-8">
```

Common values include GB2312, BIG5, GBK, and UTF-8. UTF-8 is also known as Unicode.

### Tag Semantics

Semantics means the meaning of a tag.

Choosing the most appropriate tag for the right place according to its semantics makes the page structure clearer.

### Paragraph and Line Break Tags

`<p>` can divide a web page into paragraphs.

- Text in a paragraph automatically wraps according to the browser size.

- There is spacing between paragraphs.

`<br />` forces a line break. It only starts a new line and does not add the spacing between paragraphs.

### div and span Tags

These two tags have no semantic meaning. They are containers used to hold content.

Characteristics:

- `div` is used for layout, and one `div` occupies one line.

- `span` is used for layout, and multiple `span` elements can appear on the same line.

### Image Tag

The image tag inserts an image and is a single tag.

`src` is the required attribute of an image tag and specifies the file path and file name.

Other attributes:

| Attribute | Value | Description |
| --- | --- | --- |
| src | Image path | Required attribute |
| alt | Text | Alternative text shown when the image cannot be displayed |
| title | Text | Tooltip text shown when the mouse hovers over the image |
| width | Pixels | Sets the image width |
| height | Pixels | Sets the image height |
| border | Pixels | Sets the image border thickness |

- Image tags can have multiple attributes, and they must be written after the tag name.

- Attribute order does not matter. The tag name and attributes, and attributes themselves, are separated by spaces.

- Attributes use key-value pairs.

#### Paths

- Directory folder: a normal folder used to store assets. For example, a `web` folder can contain HTML files and an `img` folder.

- Root directory: the first level opened after entering a directory folder. For example, after opening the `web` folder, the directory you see is the root directory.

When using an `image` folder to store images, we need paths to specify image file locations. Paths can be divided into:

- Relative path: a directory path established relative to the location of the referenced file. In short, it describes where the image is relative to the page using it.

Using `web/pages/index.html` as the page path:

Relative path category
Symbol
Description

Same-level path

The image is next to the page: `web/pages/img.jpg`, so the relative path is `img.jpg`.

Next-level path
/
The image is in a folder next to the page: `web/pages/img/img.jpg`, so the relative path is `img/img.jpg`.

Parent-level path
../
The image is next to the folder containing the page: `web/img.jpg`, so the relative path is `../img.jpg`.

- Absolute path: a full address, usually starting from a drive letter. You can right-click an image, open Properties, and combine the location path with the file name.

### Hyperlink Tag

`<a>` is used to jump to another page.

Link syntax:

```
<a href="target URL" target="how the target window opens">text or image</a>
```

| Attribute | Purpose |
| --- | --- |
| href | Specifies the URL of the link target. Required. |
| target | Specifies how to open the linked page. `_self` is the default; `_blank` opens it in a new window. |

Link categories:

- External links.

- Internal links.

- Empty links.

- Download links: the `href` points to a `.exe` or `.zip` file.

- Web element links: add hyperlinks to web elements.

- Anchor links.

#### Anchor Links

Clicking the link can quickly locate a specific position on the page.

- In the `href` attribute of the link text, set the value in the form `#name`, for example:

```
<a href="#two">Episode 2</a>
```

- Find the target tag and add an `id` attribute equal to the name above, for example:

```
<h3 id="two">Episode 2 introduction</h3>
```

### Table Tags

Tables are used to display data.

```
<table>
    <tr><!-- Row, must be inside the table tag. -->
        <td>Text inside the cell</td><!-- Cell, must be inside tr. -->
        ...
    </tr>
    ...
</table>
```

Table header:

```
<table>
    <tr><!-- Row, must be inside the table tag. -->
        <th>Header</th><!-- Displayed bold and centered. -->
        <td>Text inside the cell</td><!-- Cell, must be inside tr. -->
        ...
    </tr>
    ...
</table>
```

Table attributes are not commonly used. CSS is usually used for styling. However, these words will be used later in CSS and help intuitively understand the visual shape of tables.

| Attribute | Value | Description |
| --- | --- | --- |
| align | left, center, right | Specifies how the table is aligned relative to surrounding elements |
| border | 1 or "" | Specifies whether table cells have borders. Default is "", meaning no border |
| cellpadding | Pixel value | Specifies the spacing between the cell edge and its content. Default is 1 pixel |
| cellspacing | Pixel value | Specifies spacing between cells. Default is 2 pixels |
| width | Pixel value or percentage | Specifies table width |

#### Table Structure Tags

Because tables can be long, they can be semantically divided into a table head and a table body.

`<thead>` represents the header area of a table. It is not the same as `th`; `th` is a header cell, while `thead` is a region.

`<tbody>` represents the body area of a table.

#### Merging Cells

- Merge across rows: `rowspan="number of cells to merge"`.

- Merge across columns: `colspan="number of cells to merge"`.

Target cell, where the merge code is written:

- Across rows: the topmost cell is the target cell.

- Across columns: the leftmost cell is the target cell.

Steps:

- Determine whether to merge rows or columns.

- Find the target cell and write the merge code.

- Delete the extra cells.

### List Tags

Tables are used to display data, while lists are used for layout.

- Unordered lists.

- Ordered lists.

- Description lists.

#### Unordered List

```
<ul><!-- ul can only nest li. Adding other text or tags directly is not allowed. -->
    <li>List item 1</li>
    <li>List item 2</li>
    <li>List item 3</li>
    ...
</ul>
```

#### Ordered List

```
<ol><!-- ol can only nest li. Adding other text or tags directly is not allowed. -->
    <li>List item 1</li>
    <li>List item 2</li>
    <li>List item 3</li>
    ...
</ol>
```

#### Description List

Description lists are often used to explain and describe terms or nouns.

```
<dl><!-- dl can only nest dt and dd. Adding other text or tags directly is not allowed. -->
    <dt>Term 1</dt><!-- There is no limit on the number of dt or dd elements. -->
    <dd>Explanation 1 for term 1</dd>
    <dd>Explanation 2 for term 1</dd>
</dl>
```

### Form Tags

A complete form usually consists of three parts: a form area, form controls, and prompt information.

#### Form Area

The form area contains form elements. The `<form>` tag defines the form area and submits the information from form elements inside it to the server.

```
<form action="url" method="method" name="form area name">
   Various form controls 
</form>
```

| Attribute | Value | Purpose |
| --- | --- | --- |
| action | URL | Specifies the URL of the server program that receives and processes the form data |
| method | get/post | Sets the submission method of form data. Values are get or post |
| name | Name | Specifies the form name so multiple form areas on the same page can be distinguished |

#### Form Controls

- `input` form element.

- `select` dropdown form element.

- `textarea` text area element.

##### input Form Element

```
<input type="attribute value" />
```

| Attribute value | Description |
| --- | --- |
| button | Clickable button, usually starts a script through JavaScript |
| checkbox | Checkbox |
| file | Input field and browse button for file upload |
| hidden | Hidden input field |
| image | Submit button in image form |
| password | Password field |
| radio | Radio button |
| reset | Reset button, clears all data in the form |
| submit | Submit button, sends form data to the server |
| text | Single-line input field where text can be entered. Default width is 20 characters |

Besides `type`, `input` has other attributes:

| Attribute | Value | Description |
| --- | --- | --- |
| name | Custom | Name of the input |
| value | Custom | Specifies the value of the input element |
| checked | checked | Specifies that this input element should be selected on initial load |
| maxlength | Positive integer | Specifies the maximum character length of the input field |

`name` and `value` are attributes that every form element has, used by backend developers.

Radio buttons and checkboxes that belong to the same group should have the same `name`.

##### label Tag

The label tag is used to bind a form element. When the user clicks the text inside the `<label>` tag, the browser automatically focuses or selects the corresponding form element, improving user experience.

```
<label for="sex">Male</label>
<!-- The value after for is the id content. -->
<input type="radio" name="sex" id="sex" />
```

##### select Tag

Use this when there are multiple options and you want to save page space.

```
<select>
    <option>Option 1</option>
    <option>Option 2</option>
    <option>Option 3</option>
    ...
</select>
```

You can write `selected` in an attribute to indicate the default selected option.

##### textarea Text Area Tag

Use `textarea` when the user needs to enter a larger amount of text.

```
<textarea rows="3" cols="20">
	Text content
</textarea>
```

In real development, `cols` and `rows` are controlled through CSS.
