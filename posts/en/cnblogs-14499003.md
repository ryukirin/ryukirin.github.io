## __init__

The first parameter of the `__init__` method is always `self`, which represents the class instance being created.

Once a class defines `__init__`, you cannot create an instance with empty arguments unless the method allows it. You must pass parameters that match `__init__`. However, `self` does not need to be passed manually; the Python interpreter passes the instance object automatically.

```
class Student(object):
    def __init__(self, name, score):
        self.name = name
        self.score = score
student = Student("Hugh", 99)
print(student.name)
print(student.score)
```

```
Hugh
99
```

## Instance methods

Data encapsulation.

`name` and `score` are internal variables of `student`. When using them, we can wrap related logic in methods.

```
class Student(object):
    def __init__(self, name, score):
        self.name = name
        self.score = score
    def Printinfo(self):
        print(self.name + ':' + str(self.score))

ryu = Student('ryu',98)
ryu.Printinfo()
```

```
ryu:98
```

At this point, however, external code can still modify `ryu.name` and `ryu.score`.

```
ryu.name = 'tyo'
ryu.Printinfo()
```

```
tyo:98
```

Setting `name` and `score` as private variables avoids this issue. In that case, the value cannot be obtained through `object.variable`. If needed, getter and setter methods can be added.

```
class Student(object):
    def __init__(self, name, score):
        self.__name = name # private variable
        self.__score = score # private variable
        
    def printInfo(self):
        print(self.__name + ':' + str(self.__score))
        
    def getName(self):
        print(self.__name)
        
    def getScore(self):
        print(self.__score)
        
    def setName(self, name):
        self.__name = name
        
    def setScore(self, score):
        self.__score = score
        
ryu = Student('ryu',98)
ryu.printInfo()

# print(ryu.__name) would raise an error.

ryu.name = 'tyo'
ryu.printInfo()
```

```
ryu:98
ryu:98
```

```
ryu.getName()
ryu.getScore()
ryu.setName('tyo')
ryu.setScore(100)
ryu.printInfo()
```

```
tyo
100
tyo:100
```

## Inheritance

With inheritance, `self` is the instance that was passed in, not necessarily an instance of the class where `self` is written.

```
class Parent:
    def pprt(self):
        print(self)

class Child(Parent):
    def cprt(self):
        print(self)

c = Child()
c.cprt()
# This is equivalent to Child.pprt(c), so self is still an instance of Child.
# Since pprt() is not defined directly on self, Python follows the inheritance tree upward,
# finds pprt() on Parent, and calls it successfully.
c.pprt()
p = Parent()
p.pprt()
```

```
<__main__.Child object at 0x000001EF6BBB2188>
<__main__.Child object at 0x000001EF6BBB2188>
<__main__.Parent object at 0x000001EF6BBB2F08>
```

## Descriptor classes

In a descriptor class, `self` refers to the descriptor instance.

```
class Desc:
    
    # ins represents the class instance that calls it. If no instance calls it, ins is None.
    def __get__(self, ins, cls):
        print('self in Desc: %s ' % self )
        print(self, ins, cls)
class Test:
    x = Desc()
    def prt(self):
        print('self in Test: %s' % self)
t = Test()
t.prt()
t.x
```

```
self in Test: <__main__.Test object at 0x0000025C9C4C5388>
self in Desc: <__main__.Desc object at 0x0000025C9C4C5308> 
<__main__.Desc object at 0x0000025C9C4C5308> <__main__.Test object at 0x0000025C9C4C5388> <class '__main__.Test'>
```

The main question here is: shouldn't the `self` defined in `Desc` be the instance `t` that calls it? Why does it become an instance of `Desc`?

Because the call here is `t.x`, meaning the attribute `x` of the `Test` instance `t`. Since instance `t` does not define an attribute named `x`, Python finds the class attribute `x`. That attribute is a descriptor attribute, which is an instance of `Desc`. No method on `Test` is being called here.

If we access `x` directly through the class, we get a similar result.

Below is the result after changing `t.x` to `Test.x`.

```
class Desc:
    def __get__(self, ins, cls):
        print('self in Desc: %s ' % self )
        print(self, ins, cls)
class Test:
    x = Desc()
    def prt(self):
        print('self in Test: %s' % self)
t = Test()
t.prt()
Test.x
```

```
self in Test: <__main__.Test object at 0x000001EF6BF34408>
self in Desc: <__main__.Desc object at 0x000001EF6B8FCF88> 
<__main__.Desc object at 0x000001EF6B8FCF88> None <class '__main__.Test'>
```

## Cases where `self` is not used

- Class methods focus more on calling a method from the class rather than from an instance. They can be used for constructor overloading or data conversion. The first parameter is the class object.

- Static methods mainly handle logic related to the class, such as data validation. The first parameter does not need to be either the class object or the instance `self`.

```
class Student(object):
    def __init__(self, name, score):
        self.name = name
        self.score = score
        
    def Printinfo(self):
        print(self.name + ':' + str(self.score))
    
    @classmethod
    def classFunction(cls, info):
        name, score = info.split(' ')
        stu = cls(name, score)
        return stu
    
    @staticmethod
    def ifgood(score):
        return score>=90

ryu = Student('ryu',98)
ryu.Printinfo()
# ryu.classFunction() would raise: classFunction() takes 0 positional arguments but 1 was given
li = Student.classFunction('li 89')
li.Printinfo()
li.ifgood(int(li.score))
Student.ifgood(99)
```

```
ryu:98
li:89

True
```
