# JS Notes

## First Look at JS

### How Browsers Execute JS

Browsers are divided into:

- Rendering engine: used to parse HTML and CSS, commonly called the browser kernel. For example, Chrome uses Blink.

- JS engine: also called a JS interpreter. It reads JS code and processes it in the background. For example, Chrome uses V8.

### Components of JS

- ECMAScript: JavaScript syntax. It specifies JS programming syntax and core foundational knowledge, and it is the JS syntax standard followed by browser vendors.

- DOM: Document Object Model. DOM interfaces can be used to operate on page elements, such as size, position, and color.

- BOM: Browser Object Model. BOM can be used to operate on the browser window.

### Naming Rules

- Consists of letters, digits, underscores, and dollar signs, such as `usrAge`, `num01`, `_name`.

- Strictly case-sensitive.

- Cannot start with a digit.

- Cannot be a keyword.

- Variable names must be meaningful.

- Follow camelCase: first letter lowercase, later words capitalized, such as `myFirstName`.

- Try not to use `name` as a variable name.

## Data Types

In JS, a variable's data type is determined by the value on the right side of the equals sign during program execution.

A variable's data type can change.

- Complex data type: object.

### Type Conversion

- Convert to string.

- Convert to number.

- Convert to boolean.

#### Convert to String

| Method | Description | Example |
| --- | --- | --- |
| `toString()` | Convert to string | `var num=1; alert(num.toString());` |
| `String()` forced conversion | Convert to string | `var num=1; alert(String(num));` |
| Plus concatenation, implicit conversion | Result is string when concatenated with a string | `var num=1; alert(num+'');` |

#### Convert to Number

| Method | Description | Example |
| --- | --- | --- |
| `parseInt(string)` | Converts string to integer number | `parseInt('78')` |
| `parseFloat(string)` | Converts string to floating-point number | `parseFloat('78.21')` |
| `Number()` forced conversion | Converts string to number | `Number('12')` |
| JS implicit conversion with `- * /` | Uses arithmetic operations for implicit conversion to number | `'12'-0` |

- The first two conversions only convert the leading part of the string that matches a valid number format.

For example, `parseInt(12.22.1)` is 12, and `parseFloat(12.22.1)` is 12.22.

`parseInt(12.2aaaaa2.1)` is 12, and `parseFloat(12.2aaaaa2.1)` is 12.2.

But `Number()` requires the entire string to be a valid number.

For example, `Number('12.22.1')` is `NaN`.

## JS Operators

### Arithmetic Operators

`%` modulo, or remainder.

```
console.log(5 % 3);
//2
```

Floating-point arithmetic has issues, meaning precision issues. Avoid direct arithmetic with floating-point numbers.

Do not directly compare computed floating-point numbers for equality.

```
var num = 0.2 + 0.1;
console.log(num == 0.3);
//false
```

### Increment and Decrement Operators

- Prefix increment:

`++num`, increment by 1, then return the value.

- Postfix increment:

`num++`, return the value first, then increment by 1.

```
// Prefix increment: increment first, then return.
var p = 10;
console.log(++p + 10);//21
// Postfix increment: return first, then increment.
p = 10;
console.log(p++ + 10);//20
//1. p++ = 11, but returns 10.
//2. ++p = 12.
p = 10;
console.log(p++ + ++p);//22
```

### Comparison Operators

```
// == can convert string type to number type.
console.log(18 == '18');//true
console.log(18 === '18');//false
```

### Logical Operators

| Logical operator | Description | Example |
| --- | --- | --- |
| && | Logical AND | `true&&false` |
| \|\| | Logical OR | `true\|\|false` |
| ! | Logical NOT | `!true` |

#### Logical Short-Circuiting

- `&&` short-circuit:

For `expression1 && expression2`, if expression1 is true, return expression2. If expression1 is false, return expression1.

```
console.log(123 && 456);//456
console.log(0 && 456);//0
console.log(0 && 456 && 78945);//0
```

- `||` short-circuit:

For `expression1 || expression2`, if expression1 is true, return expression1. If expression1 is false, return expression2.

```
console.log(123 || 456);//123
console.log(0 || 456);//456
console.log(0 || 456 || 2367);//456
console.log(0 || 0 || 2367);//2367
```

### Assignment Operators

| Assignment operator | Description | Example |
| --- | --- | --- |
| = | Direct assignment | `var usrName = 'Jack';` |
| +=, -= | Add/subtract a number and assign | `var age = 10; age += 5;//15` |
| *=, /=, %= | Multiply/divide/modulo and assign | `var age = 10; age *= 5;//50` |

### Operator Precedence

| Priority | Operator | Order |
| --- | --- | --- |
| 1 | Parentheses | `()` |
| 2 | Unary operators | `++ -- !` |
| 3 | Arithmetic operators | **`* / %` before `+ -`** |
| 4 | Relational operators | `> >= < <=` |
| 5 | Equality operators | `== != !== ===` |
| 6 | Logical operators | **`&&` before `\|\|`** |
| 7 | Assignment operator | `=` |
| 8 | Comma operator | `,` |

### Bubble Sort

Animated demo:

![](/assets/posts/cnblogs-17681162/image-01.gif)

Idea:

There are two layers:

The outer layer treats each traversal and comparison pass as one round. It compares `array length - 1` rounds in total, because pairwise comparison can be viewed as determining two numbers at the last comparison.

The inner layer compares adjacent left and right elements. If the left is greater than the right, swap them. Each time the outer loop completes one round, the number of comparisons in the inner loop decreases by 1, because each outer round determines one number's final position.

```
var arr = [3, 2, 5, 1, 4];
var tmp = 0;
for (var i = arr.length - 1; i > 0; i--) {
    for (var j = 0; j < i; j++) {
        if (arr[j] > arr[j + 1]) {
            tmp = arr[j];
            arr[j] = arr[j + 1];
            arr[j + 1] = tmp;
        }
    }
}
console.log(arr);
```

    var min = num1, max = num2, tmp = 0, sum = 0;
    if (num1 > num2) {

        min = max;
        max = tmp;
