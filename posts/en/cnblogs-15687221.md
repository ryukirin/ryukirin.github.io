### Linux directory structure

The following content comes from [Linux system directory structure](https://www.runoob.com/linux/linux-system-contents.html?accessToken=eyJhbGciOiJIUzI1NiIsImtpZCI6ImRlZmF1bHQiLCJ0eXAiOiJKV1QifQ.eyJhdWQiOiJhY2Nlc3NfcmVzb3VyY2UiLCJleHAiOjE2Mzk0MTAwNTYsImciOiJ2NVlCR25qSkRBTXJLeWhrIiwiaWF0IjoxNjM5NDA5NzU2LCJ1c2VySWQiOjB9.PuMUd4D9l5edf61NqfyDSbvuN1g3ni-MQ5pFES3VGnE).

- **/bin**:
`bin` is short for Binaries. This directory stores the most commonly used commands.

- **/boot**:
This directory stores core files used when booting Linux, including some linked files and image files.

- **/dev**:
`dev` is short for Device. This directory stores Linux external devices. In Linux, devices are accessed in the same way as files.

- **/etc**:
`etc` is short for Etcetera. This directory stores configuration files and subdirectories required for system administration.

- **/home**:
The user home directory. In Linux, every user has their own directory, usually named after the account, such as `alice`, `bob`, and `eve` in the screenshot above.

- **/lib**:
`lib` is short for Library. This directory stores the most basic dynamic shared libraries of the system, similar to DLL files on Windows. Almost all applications need these shared libraries.

- **/lost+found**:
This directory is usually empty. When the system shuts down abnormally, some files may be stored here.

- **/media**:
Linux automatically detects some devices, such as USB drives and optical drives. After detection, Linux mounts the devices under this directory.

- **/mnt**:
This directory is provided for users to temporarily mount other file systems. For example, we can mount a CD-ROM under `/mnt/`, then enter the directory to view its contents.

- **/opt**:
`opt` is short for optional. This directory is used for extra software installed on the host. For example, an Oracle database can be installed here. It is empty by default.

- **/proc**:
`proc` is short for Processes. `/proc` is a pseudo file system, or virtual file system, storing special files related to the current kernel runtime state. This virtual directory maps system memory. We can directly access this directory to obtain system information.

Its content is not stored on disk but in memory. Some files inside it can also be modified directly. For example, the following command blocks ping requests so that others cannot ping the machine:

```
echo 1 > /proc/sys/net/ipv4/icmp_echo_ignore_all
```

- **/root**:
The home directory of the system administrator, also known as the superuser.

- **/sbin**:
`s` means Super User, and `sbin` is short for Superuser Binaries. This directory stores system administration programs used by system administrators.

- **/selinux**:
This directory is specific to Red Hat/CentOS. SELinux is a security mechanism similar to the Windows firewall, but more complex. This directory stores SELinux-related files.

- **/srv**:
This directory stores data that services need after startup.

- **/sys**:
This is a major change introduced in the Linux 2.6 kernel. It contains the `sysfs` file system introduced in that kernel.

The `sysfs` file system integrates information from three file systems: the `proc` file system for process information, the `devfs` file system for devices, and the `devpts` file system for pseudo terminals.

It is an intuitive reflection of the kernel device tree.

When a kernel object is created, the corresponding file and directory are also created in the kernel object subsystem.

- **/tmp**:
`tmp` is short for temporary. This directory stores temporary files.

- **/usr**:
`usr` is short for Unix Shared Resources. This is an important directory. Many user applications and files are placed here, similar to `Program Files` on Windows.

- **/usr/bin**:
Applications used by system users.

- **/usr/sbin**:
More advanced administration programs and system daemons used by the superuser.

- **/usr/src**:
The default location for kernel source code.

- **/var**:
`var` is short for variable. This directory stores things that keep expanding. Directories that are frequently modified are usually placed here, including various log files.

- **/run**:
This is a temporary file system that stores information since system startup. When the system reboots, files under this directory should be deleted or cleared. If `/var/run` exists on your system, it should point to `run`.

In Linux, several directories are especially important. Be careful not to delete them by mistake or modify internal files casually.

**/etc**: as mentioned above, this contains system configuration files. Modifying files under this directory may cause the system to fail to boot.

**/bin, /sbin, /usr/bin, /usr/sbin**: these are default locations for executable files. For example, `ls` is located at `/bin/ls`.

It is worth noting that `/bin` and `/usr/bin` are commands for normal system users, except root, while `/sbin` and `/usr/sbin` are commands for root.

**/var**: this is a very important directory. Many programs run on the system, and each program produces corresponding logs. These logs are recorded under this directory, specifically under `/var/log`. Mail is also stored here by default.

### Linux file and directory management

- Absolute path: starts from the root directory `/`, for example `/usr/share/doc`.

- Relative path: for example, when moving from `/usr/share/doc` to `/usr/share/man`, you can write `cd ../man`.

#### Common commands

- `ls`, short for list files: list directories and filenames.

`-a`: list all files, including hidden files whose names start with `.`.

- `-d`: list only the directory itself, not the files inside it.

- `-l`: list detailed data, including file attributes and permissions.

`cd`, short for change directory: switch directories.

```
# Use mkdir to create the runoob directory.
[root@www ~]# mkdir runoob

# Use an absolute path to switch to the runoob directory.
[root@www ~]# cd /root/runoob/

# Use a relative path to switch to the runoob directory.
[root@www ~]# cd ./runoob/

# Return to your home directory, namely /root.
[root@www runoob]# cd ~

# Go to the parent directory of the current directory, namely the parent of /root.
[root@www ~]# cd ..
```

`pwd`, short for print work directory: display the current directory.

- **-P**: show the actual path rather than a link path.

`mkdir`, short for make directory: create a new directory.

- `-m`: configure directory permissions directly, without relying on default permissions from `umask`.

- `-p`: recursively create the required directory and its parent directories.

`rmdir`, short for remove directory: delete an empty directory.

- **-p**: starting from this directory, delete multiple levels of empty directories at once.

`cp`, short for copy file: copy files or directories.

- **-a**: equivalent to `-pdr`; see the explanations below.

- **-d**: if the source file is a link file, copy the link-file attributes rather than the target file itself.

- **-f**: force. If the target file already exists and cannot be opened, remove it and try again.

- **-i**: if the destination already exists, ask before overwriting.

- **-l**: create a hard link instead of copying the file itself.

- **-p**: copy file attributes as well, often used for backups.

- **-r**: recursively copy, commonly used for copying directories.

- **-s**: copy as a symbolic link, similar to a shortcut.

- **-u**: update the destination only when it is older than the source.

`rm`, short for remove: delete files or directories.

- `-f`: force, ignore nonexistent files and do not show warnings.

- `-i`: interactive mode; ask before deleting.

- `-r`: recursive deletion, commonly used for deleting directories. This is a very dangerous option.

`mv`, short for move file: move files and directories, or rename files and directories.

- `-f`: force. If the target file exists, overwrite directly without asking.

- `-i`: if the destination exists, ask whether to overwrite it.

- `-u`: update only if the destination exists and the source is newer.

Use *man [command]* to view each command's documentation.
