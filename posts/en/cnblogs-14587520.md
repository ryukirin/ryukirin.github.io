### Parameters

- `input_size`: input dimension, equal to `embedding_size`.

- `hidden_size`: dimension of `h`.

- `num_layers`: number of stacked LSTM layers. The default is 1.

- `bias`: whether the hidden state uses bias. The default is `true`. Bias is an offset value. Without bias, the center axis or starting point is 0.

- `batch_first`: if set to `True`, the first dimension of the input data is the batch dimension. The default is `False`. See [Understanding PyTorch's `batch_first` parameter](https://www.jianshu.com/p/41c15d301542) for more details.

- `dropout`: default value is 0. It controls whether a dropout layer is added after each RNN layer except the last one. The input is a decimal between 0 and 1 that represents probability. 0 means zero dropout probability, so no dropout is applied.

- `bidirectional`: whether to use bidirectional propagation. The default is `False`.

#### dropout

As the name suggests, dropout means that something is dropped. In neural networks, some neurons are dropped, so the layer is called a dropout layer. It is used to prevent overfitting.

During training for the first batch, the process is roughly:

- Set the dropout probability for each neural-network layer.

- Drop part of the neurons according to the probability, then start training. Update the parameters and weights for the neurons that were not dropped, and keep them.

- After all parameters are updated, drop another subset of neurons according to the same probability and start training again. If a neuron used in the new training pass was already trained in the first pass, continue updating its parameters. For a neuron dropped in the second pass but already updated in the first pass, keep its weights unchanged until a later batch where it is not dropped.

```
# num_layers

import torch.nn as nn
import torch
x = torch.rand(10,24,100)

# torch.nn.lstm(input_size=100,hidden_size=16,num_layers=2)
# Input dimension 100, h dimension 16, number of LSTM layers 2.
# Using sentence training as an example: suppose each word is represented by a 100-dimensional vector,
# each sentence has 24 words, and one training step contains 10 sentences.
# Then batch_size=10, seq=24, input_size=100.
# seq is the sentence length, and input_size is the input size for one x_t.
lstm = nn.LSTM(100,16,num_layers=2)
output,(h,c) = lstm(x)
print(output.size())
print(h.size())
print(c.size())
```

torch.Size([10, 24, 16])
torch.Size([2, 24, 16])
torch.Size([2, 24, 16])

```
import torch.nn as nn
import torch
x = torch.rand(10,24,100) # batch,seq,input_size
h0 = torch.rand(1,24,16)# num_layers*num_directions, batch, hidden_size
c0 = torch.rand(1,24,16)
lstm = nn.LSTM(100,16)
output,(h,c) = lstm(x,(h0,c0))
print(output.size())
print(h.size())
print(c.size())
```

Finally, here are the parameter dimensions.

The first row shows the input dimensions used when constructing the LSTM. The following rows show the dimensions of the parameters used by the LSTM, derived from the input dimensions.

For a bidirectional LSTM, `num_directions = 2`; otherwise, `num_directions = 1`.

| Parameter | Dimension 1 | Dimension 2 | Dimension 3 |
| --- | --- | --- | --- |
| lstm=nn.LSTM | input_size | hidden_size | num_layers |
| x | seq_len | batch | input_size |
| h0 | num_layers×num_directions | batch | hidden_size |
| c0 | num_layers×num_directions | batch | hidden_size |
| output | seq_len | batch | num_directions×hidden_size |
| hn | num_layers×num_directions | batch | hidden_size |
| cn | num_layers×num_directions | batch | hidden_size |
