### 5.5 Formatting strings

- `format(String format, Object...args)`

`format`: format string.

- `args`: parameters referenced by format specifiers in the format string. Extra parameters outside the format specifiers are ignored.

`format(Local I, String format, Object...args)`

- `I`: the locale applied during formatting. If null, no localization is applied.

- `format`: format string.

- `args`: parameters referenced by format specifiers in the format string. Extra parameters outside the format specifiers are ignored.

#### Formatting date and time strings

Convert the output time string into the format you want.

- Date formatting

| Conversion | Description | Example |
| --- | --- | --- |
| %te | day of month, 1-31 | 2 |
| %tb | abbreviated month in the specified locale | Feb (en), 二月 (zh) |
| %tB | full month name in the specified locale | February (en), 二月 (zh) |
| %tA | full weekday name in the specified locale | Monday (en), 星期一 (zh) |
| %ta | abbreviated weekday in the specified locale | Mon (en), 星期一 (zh) |
| %tc | full date and time information | 星期二 三月 25 13:37:22 CST 2005 |
| %tY | 4-digit year | 2008 |
| %tj | day of year, 001-366 | 085 |
| %tm | month | 03 |
| %td | day of month, 1-31 | 02 |
| %ty | 2-digit year | 08 |

- Time formatting

| Conversion | Description | Example |
| --- | --- | --- |
| %tH | 2-digit 24-hour hour, 00-23 | 14 |
| %tI | 2-digit 12-hour hour, 01-12 | 05 |
| %tk | 24-hour hour, 0-23 | 5 |
| %tl | 12-hour hour, 1-12 | 10 |
| %tM | 2-digit minute, 00-59 | 05 |
| %tS | 2-digit second, 00-60 | 12 |
| %tL | 3-digit millisecond, 000-999 | 920 |
| %tN | 9-digit nanosecond | 062000000 |
| %tp | AM or PM in the specified locale | 下午 (zh), pm (en) |
| %tz | numeric timezone offset relative to GMT, RFC 82 format | +0800 |
| %tZ | timezone abbreviation string | CST |
| %ts | seconds since 1970-01-01 00:00:00 | 1206426646 |
| %tQ | milliseconds since 1970-01-01 00:00:00 | 1206426737453 |

- Common formatted date-time combinations

| Conversion | Description | Example |
| --- | --- | --- |
| %tF | year-month-day format, 4-digit year | 2008-03-25 |
| %tD | month/day/year format, 2-digit year | 03/25/08 |
| %tc | full date and time information | 星期二 三月 25 15:20:00 CST 2008 |
| %tr | hour:minute:second PM(AM), 12-hour format | 03:22:06 下午 |
| %tT | hour:minute:second, 24-hour format | 15:23:50 |
| %tR | hour:minute, 24-hour format | 15:25 |

#### General type formatting

Can be used for any parameter type.

| Conversion | Description | Example |
| --- | --- | --- |
| %b %B | formatted as Boolean | true |
| %h %H | formatted as hash code | A05A5198 |
| %s %S | formatted as string | "abcd" |
| %c %C | formatted as character | 'a' |
| %d | formatted as decimal integer | 40 |
| %o | formatted as octal integer | 11 |
| %x %X | formatted as hexadecimal integer | 4b1 |
| %e | decimal number in scientific notation | 1.700000e+01 |
| %a | hexadecimal floating-point number with significant digits and exponent | 0X1.C000000000001P4 |
| %n | platform-specific line separator |  |
| %% | literal `%` | % |

```
import java.util.Date;

public class Java5_5 {
    public static void main(String[] args){
        // Date formatting
        Date date = new Date();
        String year = String.format("%tY", date);
        String month = String.format("%tB", date);
        String day = String.format("%td", date);
        System.out.println(year + "年" + month + day + "日");

        // Time formatting
        String hour = String.format("%tH", date);
        String minute = String.format("%tM", date);
        String second = String.format("%tS", date);
        System.out.println(hour + "时" + minute + "分" + second + "秒");

        // Common date-time combinations
        String time = String.format("%tc", date);
        String from = String.format("%tF", date);
        System.out.println("全部的时间信息：" + time);
        System.out.println("年-月-日格式：" + from);

        // General type formatting
        String str = String.format("%d", 400 / 2);
        String str2 = String.format("%b", 3 > 5);
        String str3 = String.format("%x", 200);
        System.out.println("400的一半是" + str);
        System.out.println("3>5正确吗？" + str2);
        System.out.println("200的16进制数是" + str3);
    }
}
```

### 5.6 Using regular expressions

Regular expressions are used to determine whether a character in a statement satisfies a certain format.

The key to regular expressions is knowing how to write the pattern.

| Metacharacter | Written in regular expression | Meaning |
| --- | --- | --- |
| . | . | any single character |
| \d | \\d | any digit from 0 to 9 |
| \D | \\D | any non-digit character |
| \s | \\s | whitespace, such as '\t' and '\n' |
| \S | \\S | non-whitespace character |
| \w | \\w | character usable as an identifier, excluding "$" |
| \W | \\W | character not usable as an identifier |
| \p | \\p | lowercase letters a-z |
| \p | \\p | uppercase letters A-Z |
| \p | \\p | ASCII characters |
| \p | \\p | letter characters |
| \p | \\p | decimal digits 0-9 |
| \p | \\p | digits or letters |
| \p | \\p | punctuation characters `!"#$%'()*+,-./:;<=>?@[\\]^_`{\|}~` |
| \p | \\p | visible characters [\p{Alnum}\p{Punct}] |
| \p | \\p | printable characters [\p{Graph}\x20] |
| \p | \\p | space or tab [\t] |
| \p | \\p | control characters [\x00-\x1F\x7F] |

- To express a literal `.` in a regular expression, an escape character is needed.

- `^` matches the beginning, and `$` matches the end.

- Several characters can be enclosed in square brackets to form a metacharacter. That metacharacter can represent any character inside the brackets.

Example: `[abc]4` means `a4`, `b4`, or `c4` can all be matched.

`[^456]`: any character except 4, 5, and 6.

- `[a-r]`: any character from `a` to `r`.

- `[a-zA-Z]`: any English letter.

- `[a-e[g-z]]`: any character from `a-e` or `g-z`, union operation.

- `[a-o&&[def]]`: `d`, `e`, `f`, intersection operation.

- `[a-d&&[^bc]]`: `a`, `d`, difference operation.

| Quantifier | Meaning | Example |
| --- | --- | --- |
| ? | zero or one time | A? |
| * | zero or more times | A* |
| + | one or more times | A+ |
| `{n}` | exactly n times | `A\{2}` |
| `{n,}` | at least n times | `A{3,}` |
| `{n,m}` | n to m times | `A{2,6}` |
| `^` | matched string is at the beginning | `^A` |
| `dollar sign` | matched string is at the end | `A dollar sign` |

`$` always caused formatting issues in the table, so I used "dollar sign" instead.

```
public class Java5_6 {
    public static void main(String[] args){
        // \\w+: one or more identifier characters @ \\w+: one or more identifier characters
        // (\\.\\w{2,3})*: dot plus two or three identifier characters.
        // This mainly recognizes suffixes such as .com .cn .org, and the group appears zero or more times.
        // \\.\\w{2,3}: final .com .cn .org etc.
        // This regular expression may not match all email addresses.
        String regex = "\\w+@\\w+(\\.\\w{2,3})*\\.\\w{2,3}";
        String str1 = "aaa@";
        String str2 = "aaaaa";
        String str3 = "111@111ffyu.dfg.com";
        if (str1.matches(regex)){
            System.out.println(str1 + "是一个合法的E-mail地址");
        }
        else if (str2.matches(regex)){
            System.out.println(str2 + "是一个合法的E-mail地址");
        }
        else if (str3.matches(regex)){
            System.out.println(str3 + "是一个合法的E-mail地址");
        }
    }
}
```

### 5.7 String builder

Once a string object is created, its length is fixed and its content cannot be changed or compiled.

Although `+` can append new characters or strings, it creates a new `String` instance and a new string object in memory.

J2SE 5.0 introduced the mutable character sequence class `StringBuilder`.

A newly created `StringBuilder` has an initial capacity of 16 characters. If exceeded, its length grows automatically.

To output a string result from `StringBuilder`, use `toString`.

- `append()`: append. It accepts any data type, such as `int`.

- `insert(int offset, arg)`: insert at a specified position. It accepts any data type.

- `delete(int start, int end)`: delete a string at a specified position.

```
public class Java5_7 {
    public static void main(String[] args) {
        String str = "";
        long starTime = System.currentTimeMillis();
        for (int i=0; i < 10000; i++){
            str = str + i;
        }
        long endTime = System.currentTimeMillis();
        long time = endTime - starTime;
        System.out.println("String消耗时间：" + time);
        StringBuilder builder = new StringBuilder("");
        starTime = System.currentTimeMillis();
        for (int j=0; j < 10000; j++){
            builder.append(j);
        }
        endTime = System.currentTimeMillis();
        time = endTime - starTime;
        // Time cost is clearly shorter.
        System.out.println("StringBuilder消耗时间：" + time);
    }
}
```
