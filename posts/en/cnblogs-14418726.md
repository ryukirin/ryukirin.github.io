This module implements specialized container datatypes that provide alternatives to Python's built-in `dict`, `list`, `set`, and `tuple`.

- `ChainMap`: a dict-like container class that groups multiple mappings into a single view.

- `Counter`: a `dict` subclass that counts hashable objects.

- `deque`: a list-like container optimized for fast appends and pops at both ends.

- `defaultdict`: a `dict` subclass that calls a factory function to provide default values for missing keys.

- `namedtuple()`: a factory function for creating tuple subclasses with named fields.

- `OrderedDict`: a `dict` subclass that preserves insertion order.

- `UserDict`: a wrapper around dictionary objects that simplifies subclassing dictionaries.

- `UserList`: a wrapper around list objects that simplifies subclassing lists.

- `UserString`: a wrapper around string objects that simplifies subclassing strings.

## ChainMap

`ChainMap` is designed to quickly link multiple mappings together so they can be treated as a single unit.

It is usually much faster than creating a new dictionary and calling `update()` multiple times.

```
collections.ChainMap(*maps)
```

If no `maps` are specified, an empty dictionary is provided by default, so a new chain always has at least one mapping.

```
from collections import ChainMap

d1 = {'apple':1,'banana':2}
d2 = {'orange':2,'apple':3,'pike':1}

combined_d = ChainMap(d1,d2)
reverse_combind_d = ChainMap(d2,d1)

print(combined_d) 
print(reverse_combind_d)

'''
When iterating item by item, output starts from the first item of the second dict.
When both dicts contain the same key, only the value from the first dict is returned.
'''
for k,v in combined_d.items():
    print(k,v)
print('------------------')    
for k,v in reverse_combind_d.items():
    print(k,v)
```

## Counter

`Counter` is a `dict` subclass. It is mainly used to count the frequency of objects you access, and it provides a fast and convenient counting tool.

Common methods:

- `elements()`: returns an iterator over elements, repeating each element as many times as its count. Elements with counts below 1 are ignored.

- `most_common([n])`: returns a list of the `n` most common elements and their counts.

- `subtract([iterable-or-mapping])`: subtracts elements from an iterable or mapping. Input and output counts may be zero or negative.

- `update([iterable-or-mapping])`: counts elements from an iterable or adds counts from another mapping object or counter.

```
from collections import Counter

# Count character occurrences.
print(Counter('hello world'))

# Count words.
c = Counter('hello world hello world hello nihao'.split())
print(c)

cnt = Counter()
for word in ['red', 'blue', 'red', 'green', 'blue', 'blue']:
    cnt[word] += 1
print(cnt)

# Get the access count for a specified object; get() also works.
print(c['hello'])
print(c.get('hello'))

# Inspect elements.
print(list(c.elements()))
print(c.elements())

# Add objects, or use c.update(d).
c = Counter('hello world hello world hello nihao'.split())
d = Counter('hello world'.split())
print(c)
print(d)
print(c + d)
c.update(d)
print(c)

# Subtract objects, or use c.subtract(d).
c = Counter('hello world hello world hello nihao'.split())
print(c - d)
c.subtract(d)
print(c)

# Clear.
c.clear()
print(c)
```

## deque

`collections.deque` returns a new double-ended queue object, initialized from left to right by using `append()` over the data from `iterable`. If `iterable` is not specified, the new queue is empty.

A deque can be used as a stack or a queue. It supports thread-safe, O(1) appends and pops from either end.

Although list objects support similar operations, `deque` is optimized for fixed-length operations such as `pop(0)` and `insert(0, v)`.

If `maxlen` is not specified or is `None`, deques may grow to any length. Otherwise, the deque is bounded to the specified maximum length.

Once a bounded deque is full, adding new items discards the same number of items from the opposite end.

Supported methods:

- `append(x)`: add `x` to the right side.

- `appendleft(x)`: add `x` to the left side.

- `clear()`: remove all elements and set the length to 0.

- `copy()`: create a shallow copy.

- `count(x)`: count the number of elements equal to `x`.

- `extend(iterable)`: add elements from `iterable` to the right side.

- `extendleft(iterable)`: add elements from `iterable` to the left side. Note that when adding on the left, the order of the iterable is reversed.

- `index(x[,start[,stop]])`: return the position of `x` in the deque after index `start` and before index `stop`. It returns the first match and raises `ValueError` if none is found.

- `insert(i,x)`: insert `x` at position `i`. If this would make a bounded deque exceed `maxlen`, an `IndexError` is raised.

- `pop()`: remove the rightmost element.

- `popleft()`: remove the leftmost element.

- `remove(value)`: remove the first matching `value`. If not found, raise `ValueError`.

- `reverse()`: reverse the deque in place. Returns `None`.

- `maxlen`: the maximum size of the queue, or `None` if unbounded.

```
from collections import deque
d = deque(maxlen=20)
print(d)
d.extend('python')
print(d)

for elem in d:
    print(elem.upper())
```

## defaultdict

`defaultdict` returns a new dictionary-like object.

`defaultdict` is a subclass of the built-in `dict` class. It overrides one method and adds one writable instance variable.

The remaining behavior is the same as `dict`, so I will not repeat it here.

This object has an attribute named `default_factory`. During construction, the first argument provides the initial value for this attribute, and the default is `None`.

All other arguments, including keyword arguments, are equivalent to the arguments passed to the `dict` constructor.

```
from collections import defaultdict
d = defaultdict()
print(d)

# Use list as default_factory.
s = [('yellow', 1), ('blue', 2), ('yellow', 3), ('blue', 4), ('red', 1)]
d = defaultdict(list)

# Add directly. If you need to add a value, call .append(value) after [].
d['hello']
print(d)
print(sorted(d.items()))

for k, v in s:
    d[k].append(v)

print(sorted(d.items()))
```

```
defaultdict(None, {})
defaultdict(<class 'list'>, {'hello': []})
[('hello', [])]
[('blue', [2, 4]), ('hello', []), ('red', [1]), ('yellow', [1, 3])]
```

```
# Set default_factory to int so defaultdict can be used for counting.
s = 'mississippi'
d = defaultdict(int)
for k in s:
    d[k] += 1

print(sorted(d.items()))
```

```
[('i', 4), ('m', 1), ('p', 2), ('s', 4)]
```

## namedtuple()

Named tuples assign meaning to each position, which improves readability and self-documentation.

They can be used wherever regular tuples can be used, while also supporting access by field name. Access by index still works as well.

(Looking at the example, it feels a bit like an entity class in Java.)

```
from collections import namedtuple
Person = namedtuple('Person', ['age', 'height', 'name'])
Human = namedtuple('Human', 'age, height, name')
student = namedtuple('student', 'age height name')

tom = Person(30,178,'Tom')
jack = Human(20,179,'Jack')

print(jack)
print(tom)

print(tom.age)

print(jack.height)
```

```
Human(age=20, height=179, name='Jack')
Person(age=30, height=178, name='Tom')
30
179
```

## OrderedDict

An ordered dictionary works like a regular dictionary, but it provides a few extra features related to ordering operations.

Because the built-in `dict` class gained the ability to remember insertion order, which is guaranteed in Python 3.7 and later, `OrderedDict` has become less important.

Some differences from `dict` still remain:

- Regular `dict` is designed to be very good at mapping operations. Tracking insertion order is secondary.

- `OrderedDict` is designed to be good at reordering operations. Space efficiency, iteration speed, and update performance are secondary.

- Algorithmically, `OrderedDict` handles frequent reordering operations better than `dict`. This makes it suitable for tracking recent access, for example in an LRU cache.

- Equality checks for `OrderedDict` compare order as well.

- The `popitem()` method of `OrderedDict` has a different signature. It accepts an optional argument that specifies which element to pop.

- `OrderedDict` has a `move_to_end()` method that efficiently moves an element to either end.

- Before Python 3.8, `dict` did not have a `reversed()` method.

```
from collections import OrderedDict

od = OrderedDict()
od['country'] = '1'
od['sex'] = '0'
od['age'] = '34'

print(od)
```

```
OrderedDict([('country', '1'), ('sex', '0'), ('age', '34')])
```

```
od.move_to_end('country')
print(od)
```

```
OrderedDict([('sex', '0'), ('age', '34'), ('country', '1')])
```

## UserDict

`UserDict` acts as a wrapper around dictionary objects. The need for this class has been partially replaced by the ability to subclass `dict` directly.

However, this class is easier to work with because the underlying dictionary is available as an attribute.

## UserList

`UserList` wraps a list object. It is a useful base class for list-like classes you want to customize, because you can inherit from it, override existing methods, and add new ones.

This lets us add new behavior to lists.

The need for this class has been partially replaced by the ability to subclass `list` directly.

However, this class is easier to work with because the underlying list is available as an attribute.

## UserString

`UserString` acts as a wrapper around string objects. The need for this class has been partially replaced by the ability to subclass `str` directly.

However, this class is easier to work with because the underlying string is available as an attribute.
