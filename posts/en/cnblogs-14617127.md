First, `packed` means packaged; `padded` means filled with padding; `pack` means to package or compress; `pad` means to fill.

`pack_padded_sequence` means compressing and wrapping padded sentences.

`pad_packed_sequence` means padding sentences that have been compressed and wrapped.

**pad_packed_sequence** is the reverse of the process above.

### pack_padded_sequence

```
# Official function
torch.nn.utils.rnn.pack_padded_sequence(input, lengths, batch_first=False)
# Returned value
return PackedSequence  # a PackedSequence object
```

Function: compresses a padded variable-length sequence and returns a `PackedSequence` object.

#### Parameters

- `input`: the data to compress. When `batch_first` is `False`, the input shape is `[B, S, *]`, where `B` is `batch_size`, `S` is `seq_len` (the length of the longest sequence in the batch), and `*` can be any additional dimension. When `batch_first` is `True`, the corresponding data format must be `[S, B, *]`. The input must be sorted by sequence length, with longer sequences first and shorter sequences later. The data at the first time step must belong to the longest sequence.

- `lengths`: the length of each sequence in the input data.

- `batch_first`: when `True`, the data format must be `[B, S, *]`; otherwise, the default is `False`.

### pad_packed_sequence

```
torch.nn.utils.rnn.pad_packed_sequence(sequence, batch_first=False) → tuple
# Returned value
return (sequence_pad , list) # This tuple contains the padded sequence and the length list for sequences in the batch.
```

Function: the function above compresses a padded variable-length sequence. This operation is the reverse of `pack_padded_sequence()`: it pads the compressed sequence back. By default, the padding length is determined by the largest number in the returned list.

#### Parameters

- `sequence`: the batch to be padded. It is a `PackedSequence` object.

- `batch_first`: same role as `batch_first` in `pack_padded_sequence`, but it affects the output data format.

### Example

```
import torch

batch_size = 3   # There are 3 sequences in this batch.
max_len = 6       # The longest sequence length is 6.
embedding_size = 8 # Embedding vector size is 8.
hidden_size = 16   # Hidden vector size is 16.
vocab_size = 20    # Vocabulary size is 20.

input_seq = [[3, 5, 12, 7, 2, ], [4, 11, 14, ], [18, 7, 3, 8, 5, 4]]
lengths = [5, 3, 6]   # Effective length of each sequence in the batch.
# embedding
embedding = torch.nn.Embedding(vocab_size, embedding_size, padding_idx=0)
# LSTM recurrent neural network
lstm = torch.nn.LSTM(embedding_size, hidden_size)
```

Because `pack_padded_sequence` requires `input` to be sorted by sequence length, with longer sequences first and shorter sequences later, the data needs to be sorted first.

*`zip` is covered in the supplement.*

```
# Sort from longest to shortest.
input_seq = sorted(input_seq, key = lambda tp: len(tp), reverse=True)
lengths = sorted(lengths, key = lambda tp: tp, reverse=True)

print(input_seq)
print(lengths)

PAD_token = 0 # Padding index is 0.
def pad_seq(seq, seq_len, max_length):
    seq = seq
    seq += [PAD_token for _ in range(max_length - seq_len)]
    return seq

pad_seqs = []  # Padded data
for i,j in zip(input_seq, lengths):
    pad_seqs.append(pad_seq(i, j, max_len))

print(pad_seqs)
```

[[18, 7, 3, 8, 5, 4], [3, 5, 12, 7, 2], [4, 11, 14]]
[6, 5, 3]
[[18, 7, 3, 8, 5, 4], [3, 5, 12, 7, 2, 0], [4, 11, 14, 0, 0, 0]]

After padding, the sentences have the following format:

```
[[18, 7, 3, 8, 5, 4], 
 [3, 5, 12, 7, 2, 0], 
 [4, 11, 14, 0, 0, 0]]
```

That is `[batch_size, seq_len, *]`, so `batch_first=True` should be used in `pack_padded_sequence`.

LSTM output dimensions:

If the LSTM is bidirectional, `num_directions=2`; otherwise, `num_directions=1`.

| Parameter | Dimension 1 | Dimension 2 | Dimension 3 |
| --- | --- | --- | --- |
| output | seq_len | batch | num_directions×hidden_size |

```
pad_seqs = torch.tensor(pad_seqs)
embeded = embedding(pad_seqs)

# Compress and set batch_first to true.
pack = torch.nn.utils.rnn.pack_padded_sequence(embeded, lengths, batch_first=True)
# If batch_first is not specified, your data must be [s,b,e], otherwise a length error will occur.

# Test the result using the LSTM recurrent neural network.
state = None
pade_outputs, _ = lstm(pack, state)
# Set batch_first to true. It can also be false; that only affects the structure, not the result.
pade_outputs, others = torch.nn.utils.rnn.pad_packed_sequence(pade_outputs, batch_first=True)

# View the output tuple.
print(pade_outputs.shape)
print(others)
```

torch.Size([3, 6, 16])
tensor([6, 5, 3])
