# CSS Notes

CSS is used to make web pages look better.

CSS stands for Cascading Style Sheets.

It is sometimes also called a CSS style sheet or cascading style sheet.

It is also a markup-style language.

## Introduction to CSS

Selector + declaration:

```
h1 {
    color: red;
    ...
}
```

Selector `{ property: value }`

### CSS Code Style

- Style formatting: use expanded format, one style per line.

- Case: use lowercase.

- Spacing:

Keep one space before the property value and after the colon.

- Keep one space between the selector and the brace.

## Basic CSS Selectors

A selector is used to select tags.

Basic selectors consist of a single selector:

- Type selector: uses an HTML tag name as the selector. It selects all tags of that type and applies styles to them.

```
tagName {
	attribute1: value1;
    attribute2: value2;
    attribute3: value3;
    ...
}
```

- Class selector: selects one or several tags individually, which allows differentiated styling.

```
.className {
	attribute1: value1;
	...
}
```

Class selectors are identified by `.` followed by the class name.

- This can be understood as giving the tag a name.

- Long names or phrases can use hyphens in selector names.

- Do not use pure numbers or Chinese names. Use English where possible.

- Names should be meaningful so others can immediately understand what they are.

Class selectors have a special usage called multiple class names:

```
<div class="className1 className2">
    ...
</div>
```

This separates common styles from unique styles, saves code, and makes later changes easier.

ID selector:

```
#idName {
    attribute1: value1;
    ...
}
```

The biggest difference from class selectors is that an ID can only be called once.

In development, it is usually used for unique page elements and often paired with JavaScript.

Universal selector: selects all tags.

```
* {
    attribute1: value1;
    ...
}
```

It does not need to be called and is applied automatically. Use it only in special scenarios.

| Basic selector | Purpose | Characteristics | Usage frequency | Example |
| --- | --- | --- | --- | --- |
| Type selector | Selects all identical tags, such as `p` | Cannot differentiate individual elements | Often | `p { color: red; }` |
| Class selector | Selects one or more tags | Can select based on need | Very often | `.nav { color: red; }` |
| ID selector | Selects only one tag at a time | The ID attribute can appear only once per HTML document | Usually used with JS | `#nav { color: red; }` |
| Universal selector | Selects all tags | Selects too much, including some unnecessary elements | Special cases | `* { color: red; }` |

## CSS Font Properties

### Font Family

Use `font-family` to define the font family for text.

```
p {
    font-family: "Microsoft Yahei";
}
div {
    font-family: Arial,"Microsoft Yahei","Microsoft Yahei";
}
```

Try to use system default fonts so the page displays normally in any user's browser.

Common fonts: `"Microsoft Yahei"`, `tahoma`, `arial`, `"Hiragino Sans GB"`.

### Font Size

Use `font-size`; the `px` unit is required. Chrome defaults to 16px, but do not rely on the default. Specify the size explicitly.

Heading tags are special and need their size specified separately.

### Font Weight

`font-weight`:

- `normal`: normal.

- `bold`: bold.

- `bolder`: extra bold.

- `lighter`: lighter.

- `number`: use numbers to control weight, without units, from 100 to 900 in hundreds. This is preferred in real development. `400` is equivalent to `normal`.

### Font Style

`font-style`:

- `normal`: default standard font style.

- `italic`: italic.

### Shorthand Property

```
body {
    font: font-style font-weight font-size/line-height font-family;
}
```

The order cannot be changed. Unneeded properties can be omitted and will take default values, but `font-size` and `font-family` must remain, otherwise the shorthand will not work.

| Property | Meaning | Notes |
| --- | --- | --- |
| font-size | Font size | Usually uses px and must include a unit |
| font-family | Font | Usually follows the team's convention |
| font-weight | Font weight | Bold is 700 or bold; normal is normal or 400; numbers have no unit |
| font-style | Font style | Italic is italic; non-italic is normal. In work, normal is often used to cancel italic style |
| font | Font shorthand | Has an order; positions cannot be swapped; font size and family must remain |

## CSS Text Properties

### Text Color

`color`

- Predefined color values, such as `red` and `blue`.

- Hexadecimal, such as `#FF0000`; this is most common in development.

- RGB code, such as `rgb(255,0,0)` or `rgb(100%,0%,0%)`.

### Alignment

`text-align`

- `left`: left aligned, default.

- `right`: right aligned.

- `center`: centered.

### Text Decoration

`text-decoration`

- `none`: default, no decoration. Most commonly used to remove the default underline from links.

- `underline`: underline, commonly used.

- `overline`: overline, almost never used.

- `line-through`: strikethrough, not commonly used.

### Text Indentation

`text-indent`

- `px`: pixels.

- `em`: relative unit. It equals the size of one character of the current element's `font-size`. If not set, it uses the parent element's character size.

### Line Height

`line-height`. Line spacing = upper spacing + text height + lower spacing.

Measuring line height: measure from the bottom of the previous line to the bottom of the current line.

| Property | Meaning | Notes |
| --- | --- | --- |
| color | Text color | Commonly uses shorthand hexadecimal, such as `#FFF` |
| text-align | Text alignment | Sets alignment |
| text-indent | Text indentation | First-line indentation by two characters: `text-indent: 2em;` |
| text-decoration | Text decoration | Key usage: add underline with `underline`, remove underline with `none` |
| line-height | Line height | Controls distance between lines; line spacing = upper spacing + text height + lower spacing |

## Ways to Include CSS

Three types of style sheets:

- Inline style sheet.

- Internal style sheet.

- External style sheet.

### Internal Style Sheet

```
<style>
	css...
</style>
```

In theory, it can be placed anywhere in the HTML document, but it is usually placed inside the `head` tag. It can control the style of the whole page. Structure and style are separated, but not completely.

### Inline Style Sheet

Suitable for simple style changes.

```
<div style="color: red;">
    ...
</div>
```

It controls only the current tag's style and is not recommended for heavy use.

### External Style Sheet

This is the most used approach. Write CSS in a separate `.css` file and reference it from the HTML file.

- Create a stylesheet file with the `.css` suffix and put all CSS code inside.

- Import it into the HTML file.

```
<link rel="stylesheet" href="xxx.css">
```

- `rel`: specifies the relationship between the current document and the linked document. Here it is `stylesheet`.

- `href`: path to the linked document, either absolute or relative.

| Style sheet | Advantages | Disadvantages | Usage | Scope |
| --- | --- | --- | --- | --- |
| Inline | Easy to write, high weight | Mixes structure and style | Less | One tag |
| Internal | Partially separates structure and style | Not fully separated | Often | One page |
| External | Fully separates structure and style | Needs to be imported | Most | Multiple pages |

When settings conflict, whichever `style` or `link` appears later, closer to `body`, takes effect.

## Emmet Syntax

### Quickly Generating HTML Structure

- Generate a tag: type the tag name and press Tab.

- Generate multiple identical tags: tag name + `*` + Tab.

- Parent-child tags: use `>`, for example `ul>li` generates an unordered list.

- Sibling tags: use `+`.

- Generate tags with class or ID by writing `#id` or `.class`. The default tag is `div`; prepend a tag name to generate other tags. For example, `ul>li#two` becomes:

```
<ul>
    <li id="two"></li>

```

```

<div class="demo2"></div>
<div class="demo3"></div>
<div class="demo4"></div>
<div class="demo5"></div>
```

- To generate content inside tags, use `{}`. Example: `div{test}*5` becomes:

```
<div>test</div>
<div>test</div>
<div>test</div>
<div>test</div>
<div>test</div>
```

## Compound CSS Selectors

They can select target elements more accurately and efficiently.

- Descendant selector.

- Child selector.

- Grouping selector.

- Pseudo-class selector.

### Descendant Selector

Also called a containment selector.

It selects child elements inside a parent element.

### Child Selector

Selects only the nearest child elements of an element. In simple terms, direct children only.

```
div>p {
    ...
}
```

### Grouping Selector

Selects multiple groups of tags and defines styles for them at the same time.

```
div,
p {
    ...
}
```

### Pseudo-Class Selector

Used to add special effects to certain selectors, such as special effects for links or selecting the first or nth element.

Characteristic: written with a colon, such as `:hover`.

There are many types, including link pseudo-classes and structural pseudo-classes.

#### Link Pseudo-Classes

```
/* Selects all unvisited links. */
a:link
/* Selects all visited links. */
a:visited
/* Selects links under the mouse pointer. */
a:hover
/* Selects active links, meaning mouse pressed but not released. */
a:active
```

Do not change the order. Use LVHA order, otherwise it may not work.

Links have separate default browser styles. Styles set on `body` do not apply to links, so link styles must be specified separately.

#### focus Pseudo-Class

Used to select form elements that have focus.

Generally, `input` form elements can receive focus.

```
input:focus {
    background-color: yellow;
}
```

| Selector | Purpose | Feature | Usage | Separator/example |
| --- | --- | --- | --- | --- |
| Descendant selector | Select descendant elements | Can be children or deeper descendants | Often | Space, `div a` |
| Child selector | Select nearest-level elements | Direct children only | Less | Greater-than sign, `div>a` |
| Grouping selector | Select elements with shared styles | Can be used for collective declaration | Often | Comma, `div,a` |
| Link pseudo-class selector | Select different link states | Related to links | Often | In development, usually use `a {}` for normal link style and `a:hover` for hover style |
| focus pseudo-class selector | Select forms with cursor focus | Related to forms | Less | `input:focus` |

## CSS Element Display Modes

This means how an element is displayed.

Generally, elements are divided into block elements and inline elements.

### Block Elements

Examples include `h1` to `h6`, `p`, `div`, `ul`, `ol`, and `li`. Among them, `div` is the most typical block element.

- Occupies one line by itself.

- Height, width, margin, and padding can all be controlled.

- Width defaults to 100% of the container, meaning the parent width.

- It is a container and a box, and can contain inline or block-level elements.

Notes:

- Text-type elements should not contain block-level elements.

- `p` and `h1` to `h6` are text-type elements.

### Inline Elements

Also called inline elements.

Common examples include `a`, `strong`, `b`, `em`, `i`, `del`, `s`, `ins`, `u`, and `span`.

`span` is the most typical one.

- Width and height settings do not take effect.

- Inline elements can only contain text or other inline elements.

- An `a` tag cannot contain another `a`.

- In special cases, an `a` can contain a block-level element, but it is better to convert `a` to block mode.

### Inline-Block Elements

Elements such as `img`, `input`, and `td` have characteristics of both block and inline elements. Some materials call them inline-block elements.

- They appear on one line, but have gaps; multiple inline-block elements can be placed on one line.

- Their default width is the width of their own content.

- Height, line height, margin, and padding can be controlled.

| Element mode | Arrangement | Styling | Default width | Contains |
| --- | --- | --- | --- | --- |
| Block-level element | One element per line | Width and height can be set | 100% of the container | Can contain any tag |
| Inline element | Multiple per line | Width and height cannot be set | Width of its own content | Text or other inline elements |
| Inline-block element | Multiple inline-blocks per line | Width and height can be set | Width of its own content |  |

### Converting Display Modes

In special cases, an element needs another mode's characteristics.

For example, to increase the clickable area of a link `a`:

- Convert to block element: `display: block;`

- Convert to inline element: `display: inline;`

- Convert to inline-block element: `display: inline-block;`

### Vertical Centering

Single-line text can be vertically centered by setting line height equal to box height.

For example, text inside a `div` can be centered by setting `line-height` to the same value as `height`.

## CSS Backgrounds

### Background Color

`background-color: transparent | color;`

Transparent, which is the default, or a color.

### Background Image

Very convenient for controlling position.

`background-image: none | url();`

### Background Repeat

`background-repeat: repeat | no-repeat | repeat-x | repeat-y;`

Repeat, no repeat, repeat horizontally, repeat vertically.

### Background Image Position

`background-position: x y;`

`x` and `y` can be:

- Exact units: percentages or length values. The first value is definitely X, and the second is Y.

- Direction keywords: `top`, `center`, `bottom`, `left`, `right`. In this case, order does not matter. If the second parameter is omitted, it defaults to center.

- Mixed units: one exact unit and one direction keyword. The order must be X first and Y second.

### Fixed Background Image

`background-attachment: fixed | scroll;`

Fixed or scrolling.

### Shorthand

`background: color image repeat attachment position`

There is no fixed order, but this is the common order.

### Semi-Transparent Background Color

`background: rgba(0,0,0,0);`

The last value means opacity, between 0 and 1. `1` is opaque, `0` is fully transparent.

It does not affect the opacity of content.

| Property | Purpose | Values |
| --- | --- | --- |
| background-color | Background color | Predefined color value, hex, RGB |
| background-image | Background image | `url(path)` |
| background-repeat | Whether to repeat | `repeat` \| `no-repeat` \| `repeat-x` \| `repeat-y` |
| background-position | Background position | `x y`, exact length or direction keyword |
| background-attachment | Background attachment | `scroll` \| `fixed` |
| Background shorthand | Simpler writing | Color, image, repeat, scroll, position |
| Semi-transparent background color | Semi-transparent background | `rgba(0,0,0,0)`, last value is opacity between 0 and 1 |

## Three Major CSS Characteristics

Cascade, inheritance, and specificity.

### Cascade

When the same selector sets the same style, the cascade principle is triggered:

- For conflicting styles, the nearest style to the structure takes effect. For imported files, the lower definition takes effect.

- Non-conflicting styles do not cascade over each other.

### Inheritance

Child tags inherit some styles from parent tags, which simplifies code.

Inheritable properties include `text-*`, `font-*`, `line-*`, `color`, and similar properties.

#### Inheritance of Line Height

Line height can have a unit or no unit. Without a unit, it means a multiple of the current child element's font size.

The biggest benefit of omitting the unit is that child elements can automatically adjust based on their own text size.

### Specificity

When multiple selectors target one element:

- If selectors are the same, apply the cascade.

- If selectors are different, apply selector specificity.

Weights:

| Selector | Selector weight, does not carry between columns and only adds within the same column, compared from left to right |
| --- | --- |
| Inheritance or `*` | 0,0,0,0 |
| Element selector | 0,0,0,1 |
| Class selector, pseudo-class selector | 0,0,1,0 |
| ID selector | 0,1,0,0 |
| Inline style `style=""` | 1,0,0,0 |
| `!important` | Infinite |

Browsers set a default style for links: blue with underline. Therefore, inherited `text-decoration` and `color` are ineffective for links.

#### Specificity Addition

Compound selectors add specificity, so the weight must be calculated.

Weights do not carry between columns and only add at the same position. Comparison is from left to right.

Note:

The specificity of `a:hover` is `0,0,1,1`, not `0,0,1,0`, because there is a type selector before it.

Example: [A hover failure caused by specificity](https://blog.csdn.net/weixin_46102597/article/details/104228185)

## Box Model

Web page layout process:

- First prepare the relevant page elements. Web elements are basically boxes.

- Use CSS to style the boxes and place them in the corresponding positions.

- Put content inside the boxes.

The core of web layout is arranging layout with boxes.

### Components

The box model treats layout elements in an HTML page as rectangular boxes, that is, containers holding content.

The CSS box model is essentially a box that wraps HTML elements around it, including border, margin, padding, and actual content.

- `border`

- `content`

- `padding`: distance between border and content.

- `margin`: distance between boxes.

#### Border

Border width, border style, and border color.

```
border : border-widtn || border-style || border-color
```

Border, style, and color have no required order.

```
border-top: 1px solid red;
border-bottom: 5px solid red;
border-left: 5px solid yellow;
border-right: 5px solid purple;
```

Thin borders for tables:

```
/* Merge adjacent borders. */
 border-collapse: collapse;
```

- Borders affect the actual box size. For example, a 200 x 200px box with a 10px border becomes 220 x 220px in actual size.

#### Padding

| Number of values | Meaning |
| --- | --- |
| `padding: 5px;` | 5px on all sides |
| `padding: 5px 10px;` | 5px top/bottom, 10px left/right |
| `padding: 5px 10px 20px;` | 5px top, 10px left/right, 20px bottom |
| `padding: 5px 10px 20px 30px;` | 5px top, 10px right, 20px bottom, 30px left, clockwise |

- Padding also affects the actual box size. For example, a 200 x 200px box with 10px padding becomes 220 x 220px in actual size.

- If no `width` value is specified, the box width will not change.

```
h1 {
    /* h1 has no width set here,
    so its length defaults to the body width.
    padding only changes height and not width. */
    height: 40px;
    padding: 30px;
}
h2 {
    /* Comparison:
    h2 has width set here,
    so padding changes width. */
    height: 40px;
    padding: 30px;
    width: 1000px;
}
```

#### Margin

| Number of values | Meaning |
| --- | --- |
| `margin: 5px;` | 5px on all sides |
| `margin: 5px 10px;` | 5px top/bottom, 10px left/right |
| `margin: 5px 10px 20px;` | 5px top, 10px left/right, 20px bottom |
| `margin: 5px 10px 20px 30px;` | 5px top, 10px right, 20px bottom, 30px left, clockwise |

This is exactly the same as `padding`.

Use `margin` to horizontally center a block-level box:

- The box must have a specified width.

- Left and right margins are both set to `auto`.

- For inline elements or inline-block elements, add `text-align: center` to the parent element.

#### Margin Collapsing

- Vertical margin merging between adjacent block elements: when two vertically adjacent block elements, meaning siblings, meet, and the upper element has `margin-bottom` while the lower element has `margin-top`, the vertical spacing between them takes the larger of the two values.

- Vertical margin collapse for nested block elements: for two nested block elements, meaning parent and child, if the parent has `margin-top` and the child also has `margin-top`, then the parent's `margin-top` takes the larger of the two values, and the child will not have a separate top margin from the parent. This only appears on the first child element inside the parent.

How to solve it:

- Define a top border for the parent.

- Define top padding for the parent.

- Add `overflow: hidden` to the parent.

#### Clearing Default Margins and Padding

Many page elements have default margin and padding, and different browsers may differ. Therefore, before layout, clear the margins and padding of page elements.

```
* {
    padding: 0;
    margin: 0;
}
```

For inline elements, try to set only left and right margin/padding, because top and bottom settings do not take effect. If top/bottom spacing is really needed, use `display: block;` or `display: inline-block;`.

### Rounded Borders

```
border-radius:length;
```

`length` is the radius of the circle tangent to the rectangle corner. It can be a number or percentage. With percentages, the result is often an ellipse tangent to the rectangle corner, where the semi-major and semi-minor axes are percentages of width and height. If the box is square, it becomes a circle.

- If it is a square, `length = half the height` or `50%` can draw a circle.

- If it is a rectangle, `length = half the height` can create two semicircles plus a rectangle.

- This property is shorthand. It can take four values in the order top-left, top-right, bottom-right, bottom-left. Separate properties include `border-top-left-radius`, `border-top-right-radius`, `border-bottom-right-radius`, and `border-bottom-left-radius`.

### Box Shadow

```
box-shadow: h-shadow v-shadow blur spread color inset;
```

| Value | Description |
| --- | --- |
| h-shadow | Required. Horizontal shadow position. Positive values place the shadow to the right, negative values to the left |
| v-shadow | Required. Vertical shadow position. Positive values place the shadow below, negative values above |
| blur | Optional. Blur distance |
| spread | Optional. Shadow size |
| color | Optional. Shadow color |
| inset | Optional. Changes outer shadow, or outset, to inner shadow |

Shadows do not occupy space and do not affect layout.

### Text Shadow

```
text-shadow: h-shadow v-shadow blur color;
```

| Value | Description |
| --- | --- |
| h-shadow | Required. Horizontal shadow position. Positive values place the shadow to the right, negative values to the left |
| v-shadow | Required. Vertical shadow position. Positive values place the shadow below, negative values above |
| blur | Optional. Blur distance |
| color | Optional. Shadow color |

## CSS Float

### Float

#### Three Traditional Web Layout Methods

The essence of web layout is using CSS to arrange boxes and place them in the right positions.

- Normal flow, or standard flow.

- Float.

- Positioning.

A typical web page uses all three layout methods.

Float can change the default arrangement.

The most typical application is making multiple block-level elements appear in one row without gaps.

**Use standard flow for vertical arrangement of multiple block-level elements, and float for horizontal arrangement.**

#### Normal Flow

Arrange elements according to the default behavior of tags.

- Block-level elements occupy one line and are arranged from top to bottom.

- Inline elements are arranged from left to right and automatically wrap at the edge of the parent element.

#### Float

`float` is used to create a floating box and move it to one side until its left or right edge touches the containing block or another floated box.

```
selector {
    float: property;
}
```

| Value | Description |
| --- | --- |
| none | Element does not float, default |
| left | Floats left |
| right | Floats right |

#### Characteristics of Float

- Floated elements leave normal flow. The floated box no longer keeps its original position. It moves to the upper layer and may cover the next normal-flow box.

- Floated elements display on one line and align at the top. If the parent cannot contain them, they wrap to a new line.

- Floated elements gain characteristics of inline-block elements.

To constrain floated elements, page layout usually does this:

- Use a standard-flow parent element to arrange vertical positions, and float child elements inside it to arrange horizontal positions.

- Set box size first, then set box position.

#### Notes on Float Layout

- Combine float with standard-flow parent boxes.

Use standard-flow parent elements to arrange vertical positions first, then use float on child elements inside them for horizontal positions.

- If one element floats, theoretically its sibling elements should also float. A floated box only affects the following standard-flow boxes. For example:

When three `div`s A, B, and C are all in standard flow, they are arranged vertically. When A floats, B is standard flow, and C floats, B is covered by A, and C still appears below B vertically.

### Clearing Float

In many cases, it is inconvenient to give a parent box a fixed height. Since floated child boxes do not occupy space, when the parent has no height and the children float, the following standard-flow boxes may be partially covered by the current child boxes. Therefore, we need to clear floats.

The essence of clearing floats is clearing the effects caused by floated elements.

```
selector {
    clear: value;
}
```

| Value | Description |
| --- | --- |
| left | No floated elements allowed on the left, clears the effect of left floats |
| right | No floated elements allowed on the right, clears the effect of right floats |
| both | Clears the effects of floats on both sides, most common and basically the one used |

Strategy for clearing floats: close the float.

- Extra tag method, also called wall method.

- Add `overflow` to the parent.

- Add an `after` pseudo-element to the parent.

- Add double pseudo-elements to the parent.

#### Extra Tag Method

Add an empty tag at the end of floated elements, for example:

```
<div style="clear:both"></div>
or
<br />
```

Example:

```
<style>
    .clear {
        clear: both;
    }
</style>
<!-- Add an empty tag inside body.
Other class styles are omitted. -->
<body>
    <div class="box">
        <div class="one"></div>
        <div class="two"></div>
        <!-- The newly added tag must be a block-level element or br. -->
        <div class="clear"></div>
    </div>
    <div class="footer"></div>
</body>
```

- Advantage: easy to understand and write.

- Disadvantage: adds many meaningless tags and weakens structure.

#### Add overflow to the Parent

```
selector {
    overflow: hidden/auto/scroll;
}
```

- Advantage: concise code.

- Disadvantage: overflow content cannot be displayed.

#### Add an after Pseudo-Element to the Parent

This is an upgraded version of the extra tag method.

```
<style>
    .clearfix::after {
            content: "";
            display: block;
            height: 0;
            clear: both;
            visibility: hidden;
        }

        .clearfix {
            /* IE6 and IE7 only. */
            *zoom: 1;
        }
</style>

If the original parent class is box, write it as:

<div class="box clearfix">
    Child elements
</div>
```

- Advantage: no extra tags, simpler structure.

- Disadvantage: must support older browsers.

#### Add Double Pseudo-Elements to the Parent

```
<style>
        .clearfix::before,
        .clearfix::after {
            content: "";
            display: table;
        }

        .clearfix::after {
            clear: both;
        }

        .clearfix {
            /* IE6 and IE7 only. */
            *zoom: 1;
        }
</style>

If the original parent class is box, write it as:

<div class="box clearfix">
    Child elements
</div>
```

- Advantage: cleaner code.

- Disadvantage: must support older browsers.

## PS Slicing

### Common Image Formats

- `jpg`: preserves color information well, high-definition, supports many colors, often used for product images.

- `gif`: can store at most 256 colors. Usually used for simple graphics or text. It can preserve transparent backgrounds and animation effects, often used for small animated image effects.

- `png`: combines the advantages of gif and jpg. It has rich storage forms and can preserve transparent backgrounds.

- `PSD`: Photoshop-specific format. It can store layers, channels, masks, and other design draft information. Text can be copied, images obtained, and sizes and distances measured directly from it.

## CSS Positioning

### Why Positioning Is Needed

The following cases cannot be handled by standard flow:

- An element can move freely inside a box and overlap other boxes.

- When the window scrolls, a box stays fixed on the screen, as a floating panel.

So positioning is needed.

- Float can make multiple block boxes display without gaps, commonly used for horizontal box arrangement.

- Positioning lets a box move freely inside a box or stay fixed at a certain screen position, and it can overlap other boxes.

Positioning is also arranging boxes, moving boxes according to positioning rules.

### Components of Positioning

Positioning = positioning mode + edge offset.

#### Positioning Mode

Set with the CSS `position` property. It has four values:

| Value | Meaning |
| --- | --- |
| static | Static positioning |
| relative | Relative positioning |
| absolute | Absolute positioning |
| fixed | Fixed positioning |

#### Edge Offset

The positioned box moves to its final position using four properties: `top`, `bottom`, `left`, and `right`.

| Edge offset property | Example | Description |
| --- | --- | --- |
| top | `top: 80px;` | Top offset, distance relative to the upper edge of the parent element |
| bottom | `bottom: 80px;` | Bottom offset, distance relative to the lower edge of the parent element |
| left | `left: 80px;` | Left offset, distance relative to the left edge of the parent element |
| right | `right: 80px;` | Right offset, distance relative to the right edge of the parent element |

#### Static Positioning

The default positioning method. It means no positioning.

```
selector {
    position: static;
}
```

It is arranged according to standard flow and has no edge offset.

Rarely used.

#### Relative Positioning

When an element moves, it moves relative to its original position.

```
selector {
    position: relative;
}
```

It still occupies its original position in standard flow and does not affect following boxes. For example, if two boxes were originally stacked vertically and the upper box moves down, the lower box does not move, so it may be covered.

#### Absolute Positioning

With absolute positioning, an element moves relative to its ancestor element.

```
selector {
    position: absolute;
}
```

- If there is no ancestor element, or no ancestor element has positioning, it is positioned relative to the browser.

- If an ancestor element has positioning, it moves relative to the nearest positioned ancestor.

- Absolute positioning no longer occupies the original position, meaning it leaves normal flow.

#### Absolute Child, Relative Parent

If the child uses absolute positioning, the parent should usually use relative positioning.

- The child uses absolute positioning and does not occupy space, so it can be placed anywhere inside the parent without affecting sibling boxes.

- The parent box needs positioning to constrain the child box inside it.

- During layout, the parent box needs to occupy position, so it should use relative positioning.

#### Fixed Positioning

Fixed positioning fixes an element relative to the browser viewport. It is mainly used when an element should not move while the page scrolls, such as a floating panel.

```
selector {
    position: fixed;
}
```

- Moves the element relative to the browser viewport.

- Does not occupy its original position.

It can also be understood as a special absolute positioning mode.

##### Tip: Fixing an Element to the Right of the Layout Center

- Set the fixed-position box to `left: 50%;`, moving it to half of the browser viewport, which can also be treated as half of the layout center.

- Set `margin-left` to half of the layout center width so it moves another half center-width distance.

Example:

```
<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
        .page {
            height: 1000px;
            width: 1200px;
            margin: auto;
            background-color: bisque;
        }

        .fixed {
            position: fixed;
            top: 200px;
            left: 50%;
            margin-left: 600px;
            width: 100px;
            height: 100px;
            background-color: cornflowerblue;
        }
    </style>
</head>

<body>
    <div class="page">
        <div class="fixed">fixed</div>
    </div>
</body>

</html>
```

#### Sticky Positioning

Sticky positioning combines relative and fixed positioning. The effect is that as the page scrolls, an element first scrolls normally and then becomes fixed after reaching a certain position.

```
selector {
    position: sticky;
    top: 10px;
}
```

- Moves relative to the browser viewport, like fixed positioning.

- Occupies its original position, like relative positioning.

- One of `top`, `left`, `right`, or `bottom` must be added for it to work. This indicates where it becomes fixed.

It is not commonly used because compatibility is poor. This effect is usually implemented with JavaScript.

#### Positioning Summary

| Positioning mode | Leaves normal flow | Movement reference | Common |
| --- | --- | --- | --- |
| Static | No | Cannot use edge offset | Rare |
| Relative | No | Moves relative to itself | Common |
| Absolute | Yes | Positioned parent element | Common |
| Fixed | Yes | Browser viewport | Common |
| Sticky | No | Browser viewport | Rare at this stage |

#### Stacking Order: z-index

When boxes overlap, `z-index` controls the order on the z-axis, meaning layers.

```
selector {
    z-index: 1;
}
```

- The value can be a positive integer, negative value, or 0. The default is `auto`. The larger the value, the higher the box.

- If values are the same, later elements in source order appear on top.

- `z-index` cannot have a unit.

- Only positioned boxes have the `z-index` property.

#### Positioning Extensions

##### Centering an Absolutely Positioned Box

```
position: absolute;
/* Horizontally centered. */
left: 50%;
margin-left: -half of width
/* Vertically centered. */
top: 50%;
margin-top: -half of height
```

##### Special Characteristics of Positioning

Absolute and fixed positioning are similar to float:

- Inline elements with absolute or fixed positioning can directly set height and width.

- Block-level elements with absolute or fixed positioning default to content size if no width and height are set.

##### Boxes Leaving Normal Flow Do Not Trigger Margin Collapse

Absolute and fixed positioning do not trigger margin collapse.

##### Positioning Covers Standard-Flow Text

Floating elements do not cover text or images; they only cover boxes.

## Web Layout Summary

- Standard flow:

Used to arrange boxes vertically or horizontally. Use standard flow for vertical block-level boxes.

- Float:

Allows multiple block-level elements to appear in one row or align boxes left and right. Use float layout for horizontally displayed block boxes.

- Positioning:

Its biggest characteristic is layering. It allows multiple boxes to overlap in front and back. If an element needs to move freely inside a box, use positioning.

## Showing and Hiding Elements

- `display`

- `visibility`

- `overflow`

### display

- `display: none;` hides the object and does not preserve its position.

- `display: block;` shows the element and also means converting it to a block-level element.

Very commonly used.

### visibility

- `visibility: hidden;` hides the element and preserves its position.

- `visibility: visible;` shows it.

### overflow

- `overflow: visible;` shows overflowing content, default.

- `overflow: hidden;` hides overflowing content.

- `overflow: scroll;` displays overflow with scrollbars, and scrollbars are shown even when there is no overflow.

- `overflow: auto;` hides scrollbars when there is no overflow and shows scrollbars when there is overflow.

Use carefully with positioned boxes, because extra parts may be hidden, including the part positioned outside the box.
