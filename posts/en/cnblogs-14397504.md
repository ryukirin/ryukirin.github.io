Last week, my professor finally could not stand my lack of progress anymore, so I picked a dialogue-system project and tried to run it. My own computer was too weak to use a GPU, and CPU training would take about 30 hours. After I reported this to my professor, he recommended Colaboratory. This was my first time using it, so I wrote down the process for future reference.

## DeepQA

GitHub repository: [Deep QA](https://github.com/Conchylicultor/DeepQA)

This work tries to reproduce the results of [A Neural Conversational Model](http://arxiv.org/abs/1506.05869) (aka the Google chatbot). It uses a RNN (seq2seq model) for sentence predictions. It is done using python and TensorFlow.

## Set up Colaboratory

Reference: [How to use Google's AI tool Colaboratory](https://www.jianshu.com/p/e6f1058614c0?isappinstalled=0)

Note: using Colaboratory from mainland China requires additional network access.

Log in to Google Drive and click "New".

![](/assets/posts/cnblogs-14397504/image-01.png)

Click "More" -> "Connect more apps". In my screenshot, I had already connected it, so it is already visible.

![](/assets/posts/cnblogs-14397504/image-02.png)

On the page that opens, search for Colaboratory and connect it.

After it is connected, it will look like the screenshot above. Click Google Colaboratory, and the following page will open.

![](/assets/posts/cnblogs-14397504/image-03.png)

Click "Edit" -> "Notebook settings", choose "GPU", and then click "Save".

![](/assets/posts/cnblogs-14397504/image-04.png)

![](/assets/posts/cnblogs-14397504/image-05.png)

On the right side, click the triangle next to "Connect" -> "Connect to hosted runtime".

![](/assets/posts/cnblogs-14397504/image-06.png)

## Upload the folder to Google Drive

Click "New" -> "Folder upload".

![](/assets/posts/cnblogs-14397504/image-07.png)

## Connect to Google Drive

Enter the following code in a notebook cell:

```
!apt-get install -y -qq software-properties-common python-software-properties module-init-tools
!add-apt-repository -y ppa:alessandro-strada/ppa 2>&1 > /dev/null
!apt-get update -qq 2>&1 > /dev/null
!apt-get -y install -qq google-drive-ocamlfuse fuse
from google.colab import auth
auth.authenticate_user()
from oauth2client.client import GoogleCredentials
creds = GoogleCredentials.get_application_default()
import getpass
!google-drive-ocamlfuse -headless -id={creds.client_id} -secret={creds.client_secret} < /dev/null 2>&1 | grep URL
vcode = getpass.getpass()
!echo {vcode} | google-drive-ocamlfuse -headless -id={creds.client_id} -secret={creds.client_secret}
```

After running it, click the link in the output to authorize and verify. The result looks like this:

![](/assets/posts/cnblogs-14397504/image-08.png)

Mount Google Drive. Create a new code cell and enter:

```
!mkdir -p drive
!google-drive-ocamlfuse drive
```

![](/assets/posts/cnblogs-14397504/image-09.png)

To view your cloud files, run:

```
! ls drive/
```

![](/assets/posts/cnblogs-14397504/image-10.png)

## Run the code

At the time, Colaboratory used TensorFlow 2 by default, which caused errors when running this project. It needed to be switched to a 1.x version first.

Create a new code cell and run:

```
%tensorflow_version 1.10.0
```

![](/assets/posts/cnblogs-14397504/image-11.png)

Also, the project expects its data files to be found from the root directory by default. When running it on Colaboratory, the files cannot be found unless the code is adjusted.

Open `chatbot/chatbot.py` directly from Google Drive with Text Editor, and change line 152 to:

```
self.args.rootDir = os.getcwd() + '/drive/DeepQA-master'  #for colab run
            #self.args.rootDir = os.getcwd()
```

Download the `punkt` data in advance:

```
import nltk
nltk.download('punkt')
```

![](/assets/posts/cnblogs-14397504/image-12.png)

Run the code to train the model. This takes about two hours.

```
! python3 drive/DeepQA-master/main.py
```

![](/assets/posts/cnblogs-14397504/image-13.png)

Run a test:

```
! python3 drive/DeepQA-master/main.py --test
```

![](/assets/posts/cnblogs-14397504/image-14.png)

The result is shown in `save / model / samples_predictions.txt`.

![](/assets/posts/cnblogs-14397504/image-15.png)

If you want to chat with the model interactively:

```
! python3 drive/DeepQA-master/main.py --test interactive
```

## Additional notes: bugs

### E: Package 'python-software-properties' has no installation candidate

At one point, I accidentally ran the Google Drive connection cell again and got this error:

![](/assets/posts/cnblogs-14397504/image-16.png)

I referred to [E: Package 'python-software-properties' has no installation candidate](https://www.jianshu.com/p/5b62ff3d0a13), and fixed it by running:

```
sudo apt-get install python-software-properties
```

### Colaboratory disconnects

This did not bother me too much. I occasionally clicked around on the page, and the training still completed successfully.

There are also blog posts that discuss how to solve this issue, such as [How to keep Google Colab from disconnecting](https://blog.csdn.net/liupeng19970119/article/details/105625334).
