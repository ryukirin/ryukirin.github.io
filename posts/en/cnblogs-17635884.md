## Chapter 6: Arrays

### 6.5 Array sorting algorithms

#### Bubble sort

One of the most commonly used array sorting algorithms. It always moves smaller numbers forward and larger numbers backward, like bubbles rising in water, hence the name.

- Basic idea

Compare adjacent element values. If the condition is satisfied, swap the values.

Move smaller elements toward the front of the array and larger elements toward the back.

The diagram makes it clear.

![](/assets/posts/cnblogs-17635884/image-01.gif)

Image source: Runoob.

After the first pass, the largest element is already at the bottom. In the second pass, comparison only needs to go up to the second-to-last element.

Because elements are compared pairwise, the number of passes is length minus one. For example, with three numbers, pairwise comparison only needs two passes.

- Algorithm example

Bubble sort is implemented with a double loop.

The outer loop controls the number of sorting passes. It is usually array length minus one, because in the final loop only one array element remains and no comparison is needed.

- The inner loop mainly compares each pair of adjacent elements in the array to determine whether to swap them.

Algorithm implementation.

#### Direct selection sort

Slightly faster than bubble sort.

- Basic idea

Compare the specified sorting position with other array elements one by one. If the condition is met, swap element values.

Difference from bubble sort:

It does not swap adjacent elements, but swaps the element at the specified sorting position.

- Algorithm example

In each pass, select the smallest or largest element and place it at the end.

- Algorithm implementation

#### Reverse sorting

This simply reverses the array order.

- Basic idea

Swap the last element with the first element, the second-to-last with the second, and so on.

- Algorithm implementation

    public void bubbleSort(int[] arr){
        // Outer loop controls the number of sorting passes.
        for (int i=0; i < arr.length - 1; i++){
            // Inner loop compares adjacent elements.
            for (int j=0; j < arr.length-(i+1); j++){
                // Compare adjacent elements. If the left is greater than the right, swap.
                if (arr[j]>arr[j+1]){
                    int temp = arr[j];
                    arr[j] = arr[j+1];
                    arr[j+1] = temp;
                }
            }
        }
        /*
        This part is very common, so extract it into a function.
        for (int i : arr){
            System.out.print(i + " ");
        }
        */
    }

    public void selectSort(int[] arr) {
        int index;
        // Outer loop controls the number of sorting passes.
        for (int i = 1; i < arr.length; i++){
            // Reset the position before each pass.
            index = 0;
            // Since index starts at 0, compare from arr[1].
            for (int j=1; j <= arr.length-i; j++){
                // Index of current maximum value.
                if (arr[j] > arr[index]){
                    index = j;
                }
            }
            // Swap the maximum value with the last unsorted element.
            int temp = arr[arr.length-i];
            arr[arr.length-i] = arr[index];
            arr[index] = temp;
        }
    }

    public void reverseSort(int[] arr){
        int temp;
        for (int i=0; i<arr.length / 2; i++){
            temp = arr[i];
            arr[i] = arr[arr.length-(i+1)];
            arr[arr.length-(i+1)] = temp;
        }
    }

    public void showArr(int[] arr){
        for (int i : arr){
            System.out.print(i + " ");
        }
    }
}
