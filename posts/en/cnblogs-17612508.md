## Chapter 3: Java language basics

### 3.4 Operators

#### Assignment operator

`=`. If an expression contains more than one `=`, it is processed from the rightmost `=`.

#### Arithmetic operators

- Addition: `+`

- Subtraction: `-`

- Multiplication: `*`

- Division: `/`; 0 cannot be used as the divisor.

- Remainder: `%`

#### Increment and decrement

- `++a`: add 1 first, then use `a`. `--a`: subtract 1 first, then use it.

- `a++`: use `a` first, then add 1. `a--`: use `a` first, then subtract 1.

#### Comparison operators

The result is Boolean. This part is simple and common across languages, so I skipped details.

#### Logical operators

The result is Boolean. This is also simple, so I skipped details.

- `&` and `&&`: AND

- `||`: OR

- `!`: NOT

##### Difference between `&` and `&&`

- Short circuit: in a logical expression, if the value of the entire expression can be inferred from the left-side expression, it is called short-circuiting. In other words, the final result can be known without evaluating every part.

- `&`: non-short-circuit operator. Even if the final result can be obtained after evaluating only part of the expression, it still evaluates all parts.

- `&&`: short-circuit operator. If evaluating part of the expression already determines the final result, evaluation stops.

Examples:

`(false || false) & (!false)` evaluates both the front and back parts.

`(false || false) && (!false)` only evaluates the first part.

#### Bitwise operators

Bitwise operations operate entirely at the bit level.

Integer data is represented in memory in binary form. For example, 7 is `00000000 00000000 00000000 00000111`.

The highest bit on the left is the sign bit: 1 for negative, 0 for positive.

Negative numbers are represented using two's complement. For example, -8 is `11111111 11111111 11111111 11111000`.

About original code, inverse code, and complement code: [https://www.cnblogs.com/zhangziqiu/archive/2011/03/30/ComputerCode.html](https://www.cnblogs.com/zhangziqiu/archive/2011/03/30/ComputerCode.html)

- Bitwise AND `&`: logical AND on each bit.

- Bitwise OR `|`: logical OR on each bit.

- Bitwise NOT `~`: logical NOT on each bit.

- Bitwise XOR `^`: same bits become 0, different bits become 1.

- Shift operations: only apply to `byte`, `short`, `char`, `int`, and `long`.

`<<`: left shift. The whole value shifts left, and 0 is added on the right.

- `>>`: right shift. If the highest bit is 0, 0 is added on the left after shifting. If the highest bit is 1, 1 is added on the left after shifting.

- `>>>`: unsigned right shift. Regardless of whether the highest bit is 0 or 1, 0 is added after shifting.

- Shifts can implement `*2` and `/2`. For example, `y<<1` is `y*2`, and `y>>1` is `y/2`.

`y>>n` is `y / 2^n`.

#### Ternary operator

`condition ? value1 : value2;`

If the condition is true, take value 1. Otherwise, take value 2.

Example: `boolean b = 20<45?true:false;`, so `b=true`.

#### Operator precedence

Parentheses > positive/negative signs > increment/decrement and `!` > arithmetic operations > shift operations > comparison operations > bitwise operations > logical operations > assignment operations.

```
Java3_4 {
    public static void main(String[] args){
        int a, b, c;
        a = 15;
        /*
         This works but is not recommended. Better write:
         c = a + 4;
         b = a + 4;
         Or, if b is logically connected to c, write:
         b = c;
         */
        c = b = a + 4;
        System.out.println("c值为：" + c);
        System.out.println("b值为：" + b);

        float number1 = 45.56f;
        int number2 = 152;
        System.out.println("和为：" + (number1 + number2));
        System.out.println("差为：" + (number1 - number2));
        System.out.println("积为：" + (number1 * number2));
        System.out.println("商为：" + (number1 / number2));

        int m, n = 1, o = 1;
        // Here n++ uses the original value of n first, then adds 1.
        m = n++;
        // So the value assigned to m is still 1, before the +1.
        System.out.println(m);
        // Now n has already increased by 1, so the output is 2.
        System.out.println(n);
        // ++o adds 1 first, then uses o.
        m = ++o;
        // So m is 2 after the +1.
        System.out.println(m);

        boolean flag = 20<45?true:false;
        System.out.println(flag);
    }
}
```

### 3.5 Data type conversion

- Low precision -> high precision: no overflow, always succeeds.

- High precision -> low precision: information loss, may fail.

#### Implicit type conversion

Automatically performed by the program, but only from lower precision to higher precision:

`byte < short < int < long < float < double`

#### Explicit type conversion

High precision can be forcibly converted to low precision, but information may be lost.

Format: `(targetType)value`

When assigning an integer value to `byte`, `short`, `int`, or `long`, it cannot exceed the range. Otherwise, explicit conversion is required.

```
public class Java3_5 {
    public static void main(String[] args){
        System.out.println("隐式类型转换");
        int x = 50;
        float y = x;
        System.out.println(y);
        double z = 89.897;
        System.out.println(x + z);

        System.out.println("显式类型转换");
        int a = (int)45.678;
        System.out.println(a);
        byte b = (byte)129;
        System.out.println(b);
    }
}
```
