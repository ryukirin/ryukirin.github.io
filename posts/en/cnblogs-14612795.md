### embedding

Word embedding, in simple terms, means converting text into a sequence of numbers. Numbers are easier for computers to process.

The process of word embedding is like creating a dictionary for the computer. The computer can use this dictionary to indirectly recognize text.

An embedding vector can also be understood as the vector representation of a word in a neural network.

For more details, see [Understanding the embedding layer](https://www.cnblogs.com/USTC-ZCC/p/11068791.html).

### embedding in PyTorch

The input is a list of indexes, and the output is the corresponding word embeddings.

```
torch.nn.Embedding(num_embeddings, embedding_dim, padding_idx=None,
                   max_norm=None,  norm_type=2.0,   scale_grad_by_freq=False, 
                   sparse=False,  _weight=None)
```

#### Parameters

- `num_embeddings` (`int`): the size of the vocabulary. For example, if 5,000 words appear in total, pass 5,000. The indexes are then `0-4999`.

- `embedding_dim` (`int`): the dimension of the embedding vector, meaning how many dimensions are used to represent one symbol.

- `padding_idx` (`int`, optional): the padding ID. For example, if the input length is 100 but sentence lengths vary, a unified value is needed to pad the remaining positions. This parameter specifies that value. When the network sees the padding ID, it does not compute its relation with other symbols. It is initialized to 0.

- `max_norm` (`float`, optional): maximum norm. If an embedding vector's norm exceeds this threshold, it is renormalized.

- `norm_type` (`float`, optional): specifies which norm is used for computing and comparing against `max_norm`. The default is the L2 norm.

- `scale_grad_by_freq` (`boolean`, optional): scales gradients by the frequency of words in the mini-batch. The default is `False`.

- `sparse` (`bool`, optional): if `True`, gradients related to the weight matrix become sparse tensors. The default is `False`.

#### Example

```
import torch
import numpy as np

# Create an embedding layer. Vocabulary size is 13, embedding dimension is 3.
embed = torch.nn.Embedding(13,3)
# Sentence pairs: ['I am a boy.','How are you?','I am very lucky.']
# batch = [['i','am','a','boy','.'],['i','am','very','lucky','.'],['how','are','you','?']]

# Convert the words in the batch into dictionary indexes.
# Here the dictionary is created from these words first.
# batch = [[2,3,4,5,6],[2,3,7,8,6],[9,10,11,12]]

# Actual length of each sentence
# lens = [5,5,4]

# Add EOS marker with index=0.
# batch = [[2,3,4,5,6,0],[2,3,7,8,6,0],[9,10,11,12,0]]

# Actual sentence lengths after adding EOS at the end
lens = [6,6,5]

# After padding, PAD uses index=1.
batch = [[2,3,4,5,6,0],[2,3,7,8,6,0],[9,10,11,12,0,1]]

# At each RNN step, one word from each sample is used as input.
# Each input contains batch_size samples.
# Therefore, the outer list should be time steps, and the inner list should be batch_size.
# That is, [seq_len,batch_size].
batch = np.transpose(batch)

batch=torch.LongTensor(batch)

embed_batch = embed(batch)

print(embed_batch)
```
