# Watermelon Book | Datawhale October (3)

## Decision trees

### Basic process

Decision making is performed based on a tree structure.

A decision tree usually contains one root node, several internal nodes, and several leaf nodes.

Leaf nodes correspond to decision results. Every other node corresponds to an attribute test, and the path from the root node to each leaf node corresponds to a sequence of decision tests.

The goal of decision-tree learning is to produce a decision tree with strong generalization ability, meaning strong ability to handle unseen instances.

From a logic perspective: a combination of many `if else` statements.

From a geometry perspective: partitioning the feature space according to some criterion.

Final goal: make the samples increasingly "pure".

```
# Input: training set D = {(x_1,y_1),(x_1,y_1),...,(x_m,y_m)};
#        attribute set A = {a_1,a_2,...,a_d}.
# Process: recursive process
'''
TreeGenerate(D,A):
    Generate node
    if all samples in D belong to class C:
        Mark node as a leaf node of class C
        return
    if A = NULL or all samples in D have the same values on A:
        Mark node as a leaf node, with the class set to the majority class in D
        return
    else:
        Choose the optimal splitting attribute a_*
        for each value a_*_v of a_*:
            Create a branch for node
            Let Dv be the subset of samples whose value on a_* is a_*_v
            if Dv = NULL:
                Mark the branch node as a leaf node, with the class set to the majority class in D
                return
            else:
                Use TreeGenerate(Dv,A/{a_*}) as the branch node
    return
'''
```

### Split selection

**Self-information**

\[I(X)=-\log_{b}{p(x)}
\]

When `b = 2`, the unit is bit. When `b = e`, the unit is nat.

Combined with the formula below, it can also be written as:

\[I(D)=-\log_{b}{p_{k}}
\]

**Information entropy**, the expectation of self-information, is one of the most commonly used measures of sample-set purity.

Suppose the proportion of class `k` samples in current sample set `D` is \(p_{k}(k=1,2,...,|\mathcal{Y}|)\). The information entropy of `D` is defined as:

\[H(D)=E[I(D)]=\operatorname{Ent}(D)=-\sum_{k=1}^{|\mathcal{Y}|} p_{k} \log _{2} p_{k}
\]

The smaller \(\operatorname{Ent}(D)\) is, the higher the purity of `D`.

When computing information entropy, by convention, if \(p_k=0\), then \(p_k \log_b p_k=0\). When one value of \(X\) has probability 1, information entropy is minimized, meaning the result is most certain and the sample set is purest; its value is 0. When all values of \(X\) have equal probability, information entropy is maximized, meaning the result is most uncertain; its value is \(\log_b |D|\), where \(|D|\) denotes the number of possible values of `D`.

**Conditional entropy**: the expectation of the information entropy of `Y` with respect to probability distribution `X`.

It represents the uncertainty of `Y` after `X` is known.

\[H(Y|X)=\sum_{x}p(x)H(Y|X=x)
\]

From the perspective of a single attribute or feature `a`, suppose its possible values are \(\{a^{1},a^{2},...,a^{V}\}\), and \(D^v\) denotes the sample set where attribute `a` takes value \(a^v\in\{a^{1},a^{2},...,a^{V}\}\). \(\frac{|D^v|}{|D|}\) denotes its proportion. Then after the value of attribute `a` is known, the conditional entropy of sample set `D` is:

\[\sum_{v=1}^{V} \frac{\left|D^{v}\right|}{|D|} \operatorname{Ent}\left(D^{v}\right)
\]

**Information gain**: after the value of attribute or feature `a` is known, the amount by which uncertainty in `y` decreases. It is also the improvement in purity, namely information entropy minus conditional entropy.

\[\operatorname{Gain}(D, a)=\operatorname{Ent}(D)-\sum_{v=1}^{V} \frac{\left|D^{v}\right|}{|D|} \operatorname{Ent}\left(D^{v}\right)
\]

#### ID3 decision tree

An ID3 decision tree chooses the splitting attribute using information gain, selecting the attribute with the largest information gain.

\[a_{*}=\underset{a \in A}{\arg \max } \operatorname{Gain}(D, a)
\]

#### C4.5 decision tree

The information-gain criterion prefers attributes with too many possible values, such as IDs. This happens because each value contains too few samples. To reduce the negative effect of this preference, C4.5 uses gain ratio instead of information gain.

**Gain ratio**

\[\operatorname{Gain_{-}\operatorname{ratio}}(D, a)=\frac{\operatorname{Gain}(D, a)}{\operatorname{IV}(a)}
\]

where

\[\operatorname{IV}(a)=-\sum_{v=1}^{V} \frac{\left|D^{v}\right|}{|D|} \log _{2} \frac{\left|D^{v}\right|}{|D|}
\]

This is called the intrinsic value of attribute \(a\). The larger the number of possible values \(V\) for \(a\), the larger \(\operatorname{IV}(a)\) usually is. However, gain ratio prefers attributes with fewer possible values.

Therefore, C4.5 does not completely replace information gain with gain ratio. Instead, it first selects attributes whose information gain is above average, and then chooses the one with the highest gain ratio among them.

#### CART decision tree

**Gini value**

The Gini value is the probability that two samples randomly selected from sample set \(D\) have inconsistent class labels. Therefore, the smaller the Gini value, the smaller the probability of hitting different classes, and naturally the higher the purity.

\[\begin{aligned}
\operatorname{Gini}(D) &=\sum_{k=1}^{|\mathcal{Y}|} \sum_{k^{\prime} \neq k} p_{k} p_{k^{\prime}} \\
&=\sum_{k=1}^{|\mathcal{Y}|} p_{k}\left(1-p_{k}\right) \\
&=1-\sum_{k=1}^{|\mathcal{Y}|} p_{k}^{2}
\end{aligned}
\]

**Gini index**

The Gini index of attribute `a`:

\[\operatorname{Gini_{-}\operatorname{index}}(D, a)=\sum_{v=1}^{V} \frac{\left|D^{v}\right|}{|D|} \operatorname{Gini}\left(D^{v}\right)
\]

A CART decision tree selects the attribute with the smallest Gini index as the optimal splitting attribute.

\[a_{*}=\underset{a \in A}{\arg \text { min }} \operatorname{Gini_{-}\operatorname{index}}(D, a)
\]

The practical construction algorithm of a CART decision tree is:

- First, for every possible value \(v\) of each attribute \(a\), split data set \(D\) into two parts, \(a=v\) and \(a \neq v\), and compute the Gini index:

\[\operatorname{Gini_{-}\operatorname{index}}(D, a)=\frac{\left|D^{a=v}\right|}{|D|} \operatorname{Gini}\left(D^{a=v}\right)+\frac{\left|D^{a \neq v}\right|}{|D|} \operatorname{Gini}\left(D^{a \neq v}\right)
\]

- Then select the attribute and corresponding value with the smallest Gini index as the optimal splitting attribute and split point.

- Finally, repeat the two steps above until the stopping condition is met.

### Pruning

To prevent overfitting, decision trees mainly use pre-pruning and post-pruning.

- Pre-pruning: during decision-tree generation, estimate each node before splitting it. If splitting the current node cannot improve the generalization performance of the decision tree, stop splitting and mark the current node as a leaf node.

- Post-pruning: first generate a complete decision tree, then inspect leaf nodes from bottom to top. If replacing the subtree corresponding to a node with a leaf node improves generalization performance, perform the replacement.

The method for judging generalization improvement is the hold-out method, reserving part of the data as a validation set for evaluation.

Post-pruning has a small risk of underfitting, and its generalization is usually better than pre-pruning, but it has higher time cost.

### Continuous and missing values

#### Handling continuous values

So far, the discussion has focused on discrete attributes. When continuous attributes appear, they can be discretized.

The simplest method is binary splitting, using the midpoint of every pair of adjacent values as a split point.

For an example, see the [Pumpkin Book](https://datawhalechina.github.io/pumpkin-book/#/chapter4/chapter4?id=_47).

#### Handling missing values

First case: the attribute value is missing.

Given training set \(D\) and attribute `a`, let \(\tilde{D}\) denote the subset of samples in \(D\) that do not have missing values on attribute `a`.

Suppose attribute `a` has `V` possible values \(\{a^{1},a^{2},...,a^{V}\}\). Let \(\tilde{D}^{v}\) denote the subset of samples in \(\tilde{D}\) whose value on attribute `a` is \(a^v\), and let \(\tilde{D}_k\) denote the subset of samples in \(\tilde{D}\) that belong to class `k`. Suppose each sample `x` is assigned a weight \(w_x\). Define:

\[\rho=\frac{\sum_{x \in \tilde{D}} w_{x}}{\sum_{x \in D} w_{x}}
\]

\[\tilde{p}_{k}=\frac{\sum_{x \in \tilde{D}_{k}} w_{x}}{\sum_{x \in \widetilde{D}} w_{x}} \quad(1 \leq \mathrm{k} \leq|y|) \\
\]

\[\tilde{r}_{v}=\frac{\sum_{x \in \tilde{D}}^{v} w_{x}}{\sum_{x \in \widetilde{D}} w_{x}} \quad(1 \leq \mathrm{v} \leq \mathrm{V})
\]

For attribute `a`, \(\rho\) represents the proportion of samples without missing values, \(\tilde{p}_k\) represents the proportion of class `k` among samples without missing values, and \(\tilde{r}_v\) represents the proportion of samples without missing values whose value on attribute `a` is \(a^v\).

Therefore, the information-gain formula is generalized as:

\[\begin{array}{l}
\operatorname{Gain}(D, a)=\rho \times \operatorname{Gain}(\tilde{D}, a)=\rho \times\left(\operatorname{Ent}(\tilde{D})-\sum_{v=1}^{V} \tilde{r}_{v} \operatorname{Ent}\left(\tilde{D}^{v}\right)\right) \\
\operatorname{Ent}(\tilde{D})=-\sum_{k=1}^{|y|} \tilde{p}_{k} \log _{2} \tilde{p}_{k}
\end{array}
\]

Second case: after the splitting attribute is given, a sample's value on that attribute is missing.

- If sample \(\boldsymbol{x}\)'s value on splitting attribute \(a\) is known, assign \(\boldsymbol{x}\) to the child node corresponding to that value, and keep the sample weight as \(w_x\) in the child node.

- If sample \(\boldsymbol{x}\)'s value on splitting attribute \(a\) is unknown, assign \(\boldsymbol{x}\) to all child nodes at the same time, and adjust the sample weight in the child node corresponding to attribute value \(a^v\) to \(\tilde{r}_v \cdot w_x\). Intuitively, this is equivalent to assigning the same sample to different child nodes with different probabilities.

### Multivariate decision trees

The classification boundary generated by a decision tree has an obvious feature: it is axis-parallel. In other words, its classification boundary consists of several segments parallel to the coordinate axes. But when the true classification boundary of the learning task is complex, many segments are needed to approximate it well.

In many cases, an oblique split can simplify the model. This is what multivariate decision trees do.

In a multivariate decision tree, non-leaf nodes no longer test only a single attribute; instead, they test a linear combination of attributes. This means the learning process no longer searches for one optimal splitting attribute for each non-leaf node, but instead tries to build a suitable linear classifier.
