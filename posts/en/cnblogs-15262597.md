# Watermelon Book | Datawhale September (1)

## Introduction

### Opening notes

Condition -> result.

The main research topic of machine learning is algorithms that produce a model from data on a computer, that is, learning algorithms.

### Basic terminology

*Only terms that I personally did not fully understand are explained here.*

- Data set

- Instance / sample

- Attribute / feature

- Attribute value

- Attribute space / sample space / input space

- Feature vector: the coordinate vector of the point to which a sample belongs in the multidimensional space formed by attributes.

- Dimensionality: the number of attributes in the data set.

- Training data

- Training set

- Hypothesis: a potential pattern in the data.

- Ground truth: the underlying potential pattern itself.

- Learner: the model.

- Label: result information for a training sample.

- Label space / output space: the set of all labels, denoted by `Γ`.

- Example: an instance that has a label, usually represented as `(xi,yi)` for the i-th example, where `yi∈Γ` is the label of instance `xi`.

- Generalization: the ability of the learned model to apply to new samples.

- Distribution: the unknown rule, `D`, that all samples in the sample space are assumed to follow.

- Independent and identically distributed (i.i.d.): every sample we obtain is independently sampled from `D`.

### Hypothesis space

#### Version space

All possible hypotheses form the hypothesis space. After deleting hypotheses that differ from positive examples or match negative examples in the training set, the remaining hypotheses form the version space.

#### Induction

Specific -> general, which corresponds to generalization.

#### Deduction

General -> specific, which corresponds to specialization.

### Inductive bias

An inductive bias is the preference of a machine-learning algorithm for certain types of hypotheses during learning.

Principle: Occam's razor, choose the simplest one.

# If Table 1.1 only contains samples numbered 1 and 4, give the corresponding version space.
# Version space means finding hypotheses in the hypothesis space that match positive examples

import numpy
if __name__ == '__main__':
    # Use hexadecimal values to represent different types of watermelons.

    # In binary, one attribute can be represented by 00, 01, and 10.
    # Since binary is long, convert it to hexadecimal.
    # Dark green: 10, green: 11; slightly curled: 10, curled: 11; dull sound: 10, clear sound: 11.
    # The list below is the hypothesis space.
    melon = (0x3f, 0x3d, 0x3e, 0x37, 0x3b, 0x1f, 0x2f, 0x35, 0x36, 0x39, 0x3a, 0x1d, 0x1e, 0x2d,

    # The two samples are green-curled-clear and dark-green-slightly-curled-dull.
    # The former is a positive example, and the latter is a negative example.
    sample = (0x15,0x2a)
    # sum is a counter.
    sum = 0

        # So the version space contains hypotheses that are inconsistent with negative examples
        # and consistent with positive examples.

```
# Compared with using a single conjunction as the hypothesis representation,
# using disjunctive normal form gives the hypothesis space stronger expressive power.
# If a disjunctive normal form containing at most k conjunctions is used to express
# the hypothesis space for the watermelon classification problem in 1.1,
# estimate how many possible hypotheses there are.
# code:https://blog.csdn.net/qq_40273675/article/details/89856447
# This code uses the idea of 3*4*4 base hypotheses; another approach is 2*3*3.
# Since (A=a)∨(A=*) is equivalent to (A=*), after generating all hypotheses,
# redundant forms like this must be removed.
# Idea:
# combinations C(48,k) -> check whether there is * -> yes -> check whether the hypotheses
# other than * are contained by another hypothesis, e.g. (AAB)∨(AA*) -> yes -> delete
#                                                       no -> not redundant
#                              no -> not redundant

import re
import itertools as it

def get_combine_hyps(list_attr, k):
    # Combinations
    # Without Empty set
    list_hyps = []
    # There are three features.
    for value_attr0 in list_attr[0]:
        for value_attr1 in list_attr[1]:
            for value_attr2 in list_attr[2]:
                x = value_attr0 + value_attr1 + value_attr2
                list_hyps.append(x)
    list_com = list(it.combinations(list_hyps, k))  # combinations C48,k
    return list_com

def delete_repeated(list_in):
    # Remove redundancy.
    list_outer = []
    list_general = []
    flag_repeating = 0  # 1 means not redundant, 0 means redundant.

    for com_hyp in list_in:
        """Find generalized hypotheses in each disjunctive-normal-form group."""
        for a_hyp in com_hyp:
            # Check whether * exists.
            if "." in a_hyp:
                # If it exists, go to the next step and put the conjunction containing * into list_general.
                list_general.append(a_hyp)
            else:
                pass
        """Check whether the generalized hypothesis contains other hypotheses."""
        if len(list_general) == 0:
            flag_repeating = 1
        else:
            for b_hyp in com_hyp:
                for general in list_general:
                    # First exclude identical ones.
                    if general == b_hyp:
                        pass
                    else:
                        # re.search scans the whole string and returns the first successful match.
                        # If there is no match, it returns None.
                        # Returning None means there is no redundancy.
                        hyp_re = re.search(general, b_hyp)
                        if hyp_re is None:
                            flag_repeating = 1
                        else:
                            pass
                # Reset list_general; otherwise many terms containing * from nested loops are accumulated.
                list_general = []
        if flag_repeating == 1:
            list_outer.append(com_hyp)
            flag_repeating = 0

    return list_outer

def main(k):
    list_attribute = [["A", "B", "."], ["A", "B", "C", "."], ["A", "B", "C", "."]]
    hypothesis = get_combine_hyps(list_attribute, k)
    if k < 1 or k > 48:
        print("k must be a positive integer less than 48!")
    elif k == 1:
        print(len(hypothesis), hypothesis)
    else:
        hyps = delete_repeated(hypothesis)
        print(len(hyps), hyps)

if __name__ == "__main__":
    main(2)
```

## Model evaluation and selection

### Empirical error and overfitting

Error rate: if there are `a` incorrect samples among `m` samples, then the error rate is `E=a/m`, and the accuracy is `1−a/m`.

Error: the difference between predicted output and true output.

Empirical error: the learner's error on the training set, also called training error.

Overfitting: the learner treats properties of the training data itself as general properties of all potential samples, resulting in poor generalization.

### Evaluation methods

Test set.

- Hold-out method

Directly split the data set into two mutually exclusive subsets, while keeping the positive/negative ratio consistent.

- K-fold cross-validation

- Leave-One-Out, abbreviated LOO: each time, use only one sample as the test set and use all remaining samples as the training set.

- Bootstrapping

The probability that a sample is never selected after `m` sampling operations is `(1−1/m)^m`. Taking the limit:

\[\lim_{m \to \infty} (1-\frac{1}{m} )^{m} =\frac{1}{e}≈0.368
\]

### Performance measures

Mean squared error, the most commonly used one.

#### Error rate and accuracy

- Error rate: the proportion of errors.

- Accuracy: `1 - error rate`.

#### Precision, recall, and F1

- Confusion matrix: a classification-result table composed of true positives (TP), false positives (FP), true negatives (TN), and false negatives (FN).

- Precision `P`: among examples predicted as positive, the probability that they are truly positive.

- Recall `R`: among true positives, the probability that they are predicted as positive.

Generally speaking, these two metrics conflict: when one is high, the other tends to be low.

- Precision-recall curve, abbreviated P-R curve.

When comparing two models, the balance point can be considered.

- Balance point: the point where precision equals recall.

\[F1=\frac{2\times P\times R}{P+R}=\frac{2\times TP}{\text{total examples}+TP-TN}
\]

F1 is defined as the harmonic mean of precision and recall.

\[\frac{1}{F1}=\frac{1}{2}\cdot (\frac{1}{P}+\frac{1}{R})
\]

\(F_\beta\) is a weighted harmonic mean.

\[\frac{1}{F_\beta }=\frac{1}{1+\beta ^2}\cdot (\frac{1}{P}+\frac{\beta ^2}{R})
\]

\[F_\beta=\frac{(1+\beta ^2)\times P\times R}{(\beta ^2\times P)+R}
\]

Here `β>0` measures the relative importance of recall to precision. When `β>1`, recall has more influence. When `β<1`, precision has more influence. When `β=1`, it is the standard F1.

Across multiple confusion matrices, for example from multiple training runs or multiple data groups, precision and recall can be considered through macro-precision, macro-recall, and macro-F1.

\[macro-P=\frac{1}{n}\sum_{n}^{i=1}P_i
\]

\[macro-R=\frac{1}{n}\sum_{n}^{i=1}R_i
\]

\[macro-F1=\frac{2\times macro-P\times macro-R}{macro-P+macro-R}
\]

### Exercises

#### 2-1

A data set contains 1,000 samples, including 500 positive examples and 500 negative examples. It is split into a training set containing 70% of the samples and a test set containing 30% for hold-out evaluation. Estimate how many splitting methods there are.

Because the distribution needs to remain consistent, split `500 * 70%` positive examples and negative examples into the training set, and split `500 * 30%` positive examples and negative examples into the test set. Therefore, the number of splitting methods is \(C_{500}^{350}\times C_{500}^{350}=C_{500}^{150}\times C_{500}^{150}\).

#### 2-2

A data set contains 100 samples, half positive and half negative. Suppose the model produced by the learning algorithm predicts a new sample as the category with more training samples, and guesses randomly if the counts are equal. Give the evaluated error rates under 10-fold cross-validation and Leave-One-Out respectively.

In 10-fold cross-validation, because positive and negative examples each account for half, the error rate is 50%.

Leave-One-Out: because positive and negative examples each account for half, no matter which sample is removed, the remaining samples of the same class must be the minority. Therefore the prediction is always wrong, and the error rate is 1.

#### 2-3

If learner A has a higher F1 value than learner B, is A's BEP value also higher than B's?

In a P-R plot, every point can have an F1 value, but there is only one BEP, namely where `P=R`. So the two do not have a strong direct relationship.
