# Running GitHub code on Colab: failed attempt

## Background

The project is the code for the paper [Emotional Chatting Machine: Emotional Conversation Generation with Internal and External Memory](https://arxiv.org/abs/1704.01074).

Project repository: https://github.com/tuxchow/ecm

The original project requires:

- Python 2.7

- NumPy

- TensorFlow 0.12

This was too hard to set up on Colab, at least for me, so I looked for a higher-version alternative: [ECM-tf](https://github.com/tuxchow/ECM-tf).

This project requires:

- Python 3.5

- TensorFlow 1.0.1

- NumPy

In the end, this project was probably missing a file and did not run successfully. But the environment was set up, so I recorded the process.

## Using Colab and setting up the environment

### Import GitHub code into Colab

Mount Google Drive:

```
import os
os.chdir("/content/drive/My Drive")
```

Fetch the GitHub code:

```
!git clone https://github.com/tuxchow/ECM-tf.git
```

Enter the project folder:

```
cd ECM-tf/
------------
/content/drive/My Drive/ECM-tf
```

View the directory:

```
!ls
-------------
apply.py      emo_embedding  test_medium.py   train.py
data_dict.py  internal_mem   test.py	      train_test_split.py
data.py       model	     train_medium.py
ECM	      README.md      train_mini.py
```

### Change the Python version

Colab currently uses Python 3.7 by default, so the Python version needs to be changed to 3.5.

First install Python 3.5:

```
!apt-get install python3.5
```

Uninstall `pip` under Python 3.7 to avoid installing packages into the 3.7 environment later:

```
!python -m pip uninstall pip
```

Switch the environment to 3.5:

```
!update-alternatives --install /usr/local/bin/python3 python3 /usr/bin/python3.5 2
!update-alternatives --install /usr/local/bin/python3 python3 /usr/bin/python3.7 1
!python --version
--------------------
update-alternatives: renaming python3 link from /usr/bin/python3 to /usr/local/bin/python3
update-alternatives: using /usr/bin/python3.5 to provide /usr/local/bin/python3 (python3) in auto mode
Python 3.5.10
```

Install `pip` for Python 3.5:

```
!apt-get install python3-pip
```

### Install NumPy

Upgrade first, otherwise NumPy cannot be installed:

```
!python3.5 -m pip install --upgrade pip
```

Install NumPy:

```
!python3.5 -m pip install Numpy
```

### Install TensorFlow 1.0.1

The project requires TensorFlow 1.0.1.

So CUDA compatibility becomes a problem. The default CUDA needs to be removed and CUDA 8.0 installed instead.

```
!apt-get remove cuda
!apt-get autoremove cuda
!apt-get purge cuda
!apt-key del /var/cuda-repo-9-2-local/*.pub
!rm -rf /var/cuda-repo-8-0-local-ga2/
```

Install CUDA 8.0:

```
!sudo wget https://developer.nvidia.com/compute/cuda/8.0/Prod2/local_installers/cuda-repo-ubuntu1604-8-0-local-ga2_8.0.61-1_amd64-deb
!sudo dpkg -i cuda-repo-ubuntu1604-8-0-local-ga2_8.0.61-1_amd64-deb
!sudo apt-get update
!sudo apt-get -y install cuda-8-0
```

Check the current CUDA version:

```
!nvcc --version
--------------------
nvcc: NVIDIA (R) Cuda compiler driver
Copyright (c) 2005-2016 NVIDIA Corporation
Built on Tue_Jan_10_13:22:03_CST_2017
Cuda compilation tools, release 8.0, V8.0.61
```

Install TensorFlow 1.0.1:

```
!python3.5 -m pip install tensorflow==1.0.1
```

### Run the project

```
!python3.5 test.py -m ECM
```

Runtime error:

```
FileNotFoundError: [Errno 2] No such file or directory: 'data/data_test.pkl'
```

It should be caused by a missing file.
